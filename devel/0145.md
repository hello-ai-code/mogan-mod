# 任务 0145：表格中光标向右移动性能优化

## 问题

在 `TeXmacs/tests/tmu/0145.tmu` 的表格中，光标向右移动时性能非常差。

## 定位结果

### 瓶颈确认

通过 GUI 实测日志发现：

```
compute_env_rects took 240ms
apply_changes took 242ms
compute_env_rects took 260ms
apply_changes took 261ms
...
```

**结论**：`compute_env_rects()` 是绝对瓶颈，每次光标移动耗时 **240-277ms**。

`headless` 模式下 `apply_changes()` 不会被调用（`tm_server.cpp:217`），因此 `0145.scm` 自动化测试无法复现问题。

### 根因分析

`edit_interface.cpp:602` 的 `compute_env_rects()` 在处理 TABLE 时：
1. 遍历表格所有行和列（8x60=480 个单元格）
2. 对每个单元格调用 `eb->find_check_selection()`（内部调用 `find_box_path`）
3. 再次遍历所有单元格处理相邻关系

表格单元格的环境矩形（边框高亮）**不依赖于光标位置**，只取决于表格结构。但光标每次移动都会触发完整的重新计算。

## 优化方案

在 `edit_interface_rep` 中增加 `table_env_cache`（`hashmap<tree, rectangles>`）：

1. **`edit_interface.hpp`**：添加 `hashmap<tree, rectangles> table_env_cache`
2. **`edit_interface.cpp` 构造函数**：初始化缓存
3. **`compute_env_rects()`**：遇到 TABLE 时先查缓存，命中则直接返回；计算完成后保存到缓存
4. **`apply_changes()`**：当 `THE_TREE` 或 `THE_ENVIRONMENT` 变化时清空缓存
