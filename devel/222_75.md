# 222_75 TabPage UI 重构：统一跨平台样式并引入 SVG 图标

## 如何测试

1. 运行：`xmake config --is_community=false`
2. 编译：`xmake b stem`
3. 启动 Mogan，确认标签页栏（TabPage Bar）视觉正常。
4. 确认关闭按钮和添加按钮显示为 SVG 图标（`×` 和 `+`），而非文字或原生按钮。
5. 确认关闭按钮仅在鼠标 Hover 或标签处于选中状态时显示，移出后隐藏。
6. 确认启动页（Startup Tab）标签文字居中对齐。
7. 分别验证亮色/暗色主题下图标颜色正常（亮色主题关闭按钮 hover 变红，暗色主题图标为白色）。
8. 验证标签页拖拽排序功能正常。
9. 分别测试 macOS / Windows / Linux 下的标签页高度和按钮尺寸一致性。
10. 检查VIP按钮样式是否合适

## 2026/04/29 VIP 按钮修改

### What

为商业版标题栏新增的 VIP 按钮（`vip-button`）进行了调整优化

#### 修改文件

**src/Plugins/Qt/QTMTabPage.cpp**
- `arrangeTabPages()` 中 `reservedRight` 计算增加 VIP 按钮预留：`reservedRight += DpiUtils::scaled(80)`。
- 仅商业版生效（`#ifndef IS_COMMUNITY`），社区版不受影响。

**src/Plugins/QWindowKit/windowbar_p.hpp**
- 调整 `WindowBarItem` 枚举顺序：`VipButton` 移至 `LoginButton` 之前，使 VIP 按钮显示在登录按钮左侧。

**src/Plugins/Qt/qt_tm_widget.cpp**
- VIP 按钮样式（背景渐变、颜色、hover/pressed 状态）从代码内联 `setStyleSheet` 提取到 CSS 文件。
- `border-radius`、`font-size`、`margin-right`、`iconSize` 均改为 `DpiUtils::scaled()` 动态计算。
- VIP 按钮颜色从金/橙色渐变改为墨绿色系（主色 `#215a6a`，辅色 `#2791ad`）。

**TeXmacs/misc/themes/liii.css / liii-night.css**
- 新增 `QWK--WindowBar>QPushButton#vip-button` 样式规则，定义 normal / hover / pressed 三态渐变。

### Why

1. **标签页不遮挡**：`arrangeTabPages` 按窗口宽度分配标签页宽度，若右侧未预留 VIP 按钮空间，标签页会延伸覆盖按钮。
2. **样式集中管理**：按钮样式从 C++ 硬编码移至 CSS，便于主题切换和后续调整。
3. **DPI 自适应**：所有尺寸属性通过 `DpiUtils::scaled()` 计算，确保 HiDPI 屏幕下显示一致。

### How

- `reservedRight` 原计算 5 个系统按钮宽度（pin/min/max/close/login）+ 2×padding，现商业版额外加 `DpiUtils::scaled(80)`。
- `windowbar_p.hpp` 通过枚举顺序调整布局位置，无需修改 `setWidgetAt` 调用逻辑。
- CSS 选择器使用 `QWK--WindowBar>QPushButton#vip-button`， specificity 足够高，不会被通用按钮样式覆盖。

## 2026/04/28 针对macos的修复

### What

修复 macOS 系统按钮（traffic lights）在无边框窗口初始化时位置错误的问题，并实现通过 `setSystemButtonArea` 精确控制系统按钮的垂直/水平位置。

#### 修改文件

**src/Plugins/Qt/qt_tm_widget.cpp**
- 在 macOS 分支创建 `sysBtnArea` QWidget，设置尺寸为 `80 * scale` 宽、`nativeTitleBarHeight` 高。
- 通过 `windowAgent->setSystemButtonArea(sysBtnArea)` 将区域注册到 qwindowkitty，Cocoa 层自动将三个系统按钮居中在该区域内。
- `yOffset = (titleBarHeight - nativeTitleBarHeight) / 2` 控制系统按钮在标题栏内的垂直居中。
- 移除原来无效的 `findChildren<QAbstractButton*>` + `setGeometry` 手动定位代码（对原生 NSButton 不生效）。

**3rdparty/qwindowkitty/src/core/contexts/cocoawindowcontext.mm**
- `winIdChanged` 中调整调用顺序：`setSystemTitleBarVisible(false)` → `setSystemButtonVisible(...)` → `setScreenRectCallback`。
- 将 `setScreenRectCallback` 推迟到下一个 event-loop tick（`QTimer::singleShot(0, ...)`），确保 `show()` 和 Qt 布局完成后 `mapTo` 返回正确坐标，再同步给 `NSWindowProxy`。
- KVO `observeValueForKeyPath:"window"` 中的 `updateSystemButtonRect()` 同样推迟到 `dispatch_get_main_queue()`，避免 AppKit 内部 layout pass 覆盖刚计算的 frame。
- 新增 `#include <QtCore/QTimer>`。

**src/Plugins/Qt/QTMTabPage.cpp**
- 移除 `constexpr int TAB_RIGHT_EXTRA_GAP = 66`，因为系统按钮位置已交由 qwindowkitty 管理，不再需要手动预留右侧空白。
- `arrangeTabPages` 中 `reservedRight` 计算移除 `extraGap`。

### Why

1. **初始位置失效**：`winIdChanged` 在 `show()` 过程中同步调用，此时 Qt 布局尚未完成，`mapTo` 返回错误坐标，导致系统按钮永远停留在默认位置（只有全屏切换触发 resize 后才正确）。
2. **AppKit 覆盖 frame**：`setSystemTitleBarVisible(false)` 修改 `styleMask` 和 `titlebarAppearsTransparent` 会触发 AppKit 内部重新 layout，如果在其之前设置按钮 frame，AppKit 会立刻覆盖回默认位置。
3. **手动 setGeometry 无效**：macOS 系统按钮是原生 `NSButton`，不是 Qt `QWidget` 子对象，`btn->setGeometry` 只修改了 Qt 占位控件的几何信息，对实际 Cocoa 按钮毫无作用。
4. **统一布局管理**：移除 `TAB_RIGHT_EXTRA_GAP` 硬编码，让标签页栏的右侧留白由 qwindowkitty 通过 `setSystemButtonArea` 统一管理，避免两套逻辑冲突。

### How

- 使用 `windowAgent->windowAttribute("title-bar-height")` 获取 Cocoa title bar 高度（约 28pt），作为 `sysBtnArea` 的高度，确保坐标转换在系统允许范围内。
- `QTimer::singleShot(0, ...)` 延迟同步 callback，让 Qt `show()` → layout → `mapToGlobal` 完整走完后，qwindowkitty 才把正确的 QRect 传给 `NSWindowProxy::updateSystemButtonRect()`。
- `dispatch_async(dispatch_get_main_queue(), ^{ proxy->updateSystemButtonRect(); })` 在 KVO 路径做同样延迟，防止 AppKit 的 layout pass 覆盖自定义 frame。
- 通过调整 `sysBtnArea->move(x, y)` 和 `sysBtnArea->setFixedSize(w, h)` 即可精确控制三个系统按钮在标题栏中的位置和垂直偏移，无需修改 Cocoa 源码中的坐标计算逻辑。

## 2026/04/28 实现

### What

对 TabPage 组件进行全面 UI 重构，将关闭按钮和添加按钮从 `QToolButton` 迁移到 `QWK::WindowButton`，引入 SVG 图标资源，并彻底移除 macOS / Windows / Linux 三平台硬编码差异，实现跨平台样式统一。

#### 修改文件

**src/Plugins/Qt/QTMTabPage.cpp / hpp**
- 关闭按钮 `m_closeBtn` 和添加按钮 `m_addTabButton` 类型从 `QToolButton*` 改为 `QWK::WindowButton*`。
- 移除全部 `#ifdef Q_OS_MAC` 分支：包括 `TAB_CONTENT_VERTICAL_OFFSET`、`getScaledSystemBarHeight`、`getScaledSystemButtonHeight`、`getScaledTabCloseButtonSize` 等。
- DPI 缩放统一使用 `DpiUtils::scaled()`，新增 `getScaledAddButtonHeight()` / `getScaledCloseButtonHeight()` 替代旧函数。
- 新增 `ADD_BUTTON_SIZE = 20`、`CLOSE_BUTTON_SIZE = 18` 常量。
- `STARTUP_TAB_MAX_WIDTH_BASE` 区分社区版（`IS_COMMUNITY`）和商业版：120 / 90。
- 关闭按钮连接逻辑改为 `QPointer<QAction>` 保护，防止 action 被释放后悬空触发。
- `paintEvent` 中启动页标签文字改为 `Qt::AlignCenter`，普通标签保持 `Qt::AlignLeft`。
- `updateCloseButtonVisibility` 增加条件：仅在 `underMouse() || isChecked()` 时显示关闭按钮。
- 默认构造函数不再调用 `initializeCloseButton()`（`dummyTabPage` 不需要关闭按钮），并在所有访问 `m_closeBtn` 处补充空指针检查。
- `m_closeBtn` 成员添加 `= nullptr` 默认初始化。
- `enterEvent` 补充 `override` 关键字。
- 移除未使用的全局变量 `g_hiddentTabIndex`。
- `QTMTabPageContainer` 析构函数移除冗余的 `delete m_addTabButton`。
- `arrangeTabPages` 中 `addButtonWidth` 语义修正：`height()` → `width()`。

**src/Plugins/Qt/qt_tm_widget.cpp**
- 标题栏高度统一为 `42 * scale`（macOS 原特殊值 20 移除）。
- 系统按钮尺寸统一为 `60 * scale` 宽、`42 * scale` 高（移除 macOS 特殊逻辑）。
- 图标基础尺寸统一为 `16 * scale`（macOS 原特殊值 20 移除）。
- 工具栏高度统一为 `DpiUtils::scaled(32)`，tab row 高度统一为 `DpiUtils::scaled(38)`（移除 macOS 特殊值 30）。
- 登录按钮图标尺寸统一为 `12`（macOS 原特殊值 20 移除）。
- 登出按钮图标尺寸统一为 `20`（macOS 原特殊值 30 移除）。

**TeXmacs/misc/themes/liii.css / liii-night.css**
- `QTMTabPage` 移除硬编码 `padding`、`border-radius` 和 `border-bottom: none`，改为由 C++ 动态计算。
- 关闭按钮和添加按钮选择器从 `QToolButton` 改为 `QPushButton`。
- 移除全部 `[platform="mac"]` 规则。
- 关闭按钮通过 `qproperty-iconNormal` / `qproperty-iconHovered` 设置图标（亮色：`close.svg` / `close-white.svg`；暗色：`close-white.svg`）。
- 添加按钮通过 `qproperty-iconNormal` 设置图标（亮色：`add.svg`；暗色：`add-white.svg`）。
- 移除关闭按钮的 `min-width`、`max-width`、`margin-left` 等硬编码尺寸。
- 移除添加按钮的 `font-weight: bold`、`font-size`、`color` 等文字样式。

**TeXmacs/misc/images/images.qrc**
- 新增 4 个 SVG 资源文件：`tabpage/add.svg`、`tabpage/close.svg`、`tabpage/add-white.svg`、`tabpage/close-white.svg`。

**TeXmacs/misc/images/tabpage/*.svg**
- 新增 4 个 SVG 图标文件，基于 48×48 viewBox，提供亮色/暗色两版。

**TeXmacs/progs/texmacs/menus/tabpage-menu.scm**
- 关闭按钮文字从 `"✕"` 改为 `""`（空字符串），因为图标已由 CSS/C++ 接管。

### Why

1. **跨平台统一**：旧代码中 macOS 拥有大量独立分支（标题栏高度、按钮尺寸、字体大小等），维护成本高且视觉不一致。统一后三平台代码完全一致。
2. **图标化**：关闭按钮和添加按钮从纯文字（`✕`、`+`）升级为 SVG 图标，视觉更现代，且支持亮色/暗色主题自动切换。
3. **样式解耦**：将 `padding`、`border-radius` 等从 CSS 硬编码移至 C++ 动态计算（通过 `DpiUtils::scaled()`），确保高 DPI 下尺寸正确。
4. **交互优化**：关闭按钮仅在需要时显示（hover/选中），减少标签页视觉噪音，更接近现代浏览器标签页体验。
5. **安全性**：`QPointer<QAction>` 防护避免 action 生命周期问题；空指针检查使 `dummyTabPage` 不初始化关闭按钮的逻辑更安全。

### How

- 引入 `QWK::WindowButton`（已存在于项目中，用于系统标题栏按钮）替代 `QToolButton`，复用其 `iconNormal` / `iconHovered` 属性支持。
- 新增 `tabpage/` SVG 资源目录，通过 `.qrc` 编译进二进制。
- CSS 中通过 `qproperty-*` 语法将图标路径与按钮绑定。
- 全面替换 `int (base * getDPIScaleFactor())` 为 `DpiUtils::scaled(base)`，消除浮点截断误差。
- 移除 `dummyTabPage` 的关闭按钮初始化，并在所有 `m_closeBtn` 访问点做空指针守卫。
