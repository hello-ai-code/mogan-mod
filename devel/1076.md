## 5 What
使用双容器索引（`std::map` + `std::set<TimeIndex>`）替代运行时排序，消除 `getAllSessionIds()` 每次调用时的 O(n log² n) 排序开销。

1. 新增 `TimeIndex` 结构和 `timeIndex_` 成员，所有增删操作同步维护双容器
2. 新增 `sessionCount()`、`firstActiveSessionId()`、`findReusableSession()`、`touchSession()` 方法
3. 简化 `qt_chat_controller.cpp` 中 10 处调用点，消除不必要的排序和重复遍历
4. `createdAt` 和 `updateAt` 统一为 `time_t` 类型，排序比较为整数比较
5. `updateAt` 持久化到 manifest JSON，重启后排序顺序保持
6. 删除废弃的 `getDisplayTitles()` 方法
7. 新增 `ChatSidebar::reorderItem()` 侧边栏 UI 重排，`touchSession` 后同步刷新侧边栏
8. 补充 `archiveSession` 不改变 `updateAt` / `timeIndex_` 的测试

## 6 Why
`getAllSessionIds()` 每次调用执行完整排序（O(n log² n)），在 `createView()` 初始化时连续调用 4 次，`buildDisplayInfos()` 内部排 2 遍序。侧边栏 tooltip 判断也触发排序。随着会话数量增长，性能显著下降。

## 7 How

### 7.1 双容器架构

核心思路是空间换时间：主存储 `sessions_`（map，按 sessionId 查找）+ 有序索引 `timeIndex_`（set，按 updateAt 降序排列，始终有序无需重排）。

```
┌─────────────────────────────────────────┐
│           ChatSessionManager            │
├─────────────────────────────────────────┤
│  主存储: std::map<string, ChatSession>   │  ← O(log n) 按 sessionId 查找
│           sessions_                     │     存储完整会话数据
├─────────────────────────────────────────┤
│  索引: std::set<TimeIndex>               │  ← O(log n) 按 updateAt 排序
│         timeIndex_                      │     始终有序，无需重排
│         仅存储 updateAt + sessionId      │
└─────────────────────────────────────────┘
```

双容器同步规则：

| 操作 | `sessions_` (map) | `timeIndex_` (set) |
|------|-------------------|-------------------|
| `createSession` | `insert` | `insert` |
| `removeSession` | `erase` | `erase` |
| `insertSession` | 覆盖写入 | 先删旧再插入 |
| `archiveSession` | `archived = true` | **不操作** |
| `restoreSession` | `archived = false` | **erase + insert**（updateAt=now） |
| `touchSession` | `updateAt = now()` | **erase + insert** |

### 7.2 控制器调用点简化

| 原调用 | 优化后 | 效果 |
|--------|--------|------|
| `getAllSessionIds().size()` | `sessionCount()` | O(1) |
| `getAllSessionIds().empty()` | `sessionCount() > 0` | O(1) |
| 手动遍历找非归档会话 | `firstActiveSessionId()` | O(k)，k=归档数 |
| `determineInitialActiveSession()` 单行包装 | 内联为 `firstActiveSessionId()` | 删除冗余方法 |
| `getAllSessionIds()` + 遍历找空白会话 | `findReusableSession()` | O(n)，无需排序 |
| `getDisplayTitles()` 构建完整 map 只为取单条标题 | 直接 `getSession()` | O(log n) |
| `getDisplayTitles()` + `getAllSessionIds()` 双重遍历 | 单次遍历直接取 title | O(n) |
| `getAllSessionIds().front()` 取排序最前会话恢复全局模型 | 直接用激活的 `initialId` 取模型 | O(log n)，语义正确 |
| `touchSession` + `reorderItem` 在 Idle 时调用 | 移至 `onSendRequested` 发送时调用 | 发送时即更新排序 |

### 7.3 类型统一

- `createdAt` 从 `string` 改为 `time_t`，与 `updateAt` 类型一致
- 排序比较为整数比较，比字符串比较更快且无内存分配
- Scheme ↔ C++ 边界处用 `snprintf`/`atol` 做字符串 ↔ 数值转换

### 7.4 持久化链路

`updateAt` 需要跨重启保持，全链路改动：

```
Scheme manifest JSON ←→ C++ qt_chat_tab_restore_session ←→ ChatSession.updateAt
```

- `chat-persist-make-entry`: 新增 `updateAt` 字段，缺失时回退到 `createdAt`
- `chat-persist-update-manifest`: C++ 传入 `updateAt` 字符串
- `chat-persist-load-all`: 读取 `updateAt`，传给 C++ 回调
- `glue_widget.lua`: 参数从 7 个增加到 8 个
- 旧 manifest 无 `updateAt` 字段时自动回退，向后兼容

### 7.5 触发时机

- `touchSession()` 在 `onSendRequested` 中发送消息成功后调用（发送时即更新排序）
- 调用后立即 `updateManifest()` 持久化到磁盘
- `restoreSession()` 内部也更新 `updateAt = now()`（恢复 = 用户要继续对话，置顶）

### 7.6 侧边栏 UI 同步

`touchSession()` 更新了内存中的 `timeIndex_` 排序，但侧边栏不会自动感知。新增 `ChatSidebar::reorderItem(sessionId)` 方法：

- 将指定会话的 widget 移到 `conversationListLayout_` 第 0 位（`insertWidget(0, ...)`）
- 仅移动单个 widget，O(1) 操作，不需要重新遍历 `timeIndex_`
- 在 `onSendRequested` 中 `touchSession` 之后调用，确保发送消息时当前会话置顶
- `archiveSession` 不改变 `updateAt` 和 `timeIndex_`，无需同步侧边栏
- `moveFromArchive` 已硬编码 `insertWidget(0, ...)` 置顶，与 `restoreSession` 更新 `updateAt=now()` 一致
