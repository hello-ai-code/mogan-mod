## What

修复 Telemetry 事件字段中 `sessionId`、`platform`、`language` 的语义和实现问题。

1. **修复 sessionId 语义**：从"每次调用生成新 uuid"改为"应用启动时生成一次并缓存"，使 `sessionId` 真正标识一次使用会话
2. **改进 platform 识别**：使用 C++ 层 `get_pretty_os_name` 获取详细系统信息，并做字符串归一化（去下划线、去空格、转小写），替代原先简单的三选一判断
3. **改进 language 识别**：使用 C++ 层 `get_locale_language` 结合 `language-to-locale` 转换，替代原先读取 `LANG` 环境变量的方式，跨平台更可靠
4. 统一 `telemetry-make-event` 中 `sessionId` 的获取方式，改为调用函数而非直接引用全局变量
5. 补充三项自动化测试覆盖上述字段语义

## Why

- **sessionId**：原实现 `(define-public (telemetry-session-id) (uuid4))` 每次调用都生成新 uuid，导致同一运行周期内每个事件的 `sessionId` 都不相同，失去了"会话追踪"的意义，无法将一次使用周期内的多个事件关联起来
- **platform**：原实现仅返回 `macos`/`windows`/`linux` 三种笼统值，无法区分具体系统版本（如 macOS 13 vs 14、Windows 10 vs 11、Ubuntu 22.04 vs 24.04），对分析用户分布和兼容性问题帮助有限
- **language**：原实现依赖 `LANG` 环境变量，在 Windows 等平台上该环境变量可能不存在或格式不统一，导致语言识别失败或格式不规范

## How

### 1. sessionId 缓存

在模块加载时生成一次 uuid 并缓存，函数只返回缓存值：

```scheme
(define *telemetry-session-id* (uuid4))

(define-public (telemetry-session-id)
  *telemetry-session-id*)
```

同时删除原先末尾重复的定义 `(define-public *telemetry-session-id* (telemetry-session-id))`，该定义在语义上还会引发递归调用问题（在函数重定义前恰好能工作，但逻辑混乱）。

### 2. platform 归一化

新增 C++ 胶水函数 `get-pretty-os-name`（绑定自系统 API），返回原始系统名称字符串，然后在 Scheme 层做归一化：

```scheme
(let* ((pretty (get-pretty-os-name))
       (no-underscore (string-replace pretty "_" ""))
       (no-spaces (string-replace no-underscore " " ""))
       (normalized (string-downcase no-spaces)))
  normalized)
```

例如 `"macOS 13"` → `"macos13"`，`"Ubuntu 22.04"` → `"ubuntu2204"`。

### 3. language 标准化

新增 C++ 胶水函数 `get-locale-language`（绑定自系统 locale API），返回系统语言标识，再通过已有的 `language-to-locale` 函数转换为标准 locale 格式：

```scheme
(let ((lang (get-locale-language)))
  (language-to-locale lang))
```

确保输出格式统一为 `en_US`、`zh_CN` 等含下划线的标准形式。

### 4. C++ 胶水层

在 `glue_basic.lua` 中注册两个新函数：

| scm_name | cpp_name | ret_type |
|---|---|---|
| `get-pretty-os-name` | `get_pretty_os_name` | `string` |

在 `init_glue_l5.cpp` 中引入 `locale.hpp` 头文件以提供上述 C++ 函数声明。
