# [0215] 修复 AI Chat session 重新加载后顺序倒置

## 任务相关的代码文件
- `TeXmacs/progs/dynamic/chat-session-persist.scm`

## 如何测试

### 非确定性测试（文档验证）
1. 创建多个 AI Chat session（A → B → C）
2. 关闭并重新打开 Mogan
3. 确认侧边栏中 C（最新）显示在最上面

## What

在 `chat-persist-load-all` 中，将 manifest 的 entries 反转后再遍历加载，使最新的 session 显示在侧边栏最上面。

## Why

manifest 中 session 按创建/更新顺序排列（旧的在前，新的追加在末尾）。加载时按 manifest 顺序依次 append 到 C++ 侧的 `conversations_` 列表，侧边栏直接按该列表顺序显示。导致最新的 session 排在最下面，用户需要滚动才能找到最近使用的对话。

## How

修改 `chat-persist-load-all` 中的 `for-each`，将 `entries` 替换为 `(reverse entries)`：

```scheme
(for-each (lambda (entry) ...)
  (reverse entries)   ;; 原为 entries
) ;for-each
```
