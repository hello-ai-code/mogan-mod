## 5 What

1. ChatSession 新增 `bool search` 字段，ChatSessionManager 新增 `setSearch`/`getSearch` 方法
2. ChatConversationPanel 在 thinkingButton_ 左侧新增 Internet Search 切换按钮，样式与 thinking 按钮完全一致
3. 提取 `make_toggle_btn` 公共函数，消除 thinking/search 按钮的重复代码
4. ChatController 连接 `searchToggled` signal，调用 `sessionManager_.setSearch` 并更新 manifest
5. Scheme 持久化层（chat-persist.scm）支持 search 字段的读写，向后兼容（缺失时默认 "disabled"）
6. Scheme 发送链（chat-protocol.scm）将 search 参数一路传递到 JSON params
7. glue_widget.lua 增加第 9 个参数
8. 浅色/深色主题 CSS 各新增搜索按钮样式

## 6 Why

与 thinking（推理模式）对称，用户需要在发送消息时选择是否启用网络搜索功能。search 字段通过 manifest 持久化，确保重启后状态恢复。按钮放置在 thinking 按钮左侧，保持 UI 一致性。

## 7 How

### 按钮创建
提取 `make_toggle_btn` 函数，接受 parent、objectName、text 三个参数，返回配置好的 `QTMStateToolButton`。search 和 thinking 按钮共用此函数，仅 objectName 和文本不同。

### 信号流
`searchButton_ toggled` → `ChatConversationPanel::searchToggled` → `ChatController::onSearchToggled` → `sessionManager_.setSearch` + `updateManifest`。

### 发送流
C++ `onSendRequested` 传递 `session->search` → Scheme `chat-tab-send` → `chat-tab-session-send` → `chat-tab-session-feed` → `chat-tab-build-context-input` → JSON `params.search`。

### 持久化
manifest.json 中新增 `"search": "enabled"/"disabled"` 字段。加载时 `chat-persist-load-all` 提取该字段传给 `qt-chat-tab-restore-session`，C++ 恢复到 ChatSession 和按钮状态。
