# [0206] Chat Tab Sidebar Session 多选与删除功能

## 如何测试

### 编译
```bash
xmake b stem
```

### 功能测试

#### 单个删除
1. 启动 Mogan STEM，打开 Chat Tab
2. 创建 2-3 个会话，发送消息使标题自动生成
3. 右键某个会话 → 菜单应显示：重命名、归档、删除、分隔线、多选
4. 点击「删除」→ 该会话从 sidebar 消失，自动切换到其他会话
5. 再次右键删除剩余唯一会话 → 应自动创建新的空会话
6. 重启 Mogan STEM → 确认已删除的会话不会恢复

#### 多选删除（Active Sessions）
1. 创建 3 个会话
2. 右键任一会话 → 点击「多选」
3. 每个 active session 前出现蓝色复选框，顶部出现操作栏：取消、全选、归档、删除
4. 勾选 2 个会话 → 点击「删除」→ 2 个会话被删除，剩余 1 个
5. 退出多选：点击「取消」→ 复选框消失

#### 多选归档（Active Sessions）
1. 创建 3 个会话
2. 右键 → 多选 → 勾选 2 个 → 点击「归档」→ 2 个会话移入归档区
3. 点击「全选」→ 所有 active session 被勾选

#### 多选删除（Archived Sessions）
1. 先归档若干会话
2. 展开归档区，右键某个归档会话 → 点击「多选」
3. 操作栏显示：取消、全选、删除（无归档按钮）
4. 勾选归档会话 → 点击「删除」

#### 关闭 Tab
1. 删除若干 session 后，点击 Tab 关闭按钮
2. Tab 应立即关闭，不弹出"文档未保存"确认对话框

### 边界测试
- 删除所有会话 → 自动创建新会话
- 多选模式下点击「取消」→ 所有 checkbox 清除，操作栏隐藏
- 在归档区进入多选，操作栏不显示归档按钮
- 删除会话后重启 → 确认已删除会话不会恢复（持久化数据已清理）

## How

### 1. 交互设计

右键会话弹出上下文菜单，包含「多选」选项。点击「多选」后：
- 每个 session 前出现蓝色复选框（默认隐藏，仅在多选模式下显示）
- 顶部出现批量操作栏：取消、全选、归档（仅 active sessions）、删除
- active sessions 多选时显示归档 + 删除按钮
- archived sessions 多选时只显示删除按钮

### 2. 数据流

```
右键会话 → 上下文菜单「多选」
  → enter_multi_select_mode(archived)
    → 显示 checkbox、操作栏
    → 勾选后点击「删除」/「归档」
      → get_checked_panels()
      → delete_sessions() 或逐个 archiveSession()
```

```
delete_sessions(panels)
  → 对每个 panel:
    1. 从 QStackedWidget 移除页面
    2. 从 conversations_ 列表移除
    3. 清理 activeConversation_ 指针
    4. call("chat-persist-delete-one") — 清理磁盘持久化
    5. call("chat-tab-session-destroy") — 清理 Scheme 层状态
    6. sessionManager_.removeSession() — 清理 C++ 会话管理器
    7. 断开信号连接
    8. 隐藏并脱离父控件（zombie panel 模式）
  → 切换到剩余会话或新建空会话
  → buffer-pretend-saved 所有会话 + 主 chat-tab buffer
```

### 3. Zombie Panel 模式

删除会话时不立即释放 widget 内存，而是：
- `setParent(nullptr)` + `hide()` 脱离 Qt 父子关系
- 移入 `zombiePanels_` 列表
- 在 `~QTChatTabWidget()` 析构时统一 `delete`

原因：Qt 父子控件的自动删除机制与 `texmacs_input_widget` 的引用计数智能指针存在冲突，立即 `delete` 会导致 double-free。

### 4. 恢复会话的多选适配

`restore_conversation()` 使用与 `create_conversation()` 相同的 `itemWidget` + `selectCheckBox` 包装模式，确保持久化恢复的会话也支持多选操作。

### 5. 持久化删除

新增 `chat-persist-delete-one`（Scheme）：
- 删除会话目录 `<base-dir>/<sessionId>/message.tmu`
- 从 `manifest.json` 中移除对应条目

在 `delete_sessions` 中于清理内存状态之前调用，确保磁盘数据同步删除。

## 影响范围确认

本次修改涉及：
- Chat Tab 控件 `qt_chat_tab_widget.hpp`/`.cpp` — 多选 UI、删除逻辑
- Scheme 会话状态 `chat-tab-session.scm` — `chat-tab-session-destroy`
- Scheme 持久化 `chat-session-persist.scm` — `chat-persist-delete-one`

不影响：
- 会话发送、取消等消息逻辑
- LLM 插件内部实现
- 普通文档标签页的关闭行为
- 其他 Qt 插件或窗口组件
