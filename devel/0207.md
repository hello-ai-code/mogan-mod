# [0207] Chat Tab 排序与默认新对话

## 如何测试

### 编译
```bash
xmake b stem
```

### 功能测试

#### Active Sessions 由新到旧排序
1. 启动 Mogan STEM，打开 Chat Tab
2. 创建第一个会话 "Chat 1"
3. 点击 "New chat" 创建第二个会话 "Chat 2"
4. 观察侧边栏：从上到下应为 Chat 2、Chat 1（新的在上）
5. 继续创建 Chat 3 → 侧边栏从上到下应为 Chat 3、Chat 2、Chat 1

#### 打开 Chat Tab 默认显示新对话
1. 切换到其他 tab（如文档编辑）
2. 重新打开 Chat Tab
3. 应自动显示一个新的空白对话，而不是切回之前已有会话
4. 如果顶部已有空白对话且 model 一致，则复用该空白对话，不再新建
5. 如果顶部空白对话的 model 与当前 model 不一致，则新建绑定当前 model 的空白对话

#### 关闭 Tab
1. 打开 Chat Tab，创建若干会话
2. 点击 Tab 关闭按钮
3. Tab 应立即关闭，不弹出"文档未保存"确认对话框

## 任务相关的代码文件

- `src/Plugins/Qt/qt_chat_tab_widget.hpp` - `ensure_new_conversation()`、`create_new_conversation_with_model()` 声明
- `src/Plugins/Qt/qt_chat_tab_widget.cpp` - 会话排序、默认新对话、buffer pretend-saved、精简调用链
- `src/Plugins/Qt/qt_tm_widget.cpp` - `sync_chat_tab_mode` 中调用 `ensure_new_conversation`
- `TeXmacs/progs/dynamic/chat-tab-session.scm` - `chat-tab-append-round!` 从 session state 取 model
- `TeXmacs/progs/dynamic/chat-adapter.scm` - `open-llm-chat-tab` 去掉冗余 `qt-chat-tab-new-session` 调用
- `src/Scheme/L5/glue_widget.lua` - 删除已废弃的 `qt-chat-tab-new-session` 绑定

## What

1. **Active Sessions 由新到旧排序**：将 `conversations_.append` 改为 `prepend`，sidebar button 使用 `insertWidget(0, ...)`，使新会话始终显示在最上方。
2. **打开 Chat Tab 默认新对话**：新增 `ensure_new_conversation()`，在 `sync_chat_tab_mode` 复用已有 `QTChatTabWidget` 时调用。只检查**最新的 session**，如果最新 session 是空白的就不新建，否则新建一个。
3. **修复关闭 Tab 时弹出保存对话框**：在 `create_new_conversation_with_model` 和 `delete_sessions` 中对所有 session buffer 和 `tmfs://chat-tab` buffer 调用 `buffer-pretend-saved`。
4. **修复 prompt 标签 model 不一致**：`chat-tab-append-round!` 改为接收 `session-id` 参数，从 session state 取 model 而非全局变量 `chat-tab-current-model`，确保 prompt 标签与 session 绑定 model 一致。
5. **`ensure_new_conversation` 增加 model 一致性检查**：检查顶部空白 session 的 model 是否与当前全局 model 一致，一致则复用，不一致则新建。
6. **精简 new-conversation 调用链**：删除冗余的 `create_new_conversation()`（无参透传）、`newSessionWithModel()`（单调用透传）、`qt_chat_tab_new_session()`（已无 Scheme 调用者），所有调用点改为直接调用 `create_new_conversation_with_model(model)`。
7. **精简 `open-llm-chat-tab`**：去掉 `qt-chat-tab-new-session` 调用，由 `switch-to-buffer` → `ensure_new_conversation` 统一兜底，避免重复新建 session。
