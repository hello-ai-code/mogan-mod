## 5 What
为 Chat Tab 的 C++ 代码添加 Doxygen 注释和单元测试。

1. 为 `qt_chat_session.hpp` 中 `ChatSessionManager` 的 15 个公共方法添加 `@brief`/`@param`/`@return` 注释
2. 为 `qt_chat_controller.hpp` 中 8 个公共槽方法和 9 个私有方法添加 Doxygen 注释
3. 为 `qt_chat_tab_widget.hpp` 中三个类（`ChatConversationPanel`、`ChatSidebar`、`QTChatTabWidget`）的所有方法和成员变量添加 Doxygen 注释
4. 为 `qt_chat_controller.cpp` 中 5 个复杂方法（`onSendRequested`、`onDeleteRequested`、`onArchiveRequested`、`notifyStateChanged`、`ensureNewConversation`）的关键逻辑段添加行内注释
5. 新建 `tests/Plugins/Qt/qt_chat_session_test.cpp`，包含 19 个测试用例，覆盖 `ChatSessionManager` 所有公共方法

## 6 Why
Chat Tab 的 C++ 代码（MVC 架构）缺少完整的 API 文档注释和单元测试。Doxygen 注释有助于代码维护和 AI 辅助开发时的上下文理解，单元测试保障数据层的正确性。

## 7 How

### 注释规范

遵循项目已有的 Doxygen 风格：
- `/** @brief` 用于类和函数文档
- `@param`/`@return` 用于参数和返回值说明
- `///<` 用于成员变量的行尾注释
- 中文描述

### 单元测试设计

`ChatSessionManager` 是纯数据层，不依赖 Scheme 解释器和 Qt widget，可独立测试。测试覆盖：

| 方法 | 测试要点 |
|------|----------|
| `createSession` | UUID 非空、默认值（state=Idle, archived=false, title/model 为空, createdAt 非空）|
| `createSession` | 多次创建返回不同 UUID |
| `removeSession` | 删除后 getSession 返回 nullptr |
| `removeSession` | 删除不存在的 ID 不崩溃 |
| `archiveSession` | 归档后 archived=true |
| `restoreSession` | 恢复后 archived=false |
| `archiveSession` | 归档不存在的 ID 不崩溃 |
| `setTitle` | 设置后读取一致 |
| `getModel` | 默认返回空字符串 |
| `setModel/getModel` | 设置后读取一致 |
| `setState` | 设置为 Generating 后验证 |
| `getAllSessionIds` | 用 insertSession 注入已知时间戳的会话，验证降序排列 |
| `getSession` | 返回正确指针和 sessionId |
| `getSession` | 不存在的 ID 返回 nullptr |
| `findSessionByPanel` | 用假指针设置 panel 后反查成功 |
| `findSessionByPanel` | 未设置的指针返回 nullptr |
| `insertSession` | 手动构造会话插入后可检索，字段一致 |
| `messageBufferUrl` | 验证 `tmfs://chat-message-{id}` 格式 |
| `inputBufferUrl` | 验证 `tmfs://chat-input-{id}` 格式 |

`ChatController` 和 View 层未编写测试：Controller 所有方法依赖 `call()`（Scheme 解释器），View 需要 Qt widget 上下文，无法隔离测试。
