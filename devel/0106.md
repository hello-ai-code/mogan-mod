## What
修复「文档 -> 页面」设置面板中页面种类（Page type）下拉菜单显示为小写的问题。

1. 在 `document-widgets.scm` 和 `format-tools.scm` 中引入 `page-type-pretty` 和 `page-type-raw` 辅助函数，负责标准纸张类型（如 a4, b5 等）的大小写转换。
2. `page-type-pretty` 将内部存储的小写值（如 "a4"）转换为大写显示（如 "A4"）。
3. `page-type-raw` 将用户选择的大写显示（如 "A4"）转换回内部存储的小写值（如 "a4"），确保与 C++ 层 `page_type.cpp` 中的页面尺寸数据库兼容。
4. 更新 `page-size-list` 和 `window-page-size-list` 返回大写的选项列表。
5. 更新 `page-formatter-format` 和 `page-format-tool` 中的 `enum` 控件，在读取和设置 `page-type` 时进行大小写转换。

## Why
标准纸张类型的国际通用写法为首字母大写（如 A4、B5）。原实现直接将内部存储的小写字符串显示在 UI 上，不符合用户习惯和视觉规范。

## How
在 UI 层做显示值与实际存储值的映射，保持底层数据格式不变，避免影响 C++ 页面尺寸解析逻辑。类似的做法已在 `document-menu.scm` 的菜单项中采用（如 `("A4" (init-page-type "a4"))`）。
