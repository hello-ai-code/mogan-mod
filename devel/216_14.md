# 216_14 启动页通用键盘桥接

## 如何测试
1. 启动 Mogan，进入 `tmfs://startup-tab` 启动页。
2. 在启动页按 `Ctrl+T`，应新建标签页/文档（行为与编辑区一致）。
3. 在启动页按 `Ctrl+1` ~ `Ctrl+6`，应切换到对应标签页（与既有键位映射一致）。
4. 在启动页按其它已定义快捷键（如平台上可用的标准快捷键），应按 Scheme 键盘映射执行，不需要 Qt 层单独适配。
5. 切回普通编辑页，确认原有快捷键行为无回归。

## 2026/04/16 启动页通用键盘桥接实现

### What
将启动页从“单独绑定个别快捷键”改为“通用键盘事件桥接”：
- 启动页直接把 Qt 键盘事件转换为 TeXmacs 键串，并转发给 Scheme `key-press`。
- 覆盖按键按下/释放两条路径，保持与编辑区事件链一致。
- 进入启动页时主动设置焦点，确保桥接稳定生效。

#### 新增文件：

无新增文件

#### 修改文件：

**src/Plugins/Qt/qt_startup_tab_widget.hpp** (修改)
- 增加 `keyPressEvent` / `keyReleaseEvent` 覆盖声明。

**src/Plugins/Qt/qt_startup_tab_widget.cpp** (修改)
- 删除 `QShortcut + Ctrl+T/Ctrl+1..6` 的硬编码绑定逻辑。
- 新增通用键盘桥接实现：
  - 使用 `from_key_press_event` / `from_key_release_event` 生成 TeXmacs 键串；
  - 通过 `eval_scheme("(key-press \"...\")")` 转发到 Scheme 键盘映射层；
  - 增加 Scheme 字符串转义，避免反斜杠与引号破坏命令。
- 启动页自身焦点策略改为 `Qt::StrongFocus`，可直接接收键盘事件。

**src/Plugins/Qt/qt_tm_widget.cpp** (修改)
- 在 `startupTabMode` 下展示启动页后，调用 `startupContentWidget->setFocus(...)`，保证启动页键盘输入路径可用。

### Why
1. **避免重复维护**：不再为每个新快捷键在启动页 Qt 代码里补一条 `QShortcut`。
2. **行为一致**：启动页与编辑区共用同一套 Scheme 键盘映射规则，减少“某页可用/某页失效”的分叉行为。
3. **可扩展性更好**：后续键位调整主要在键盘映射层完成，不需要改动启动页 C++ 绑定代码。

### How
1. 在启动页控件中拦截 `keyPressEvent` / `keyReleaseEvent`。
2. 复用现有 Qt->TeXmacs 键串转换函数（与编辑区一致）。
3. 将转换结果桥接到 Scheme `key-press`，让既有键盘映射机制完成命令分发。
4. 在切入启动页模式时确保焦点落在启动页控件上，形成稳定输入链路。
