## 5 What
针对 Chat Tab 中的 session 会话，动态加载样式宏包并添加 `<session>` 环境包裹。

### 环境包裹分析
通过对比 session-edit 和 chat-tab-session 的分工：
- **session-edit 包裹**：`input-text` → `unfolded-io-text`（`field-insert-output`）
- **chat-tab-session 已做**：直接创建 `unfolded-io-text`（等价）
- **插件输出**：`folded-explain`、`strong` 等自然流入 output 区域
- **结论：无需额外环境包裹，只需加载样式包和添加 `<session>` 包裹**

### 具体工作
1. C++ `make_chat_embedded_style()` 只保留 `"generic"`，业务样式包全部由 Scheme 动态加载
2. 新增 `chat-tab-add-default-style-packages!`：偏好驱动加载样式包
3. 修改 `chat-tab-ensure-session!`：初始化 message buffer 为 `<session>` 结构并加载样式包

## 6 Why
当前 Chat Tab 的样式包在 C++ 中硬编码为 `("generic", "llm")`，存在两个问题：
1. **样式不足**：缺少 `chinese`、`number-europe` 等文档级样式包，导致 CJK 文字、数字格式等渲染不正确
2. **不可扩展**：每次添加样式包都需改 C++ 并重新编译

参考 session-edit 的 `make-session` 和 `buffer-set-default-style` 的动态加载模式，将样式包加载逻辑移到 Scheme 侧。

## 7 How

### 1. C++ 侧（`qt_chat_tab_widget.cpp:80-83`）
```cpp
tree make_chat_embedded_style () {
  return compound ("style", tuple ("generic"));
}
```
只保留最基础的 `"generic"` 样式。

### 2. Scheme 侧（`chat-tab-session.scm`）

新增 `chat-tab-add-default-style-packages!`，参考 `buffer-set-default-style`（tm-files.scm:130-146）和 `make-session`（session-edit.scm:547-548）：

- **文档级样式包**（偏好驱动）：`number-europe`、`preview-ref`
- **语言相关**（`get-preference "language"`）：`chinese`、`table-captions-above`
- **插件样式包**（动态检测）：通过 `url-exists?` 检测 `$TEXMACS_STYLE_PATH/llm.ts` 是否存在

修改 `chat-tab-ensure-session!`：
- 初始化 message buffer body 为 `(session "llm" ,plugin-ses (document))` 结构
- `chat-tab-message-document` 已有 `(tree-is? doc 'session) → tree-ref doc 2` 的处理（第111-112行），因此 `chat-tab-append-round!`、`chat-tab-output` 等函数无需修改
- 为 message buffer 和 input buffer 调用 `chat-tab-add-default-style-packages!`
