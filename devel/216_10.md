# 216_10 启动页模板页与预览组件整体改造（相对 main）

## 如何测试
1. **启动 Mogan STEM，进入启动页模板区域，确认卡片网格会随窗口宽度变化自动调整列数（1~6列）**
2. 切换分类，确认分类名称可翻译显示（如 “All” -> “全部”）
3. 在高 DPI（如 150%/200%）下查看模板缩略图，确认图片清晰、边框样式正常
4. 打开模板预览弹窗，确认对话框尺寸、标题/描述/按钮样式符合新主题
5. 预览 PDF 模板，确认悬停时显示左右翻页按钮与底部页码，多页可翻页，单页不显示控件
6. 预览图片模板，确认图片按预览区域自适应显示
7. 在浅色/深色主题下分别检查模板按钮与 PDF 预览控件样式

## 2026/04/15 分支相对 main 差异

### What
本分支相对 `main` 主要包含三块改动：
- 启动页模板网格改为响应式布局，并优化卡片/预览区域尺寸与高 DPI 显示效果。
- PDF 预览能力从“静态显示”升级为“可翻页、悬停控件、DPI 友好”的可复用组件，并统一图片预览流程。
- 补齐模板页与 PDF 预览相关主题样式及中英文词条翻译。

### Why
1. **响应式体验**：固定列数在不同窗口尺寸下利用率低，模板页需要自适应布局提升可用性。
2. **预览可读性**：模板预览是选模板的关键入口，需要更大的显示区域和更清晰的高 DPI 呈现。
3. **交互完整性**：PDF 模板常为多页，需提供一致的翻页与页码反馈，避免“只能看第一页”。
4. **主题与文案一致性**：新控件引入后需同步浅/深色样式与翻译词条，避免 UI 与语言不完整。

### How
1. 在模板页引入“按 viewport 宽度计算列数”的网格布局策略，并在 `resizeEvent` 中按需重排。
2. 重构 PDF 预览组件为复合 `QWidget`，封装 PDF/图片加载、渲染、翻页、悬停控件与状态管理。
3. 在主题文件中新增对应对象名样式，在翻译词典补齐新增文案，实现功能、样式、文案三者同步落地。
