## 3 如何测试

### 3.1 非确定性测试（文档验证）
1. 启动 Mogan STEM，打开 Chat Tab
2. 发送一条消息
3. 观察 message buffer 中每轮对话的 prompt 前缀，应显示为 `kimi-k2.6> ` 而非 `VLM> `
4. 切换模型后，prompt 应显示对应配置中的 `"model"` 字段值

## 4 如何提交

```bash
xmake b stem
```

## 5 What

将 chat tab 的 prompt 前缀生成逻辑从 mogan 侧硬编码解析改为插件主导：

1. **mogan 侧适配**：让 chat tab 支持接收并应用插件通过 `flush-prompt` 发送的 prompt。
2. **插件侧主导**：LLM 插件在 welcome 阶段从 `liii_llm_menu.json` 配置中读取 `"model"` 字段，通过 `flush-prompt` 发送给 mogan。

## 6 Why

当前 chat tab 的 prompt 由 `chat-tab-model-prompt` 函数根据模型 key 字符串自行解析。对于 `"Kimi-VLM"`，生成的是 `"VLM> "`，无法反映真实模型名称 `kimi-k2.6`。改为插件主导后，prompt 内容可由插件根据配置动态决定，mogan 只做展示适配。

## 7 How

### mogan 侧适配（`chat-tab-session.scm`）

- **`chat-tab-session-do`**：在每一轮任务启动写入插件输入后，从全局 `plugin-prompts` 表中读取该 session 的 prompt，设置到当前 `out` 节点所在 `unfolded-io-text` 的 prompt 子节点（索引 0），覆盖 `chat-tab-model-prompt` 生成的默认值。
- **`chat-tab-session-notify`**：将 `"prompt"` 通道的处理从 `(noop)` 改为实际更新逻辑——用插件发送的 prompt tree 更新当前 `out` 的父节点的 prompt 子节点。

### 插件侧主导（`tm-llm.scm`、`liii/llm.scm`）

- `tm-llm.scm` 重写为真正的 LLM 入口，加载配置并构造 `model-profile`。
- `liii/llm.scm` 中的 `llm-welcome` 使用 `(mp 'model)`（即从 `liii_llm_menu.json` 读取的 `"model"` 值）作为 `flush-prompt` 的参数，取代之前的 `(mp 'name)`。
