## 5 What

修复 Chat Tab 发送含非 ASCII 字符（拉丁字符、CJK 字符）的消息时程序 crash 的问题。

1. 将 `session->title` 存储编码从 Cork 改为 UTF-8
2. `chat-persist-extract-title` 返回 verbatim 原文（不做 cork->utf8 转换）
3. `onSendRequested` 和 `endEditTitle` 使用 `from_qstring_utf8` 存储 UTF-8
4. 清理调试日志

## 6 Why

C++ `updateManifest` 将 Cork 编码的 title（ö=0xF6）传给 Scheme `chat-persist-make-entry`，其中 `njson-set!` 将 Cork 字节写入 nlohmann::json 对象。后续 `njson->file` 调用 `json.dump()` 时遇到无效 UTF-8 字节 0xF6，抛出 `type_error.316` 导致 crash。

## 7 How

### 核心发现：`to_qstring` 自动检测编码

`qt_utilities.cpp` 中的 `to_qstring` 使用 `looks_utf8()` 检测：
- 看起来像 UTF-8 → 直接用 `utf8_to_qstring`
- 否则 → 先 `cork_to_utf8` 再转 QString

这意味着 `session->title` 无论存 Cork 还是 UTF-8，`to_qstring` 都能正确显示。

### 不能在 Scheme 做 `cork->utf8` 的原因

`texmacs->verbatim` 返回**混合编码**：
- 拉丁字符（如 ö）→ Cork (0xF6)
- CJK 字符（如 你好）→ UTF-8 (E4 BD A0 E5 A5 BD)

`cork->utf8` 会把已经是 UTF-8 的 CJK 字节当作 Cork 再次转换，产生双重编码乱码。

### 最终方案

**不在 Scheme 侧做编码转换，交给 C++ `to_qstring` 自动检测。**

| 改动 | 文件 | 说明 |
|------|------|------|
| 返回 verbatim 原文 | chat-session-persist.scm | 去掉 `cork->utf8` |
| `from_qstring` → `from_qstring_utf8` | qt_chat_controller.cpp | 存储 UTF-8 |
| `from_qstring` → `from_qstring_utf8` | qt_chat_tab_widget.cpp | 重命名存 UTF-8 |

**数据流**：
```
Scheme extract-title → verbatim 原文（Cork/UTF-8 混合）
  → C++ to_qstring(extracted) → 自动检测，正确显示
  → QString 截断
  → from_qstring_utf8(qTitle) → 存储 UTF-8 到 session->title
  → updateManifest → UTF-8 传给 Scheme njson → 不 crash
```

## 2026/06/05 修复非 ASCII 标题 Crash

### What
- `chat-persist-extract-title` 返回 verbatim 原文（不做 cork->utf8）
- `onSendRequested` 使用 `from_qstring_utf8` 存储 UTF-8 title
- `endEditTitle` 使用 `from_qstring_utf8` 发送 UTF-8 title
- 清理调试日志

### Why
Cork 编码的 title（如 ö=0xF6）传入 nlohmann::json 导致 `type_error.316` crash。
`texmacs->verbatim` 返回混合编码（拉丁 Cork + CJK UTF-8），不能简单用 `cork->utf8` 转换。

### How
将 `session->title` 存储编码改为 UTF-8，利用 `to_qstring` 的 `looks_utf8` 自动检测能力正确显示。
不在 Scheme 侧做编码转换，避免双重编码问题。
