## 5 What
在 AI Chat 发送消息给插件进程时，将 sessionId 写入 `%chat` JSON，使插件侧可以感知当前会话标识。

1. `chat-tab-build-context-input` 新增 `session-id` 参数，在构建的 JSON 对象中加入 `sessionId` 字段
2. `chat-tab-session-feed` 调用 `chat-tab-build-context-input` 时传入 `session-id`

## 6 Why
为后续插件侧实现 history JSON 持久化提供必要的会话标识。插件进程（如 goldfish）收到数据后需要知道当前会话的 sessionId，才能将聊天记录写入对应会话目录的 `message.json` 文件。

## 7 How
在 `chat-tab-build-context-input` 中，通过 `njson-set!` 在 JSON 对象中添加 `sessionId` 字段。该函数原本只接收 `input`、`model`、`thinking` 三个参数，新增 `session-id` 参数后，JSON 结构变为：

```json
{
  "sessionId": "3B8F0056-9F6C-4760-8E11-7BE80E...",
  "params": {
    "model": "Kimi-VLM",
    "thinking": "disabled"
  },
  "content": "用户消息纯文本",
  "images": [...]
}
```

`session-id` 在整个调用链中已经存在（从 C++ ChatController → Scheme chat-tab-send → chat-tab-session-send → chat-tab-session-feed），无需额外传递，只需在构建 JSON 时写入即可。
