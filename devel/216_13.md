# 216_13 模板分类 JSON 缓存 (categories.json) 和 缩略图显示

## 如何测试

### 分类缓存测试
1. 启动 Mogan，进入 `tmfs://startup-tab` 启动页的"模板中心"。
2. 首次加载时，分类列表应从 Scheme 配置文件读取。
3. 联网状态下刷新模板列表，分类数据应被缓存到 `categories.json`。
4. 关闭应用，断开网络，再次启动 Mogan。
5. 进入模板中心，分类列表应能正常显示（从本地缓存加载）。
6. 检查缓存目录：`$TEXMACS_HOME_PATH/system/template_cache/categories.json` 应存在且格式正确。
7. 删除缓存文件后重启，应能自动从 Scheme 配置文件恢复。

### 缩略图缓存显示测试
1. 清空缩略图缓存目录：`$TEXMACS_HOME_PATH/system/cache/thumbnails/`。
2. 启动 Mogan，进入模板中心，观察缩略图首次加载（从网络获取）。
3. 关闭应用，重新启动 Mogan，再次进入模板中心。
4. 观察缩略图从缓存加载，确认：
   - 缩略图显示清晰，无模糊或失真
   - 缩略图显示完整，无截断或错位
   - 在高分屏（Retina/HiDPI）上显示比例正确，无异常放大
5. 对比首次加载和缓存加载的显示效果，应完全一致。

### PDF 预览缓存显示测试
1. 清空 PDF 预览缓存：`$TEXMACS_HOME_PATH/system/cache/pdf_previews/`（如存在）。
2. 启动 Mogan，进入模板中心，点击带有 PDF 预览的模板卡片。
3. 观察 PDF 预览首次加载（从网络获取或实时渲染）。
4. 关闭应用，重新启动 Mogan，再次进入同一模板详情页。
5. 观察 PDF 预览从缓存加载，确认：
   - 预览显示清晰，无模糊或失真
   - 页面比例正确，无异常放大或缩小
   - 在高分屏（Retina/HiDPI）上显示效果与首次加载一致
6. 对比首次加载和缓存加载的显示效果，应完全一致。

## 2026/04/16 模板分类 JSON 缓存实现

### What
为模板中心添加分类数据的 JSON 文件缓存机制：
- 新增 `categories.json` 作为分类数据的本地持久化存储。
- 优先从缓存加载分类，无缓存时回退到 Scheme 配置文件。
- 从远程 API 获取分类后自动更新本地缓存。

#### 新增文件：

无新增文件

#### 修改文件：

**src/Mogan/TemplateCenter/template_cache.hpp** (修改)
- 新增 `loadCategoriesCache()` 方法声明，用于从 JSON 文件加载分类列表。
- 新增 `saveCategoriesCache()` 方法声明，用于将分类列表保存到 JSON 文件。
- 新增 `categoriesCachePath()` 私有方法声明，返回缓存文件路径。

**src/Mogan/TemplateCenter/template_cache.cpp** (修改)
- 实现 `loadCategoriesCache()`：
  - 读取 `categories.json` 并解析为 `QList<TemplateCategory>`。
  - **检查缓存过期**：如果缓存超过 24 小时，自动返回空列表触发刷新。
  - 按 `order` 字段排序分类列表。
  - 验证必要字段（id、name）确保数据完整性。
- 实现 `saveCategoriesCache()`：
  - 将分类列表序列化为 JSON 格式。
  - 包含版本号和时间戳便于调试。
- 实现 `categoriesCachePath()`：返回 `$TEXMACS_HOME_PATH/system/template_cache/categories.json`。
- 添加**并发访问保护**：
  - 使用 `QLockFile` 文件锁机制，防止多线程同时读写 `categories.json` 导致文件损坏。
  - 读写操作都通过 `.lock` 文件进行同步，超时时间为 5 秒。
- 增强**错误处理与日志**：
  - 使用 `QJsonParseError` 获取详细的 JSON 解析错误信息（错误描述和偏移位置）。
  - 缓存文件损坏时自动删除，触发重新生成。
  - 添加调试日志记录加载/保存的分类数量和缓存过期事件。
  - 无效分类记录时输出警告日志，显示具体 ID 和 Name。
- 修复**时区问题**：
  - 时间戳统一使用 UTC 格式存储（`currentDateTimeUtc()`）。
  - 缓存过期比较时使用 UTC 时间，确保跨时区场景下判断正确。
  - 使用 `setTimeZone(QTimeZone::UTC)` 替代已弃用的 `setTimeSpec`。
- 增强**写入可靠性**：
  - 使用 `QIODevice::Truncate` 标志确保文件正确截断。
  - 检查写入字节数，写入失败时删除损坏的缓存文件。

**src/Mogan/TemplateCenter/template_manager.hpp** (修改)
- 新增 `loadCachedCategories()` 方法声明，封装缓存优先的加载逻辑。
- 新增 `loadLocalCategoriesFromScheme()` 方法声明，提取 Scheme 加载逻辑以便复用。

**src/Mogan/TemplateCenter/template_manager.cpp** (修改)
- 修改初始化流程：
  - 将 `loadLocalCategories()` 替换为 `loadCachedCategories()`。
- 实现 `loadCachedCategories()`：
  - 先尝试从 `TemplateCache::loadCategoriesCache()` 加载。
  - 缓存为空时回退到 `loadLocalCategoriesFromScheme()`。
- 实现 `loadLocalCategoriesFromScheme()`：
  - 从原有 `loadLocalCategories()` 提取的逻辑。
  - 保持与 Scheme 配置文件的兼容性。
- 修改远程数据加载回调 `onRemoteMetadataLoaded()`：
  - 获取分类列表后调用 `cache_->saveCategoriesCache()` 更新缓存。

**src/Plugins/Qt/qt_template_page.cpp** (修改)
- 修复缩略图从缓存加载时的 DPR（设备像素比）问题：
  - 从缓存获取缩略图后，设置正确的 `devicePixelRatio` 以匹配当前显示设备。
  - 确保高分屏上缓存缩略图与网络加载的显示效果一致。
- 问题原因：网络加载时设置了 `pixmap.setDevicePixelRatio(dpr)`，但缓存读取时遗漏此步骤。

**src/Plugins/Qt/qt_pdf_preview_widget.cpp** (修改)
- 修复 PDF 预览从缓存加载时的 DPR（设备像素比）问题：
  - 在 `loadFromUrl()` 和 `loadFromFile()` 中从缓存获取 PDF 预览后，设置正确的 `devicePixelRatio`。
  - 确保高分屏上缓存的 PDF 预览与实时渲染的显示效果一致。
- 与缩略图修复类似，遗漏了缓存读取时的 DPR 设置。

### Why
1. **离线可用**：用户首次联网加载后，分类数据被缓存，离线状态下仍可浏览模板分类。
2. **加载性能**：从 JSON 文件解析比每次解析 Scheme 表达式更高效。
3. **数据一致性**：远程 API 更新分类后，本地缓存与远程保持同步（24小时自动刷新）。
4. **向后兼容**：无缓存或缓存损坏时自动回退到 Scheme 配置，保证功能可用。

### How
1. 应用启动时，`TemplateManager::initialize()` 调用 `loadCachedCategories()`。
2. `loadCachedCategories()` 尝试读取 `categories.json`：
   - 成功则解析并加载分类列表。
   - 失败则调用 `loadLocalCategoriesFromScheme()` 从 Scheme 文件加载。
3. 当远程元数据加载完成时，`onRemoteMetadataLoaded()` 调用 `saveCategoriesCache()` 更新缓存。
4. 缓存文件格式为 JSON，包含版本号便于后续格式升级。
