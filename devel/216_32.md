# [216_32] Recent 列表优化：横向布局、时间显示与清空功能

## 如何测试

1. 编译：`xmake b stem`
2. 启动 Mogan，打开启动页 **File** 页面。
3. **布局验证**：确认 Recent 列表每项为**一行**，左侧是文件名，右侧是 `Last opened: yyyy-MM-dd hh:mm`，**不再显示完整路径**。
4. **时间验证**：打开任意文档后返回启动页，确认该项时间更新为当前时间。
5. **数量验证**：连续打开超过 10 个不同文档，确认 Recent 列表可显示**最多 50 条**。
6. **右键菜单**：在任意 Recent 条目上右键，确认菜单有 **"Remove from list"** 和 **"Clear list"** 两项。
7. **清空功能**：点击 **"Clear list"**，确认整个 Recent 列表被清空，显示 "No recent documents"。
8. **macOS tooltip**：在 macOS 下将鼠标悬停在任意条目上，确认 tooltip 正常显示完整文件路径。
9. **高度验证**：确认 Recent 列表项高度明显比 216_31 更紧凑（64px → 40px）。
10. **主题验证**：分别切换亮色/暗色主题，确认文件名和时间文字颜色正常。
11. **中文验证**：切换中文界面，确认 "Last opened" 显示为 "最后打开时间"，"Clear list" 显示为 "清空列表"。

## 2026/04/24 实现说明

### What

对启动页 File 页面的 Recent 板块进行全面优化：重构列表项布局、增加最后打开时间、提升文档数量上限、增加一键清空功能，并修复 macOS 下 tooltip 显示问题。

#### 修改文件

**src/Plugins/Qt/qt_file_page.cpp**
- `renderRecentDocs()`：列表项布局从 `QVBoxLayout` 改为 `QHBoxLayout`。
  - 左侧保留文件名（加粗）。
  - 右侧新增 `Last opened: yyyy-MM-dd hh:mm` 时间标签，11px 字号，右对齐。
  - **删除**原路径标签 `QLabel#startup-tab-recent-path`，不再显示完整文件路径。
- `setupRecentDocs()`：减小列表项高度 `kRecentItemHeight`：64 → **40**。
- `MAX_RECENT_DOCS`：10 → **50**。
- `MAX_GLOBAL_RECENT_DOCS`：25 → **100**。
- 新增 `clearAllRecentDocs()`：调用 Scheme `(startup-tab-clear-all-recent)` 清空全局状态，清空内存列表，保存并刷新。
- `onRecentDocContextMenu()`：右键菜单新增 **"Clear list"** 选项，与 **"Remove from list"** 共存。
- `rowWidget` 添加 `Qt::WA_TransparentForMouseEvents`，使鼠标事件穿透自定义 widget，确保 macOS 下 `item->setToolTip()` 正常显示完整路径。

**src/Plugins/Qt/qt_file_page.hpp**
- 新增 `clearAllRecentDocs()` 私有方法声明。

**TeXmacs/misc/themes/liii.css / liii-night.css**
- 删除已废弃的 `QLabel#startup-tab-recent-path` 选择器（路径显示已移除）。

**TeXmacs/plugins/lang/dic/en_US/zh_CN.scm**
- 新增 `"last opened"` → `"最后打开时间"` 翻译。
- 新增 `"Clear list"` → `"清空列表"` 翻译。

**TeXmacs/progs/startup-tab/startup-tab-file.scm**
- `startup-tab-clear-all-recent` 从 `(noop)` 改为 `(forget-interactive "recent-buffer")`，与 File → Open recent → Clear menu 行为一致。

### Why

1. **信息密度优化**：原设计垂直堆叠文件名和完整路径，路径过长时视觉嘈杂且占用空间大。改为单行横向布局后更紧凑，关键信息（文件名+时间）一目了然。
2. **时间更有价值**：完整路径可通过 tooltip 查看，而最后打开时间是区分同类文件（如 `draft.tm`、`draft (1).tm`）的关键元数据。
3. **上限提升**：10 条上限对于高频用户很快会被占满，50/100 更符合实际使用需求。
4. **一键清空**：逐个删除 50 条记录不现实，"Clear list" 是高频刚需。
5. **macOS tooltip 修复**：`QListWidget` + `setItemWidget()` 在 macOS 上存在事件穿透问题，自定义 widget 会拦截 tooltip 和 hover 事件。`WA_TransparentForMouseEvents` 是最小侵入的修复方案。

### How

- 布局重构：`QVBoxLayout` → `QHBoxLayout`，文件名左侧 stretch，时间右侧固定。
- 时间格式：使用 `doc.openedAt.toString("yyyy-MM-dd hh:mm")`，前缀通过 `qt_translate("Last opened")` 支持多语言。
- 清空逻辑：C++ 侧清空内存列表并保存 JSON；Scheme 侧通过 `forget-interactive` 同步清理全局 `recent-buffer` 状态。
- 高度精简：从 64px 压缩至 40px，配合 50 条上限使单屏可展示更多记录。
