### 3.2 非确定性测试（文档验证）
1. 编译并运行 Mogan STEM：
   ```bash
   xmake b stem && xmake r stem
   ```
2. 打开「用户首选项」（User Preferences）->「键盘」（Keyboard）标签页。
3. 验证是否有一个名为「快捷键提示类型」（Keyboard shortcut style）的下拉选项：
   - **macOS**：可以正常在键盘标签页下看见此下拉选项，并且其选项为「文本」（Text）与「符号」（Symbol）。
   - **Windows/Linux**：由于系统不支持快捷键符号显示，为了界面整洁，**此下拉配置项在 Windows/Linux 系统下会完全隐藏，不予显示**。
4. 验证中英文显示。
5. 在 macOS 下选择不同的选项并点击，应该会提示重新启动应用以使更改生效：
   - **文本 (Text)**（也是默认选项）：显示完全一致的文本格式快捷键。包括 `Cmd+B`、`Ctrl+$`，并且像方向键等正确显示为 `Option+Shift+↑` / `Option+Shift+↓` 等符号，也绝无“命令+B”、“制表符”等任何中文翻译。
   - **符号 (Symbol)**：显示为纯符号（`⌘B`、`⌥⇧↑/⌥⇧↓` 等）。
6. 在 Windows/Linux 下，快捷键会默认显示正常的纯英文文本，没有任何设置项。

## 5 What
1. 在用户首选项的「键盘」选项卡中，增加了一个「快捷键提示类型」（Keyboard shortcut style）选项，使用户可以在「文本」和「符号」两种快捷键提示形式中自由选择，并且提供了中文翻译。
2. 默认快捷键提示类型设为文本样式（以保证开箱即用最清爽，杜绝任何混杂），物理 Command 键与 Control 键符号/文字精确分离，并阻止了方向键被过度本地化为中文。

## 6 Why
1. 目前 Mogan 快捷键的显示是根据系统强制绑定的，在 macOS 强制使用符号显示，而在 Windows/Linux 强制使用文字显示。提供该配置可使用户根据个人偏好自主定制显示样式。
2. 在升级到 Qt 6 后，Qt 在 macOS 上将 `Qt::ControlModifier` 映射到了物理 Control 键，`Qt::MetaModifier` 映射到了物理 Command 键。而 Mogan 原来的 C++ 键盘映射层仍保留着 Qt4/5 时代对两个修饰键进行对冲交换的逻辑。若不将文本渲染层的映射重新对冲，将导致文字样式在 macOS 平台下的 Ctrl 和 Cmd 完全显示反。

## 7 How
1. **Scheme 偏好层**：
   - 在 `auto-close.scm` 中声明 `"keyboard shortcut style"` 偏好。
   - 在 `preferences-widgets.scm` 中注册 pretty-names 并添加到「键盘首选项」部件，使用 `set-pretty-preference*` 函数以支持重启提示。
   - 在 `zh_CN.scm` 中添加中文翻译 `"快捷键提示类型"`。
2. **C++ 文本渲染兼容层 (`qt_ui_element.cpp`)**：
   - 保持 macOS 下 C++ 原生对冲键位交换以兼容 Qt 内置解析（确保默认 Symbol 模式下 NativeText 的 `⌘` 与 `⌃` 精确显示）。
   - 针对 macOS 纯文本选项，将 `QKeySequence::PortableText` 输出的修饰键名利用中间占位变量，进行双重反转：
     - `"Meta+"` $\rightarrow$ `"Ctrl_TEMP+"`
     - `"Ctrl+"` $\rightarrow$ `"Cmd+"`
     - `"Ctrl_TEMP+"` $\rightarrow$ `"Ctrl+"`
     - 确保了 Control 键物理上被识别为 `"C-"` 时显示为 `"Ctrl+"`，而 Command 键物理上被识别为 `"M-"` 时显示为 `"Cmd+"`。
