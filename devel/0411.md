## 5 What
修复 `0612.tmu` 导出 PDF 时因无限递归导致崩溃的问题。

1. 限制了极小尺寸图像栅格化回退机制的触发条件，仅对 `pdf` 和 `svg` 后缀的文件生效。
2. 避免了临时生成的 PNG 图片由于极其微小而再次触发回退判定，从而阻断了无限循环递归调用。

## 6 Why
在之前的修复（见 `devel/0160.md`）中，为了避免极小 PDF 在嵌入 Form XObject 时产生错误的变换矩阵（由于 Qt QImage loader 对极小 PDF 返回 1x1 尺寸导致），加入了 `im->w <= 1 || im->h <= 1` 时回退到栅格化（将 PDF 转换为临时 PNG）的逻辑。然而，对于本身就是 1x1 或因分辨率低被转换为小于等于 1 像素的临时 PNG 文件，在被递归加载时会再次触发 `im->w <= 1 || im->h <= 1` 的逻辑并生成新的临时 PNG，造成无限递归导致栈溢出（Access Violation 0xC0000005）崩溃。

## 7 How
仅对由于 `image_size` 返回不可靠的矢量/可缩放格式文件（如 `pdf` 或 `svg`）进行栅格化回退判定。通过获取图片 URL 的后缀，检查是否属于矢量文件：
```cpp
  string ext= suffix (u);
  if ((ext == "pdf" || ext == "svg") && (im->w <= 1 || im->h <= 1)) {
```
如果是普通的 PNG 栅格文件（包括之前步骤栅格化出的临时 PNG 文件），将直接进行正常的图片渲染而不会再次触发回退，完美避免了无限递归崩溃。
