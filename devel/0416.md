# [0416] HTML导出时嵌入式图片直接编码为Base64内联

## 1 相关文档
- [0623.md](0623.md) - 支持在 HTML 导出时将所有生成的图片均以 Base64 形式内联嵌入，实现单网页自包含导出

## 2 任务相关的代码文件

## 3 如何测试

### 3.1 确定性测试（单元测试）
运行单元测试验证嵌入式图片 Base64 编码路径的正确性：
```bash
xmake r 0416
```

### 3.2 非确定性测试（文档验证）
1. 在 Mogan 中打开或创建一个包含嵌入式图片的文档。
2. 选择 **文件 -> 导出 -> HTML**。
3. 确保导出时启用了 Base64 选项（`tmhtml-base64?` 为 true）。
4. **检查 HTML 输出**：在生成的 HTML 文件中搜索 `<img>` 标签，确认 `src` 属性为 `data:image/...;base64,...` 格式，而非外部文件路径。

## 5 What
优化了 HTML 导出时嵌入式图片的处理逻辑：当 `tmhtml-base64?` 为 true 时，将嵌入式图片直接编码为 Base64 data URI 内联到 HTML 中，不再先写入临时文件。

具体改动包括：
- 将 `tmhtml-image` 从 `define` 改为 `tm-define`，使其可被单元测试访问。
- 在 `tmhtml-image` 函数的嵌入式图片分支中，当 `tmhtml-base64?` 为 true 时，直接使用 `(liii base64)` 模块将图片二进制数据编码为 Base64 字符串。
- 生成 `data:image/<ext>;base64,<base64-str>` 格式的内联 URI 作为 `src` 属性。
- 支持提取并设置图片的 `width` 和 `height` 属性。
- 移除了原先在 Base64 模式下写入临时文件后再删除临时文件的冗余逻辑。
- 新增单元测试文件 `TeXmacs/tests/0416.scm`，覆盖 Base64 直接编码、尺寸属性提取、无尺寸属性三种场景。

## 6 Why
1. **减少磁盘 I/O**：原先的实现即使在 Base64 模式下也会先将图片写入临时文件，生成 HTML 后再删除，产生了不必要的磁盘操作。直接编码可以避免这一过程。
2. **简化代码逻辑**：将 Base64 路径和普通路径的处理逻辑分开，使各自的分支更加清晰。
3. **提升导出效率**：直接内存编码避免了文件系统调用的开销，特别是在批量处理大量嵌入式图片时效果更明显。

## 7 How
1. **条件分支重构**：将原先统一的"先写临时文件"逻辑改为条件分支：
   - 当 `tmhtml-base64?` 为 true 时，走 Base64 直接编码路径。
   - 当 `tmhtml-base64?` 为 false 时，保持原来的临时文件写入路径。
2. **Base64 编码**：使用 `(import (liii base64))` 和 `bytevector-base64-encode` 函数将二进制数据编码为 Base64 字符串。
3. **生成 data URI**：拼接 `data:image/<ext>;base64,<base64-str>` 作为图片的 `src` 属性。
4. **尺寸属性提取**：通过 `tmlength->htmllength` 提取图片的宽度和高度，作为 `width` 和 `height` 属性添加到 `<img>` 标签中。
5. **生成标准 HTML 结构**：使用 `(h:img (@ (class "image") (src ,src-uri) ...))` 格式输出标准 HTML 图片标签。
