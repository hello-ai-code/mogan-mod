## 5 What
将 `text-bg-color` 路径下的文本背景色从逐 text_box 直角矩形绘制，改为在 `composite_box`（如 `concat_box`）层级统一绘制完整圆角矩形，使视觉效果更加柔和美观。

1. 在 `composite_box`（如 `concat_box_rep`）的 `pre_display()` 中统一扫描并绘制连续背景色片段的圆角矩形
2. 屏蔽子 `text_box` 的重复背景绘制，避免覆盖
3. 参考 `highlight_boxes.cpp` 中已有的圆角实现，保持代码风格一致
4. 确保圆角半径与字体大小成合理比例，在不同字号下均协调

## 6 Why
当前文本背景色在每个 `text_box` 中独立绘制直角矩形。由于文本排版后一个中文通常对应一个 `text_box`，一段有背景色的文字实际上是由多个相邻直角矩形拼接而成。这种方式不仅视觉效果生硬，而且字符之间可能存在微小间隙。将绘制层级提升到 `composite_box`，可以一次性绘制完整、美观的圆角背景，彻底解决拼接问题。

## 7 How

### 7.1 当前实现分析

#### 7.1.1 背景色文本框的创建流程

背景色文本框的创建发生在排版阶段，核心路径如下：

1. `concat_text.cpp:56-66` 的 `build_text_box()` 函数读取环境变量 `TEXT_BG_COLOR`
2. 若存在有效背景色（非空且非 white），调用 `text_box_with_bg()` 创建带背景色的文本框
3. `text_box_with_bg()`（`text_boxes.cpp:718-721`）将颜色参数传入 `text_box_rep` 构造函数，保存在成员变量 `bg_color` 中

#### 7.1.2 背景色的绘制实现

背景色的实际绘制发生在 `text_boxes.cpp:201-230` 的 `text_box_rep::display()` 方法中：

```cpp
void text_box_rep::display (renderer ren) {
  // 如果有背景色（非透明），先绘制背景
  int r, g, b, a;
  get_rgb_color (bg_color, r, g, b, a);
  if (a > 0) {
    brush bg_brush (bg_color);
    ren->set_background (bg_brush);
    SI     bg_x1= x1;
    SI     bg_x2= x2;
    metric ex_m;
    fn->get_extents ("M", ex_m);
    SI bg_y1= ex_m->y1 - 10 * ren->pixel; // 向下延伸10个像素
    SI bg_y2= ex_m->y2 + 10 * ren->pixel; // 向上延伸10个像素

    // 检查背景宽度是否足够
    SI bg_width= bg_x2 - bg_x1;
    SI m_width = ex_m->x2 - ex_m->x1;
    if (bg_width >= m_width / 4) {
      ren->clear (bg_x1, bg_y1, bg_x2, bg_y2);  // <-- 直角矩形填充
    }
  }
  // ... 绘制文本
}
```

关键问题在第 220 行：`ren->clear(bg_x1, bg_y1, bg_x2, bg_y2)` 调用的是渲染器的矩形清除方法，只能填充直角矩形，无法直接绘制圆角。

#### 7.1.3 逐字（逐 text_box）绘制背景色的特点

这是当前实现中最关键的约束。`text_box_rep::display()` 中绘制的背景色矩形范围是**当前 text_box 自身的边界**（`x1` 到 `x2`），而非整段连续文本的边界。

在排版阶段（`concat_text.cpp:125-149` 的 `typeset_text_string`），文本会被语言分词器切分为若干段（`typeset_substring` 的调用粒度）。对中文而言，通常**一个字符对应一个 `text_box`**；对英文而言，通常**一个单词对应一个 `text_box`**。每个 `text_box` 独立调用 `display()` 绘制自己的背景。

例如输入"中国人"三个字并设置背景色，实际绘制过程是：先绘制"中"字的背景色矩形，再绘制"国"字的背景色矩形，再绘制"人"字的背景色矩形。三个直角矩形拼接成一条连续背景色带。

这意味着：
- 一段有背景色的连续文本，实际上是由多个相邻的直角矩形拼接而成
- 字符之间**可能已经存在微小间隙**，在直角情况下通常不明显；但如果引入圆角，间隙会导致背景色断裂
- 如果直接在每个 `text_box` 上绘制完整圆角，相邻矩形的连接处会出现背景色断裂（圆角凹陷露出底色）或重叠变形

#### 7.1.4 marked（高亮）的实现路径（已移除表格方案）

`marked` 标签原在 `std-markup.ts:483` 中通过表格（`tabular`）实现背景色。由于数学模式下字符结构特殊，表格曾是必要的临时方案。

现在 `text-bg-color` 已实现 box 级别统一绘制圆角背景，足以覆盖数学公式和混合内容。因此已将 `marked` 改为直接使用 `text-bg-color` 实现，移除了表格临时方案。

当前编辑层的 `mark-text`（`generic-edit.scm:1984-1993`）统一使用 `text-bg-color` 路径，不再区分纯文本与非纯文本结构。

#### 7.1.5 现有圆角绘制参考

`highlight_boxes.cpp` 中已经实现了完善的圆角矩形绘制，可直接参考：

1. `rounded()` 函数（`highlight_boxes.cpp:258-301`）：通过三角函数参数化生成 90 度圆弧的顶点序列，支持 `ROUNDED_NORMAL`（正常圆弧）、`ROUNDED_ANGULAR`（近似直角）、`ROUNDED_CARTOON`（卡通风格）三种风格
2. `display_rounded()` 方法（`highlight_boxes.cpp:315-413`）：调用 `rounded()` 生成四个角的顶点，拼接成闭合多边形，最后用 `ren->polygon(xs, ys)` 填充

#### 7.1.6 渲染流程的关键时机

从 `box_rep::redraw()`（`boxes.cpp:633-677`）的渲染流程可以看出：

```
pre_display()        // 父 box 先执行
  → 递归 subbox->redraw()  // 所有子 box 绘制
  → display()            // 父 box 自身绘制（composite_box 为空）
→ post_display()
```

`composite_box_rep::display()` 当前是空实现，所有子 box 的 `display()` 被独立调用。`highlight_box` 正是利用 `pre_display()` 在子内容绘制**之前**画背景，从而背景不会覆盖文本。

这一流程为"在更高层级统一绘制背景色"提供了天然时机：在 `concat_box_rep` 的 `pre_display()` 中先统一画好圆角背景，随后子 `text_box` 的 `display()` 只负责绘制文本即可。

### 7.2 修改方案：在 composite_box 层级统一绘制

#### 7.2.1 核心思路

对于同一行（或同一段落）中连续有背景色的内容，不在每个 `text_box` 里分别绘制背景，而是提升到承载这些 text_box 的 `composite_box`（如 `concat_box`）中统一绘制一次完整的圆角矩形。

具体步骤如下：

1. **扫描子 box**：在 `concat_box_rep` 的 `pre_display()` 中遍历所有子 box，识别类型为 `TEXT_BOX` 且 `bg_color` 非透明的 box
2. **归并连续片段**：将相邻且背景色相同的 `text_box` 归为一个连续片段
3. **计算整体边界**：取片段内所有子 box `x1`/`x2` 的最小/最大值作为水平边界；取统一的 `y1`/`y2`（可用字体高度或 "M" 字度量）作为垂直边界
4. **绘制圆角背景**：仿照 `highlight_boxes.cpp` 中的 `rounded()` 函数生成四圆角顶点，用 `ren->set_brush()` + `ren->polygon()` 一次性填充完整圆角矩形
5. **屏蔽重复绘制**：临时将已统一绘制背景的子 `text_box` 的 `bg_color` 设为透明（`rgb_color(0,0,0,0)`），使其 `display()` 不再重复绘制背景；在 `post_display()` 中恢复原始颜色

#### 7.2.2 涉及的代码修改点

1. **`text_box_rep` 暴露背景色信息**：
   - 增加 `get_bg_color()` 公共方法，或声明 `concat_box_rep` 为友元，供上层扫描使用
   - 确保 `bg_color` 在运行时可被临时修改（当前已是非 const 成员）

2. **`concat_box_rep` 增加 `pre_display()` 与 `post_display()`**：
   - 在 `concat_boxes.cpp` 中重写 `pre_display(renderer& ren)`
   - 在 `pre_display()` 中实现扫描、归并、绘制逻辑
   - 重写 `post_display(renderer& ren)` 恢复被临时修改的子 box 背景色

3. **圆角顶点生成**：
   - 将 `highlight_boxes.cpp` 中的 `rounded()` 函数或其简化版本提取到公共头文件，或直接在 `concat_boxes.cpp` 中内联实现
   - 圆角半径以字体高度（`fn->yx`）为基准取固定比例（如 1/5），确保不同字号下协调

4. **退化处理**：
   - 若片段整体宽度小于圆角直径，缩小圆角半径或退化回直角矩形

#### 7.2.3 其他 composite box 的扩展

`cell_box`（表格单元格）也是 composite box，内部同样包含 `text_box`。若后续需要表格单元格背景色也支持圆角，可采用相同模式：在 `cell_box` 的 `pre_display()` 中统一绘制。本次可先聚焦 `concat_box`。

### 7.3 注意事项

1. **渲染顺序至关重要**：`pre_display()` 在子 box 绘制之前执行，因此统一绘制的背景不会覆盖文本。切勿在 `display()` 中绘制背景，否则背景会盖住子内容
2. **临时修改的恢复**：`redraw()` 是同步单线程调用，将子 box `bg_color` 临时设为透明并在 `post_display()` 恢复是安全的。但需注意异常路径（如 `gui_interrupted()` 提前返回）是否会导致颜色未恢复
3. **跨 composite box 的连续性**：如果一段背景色文本跨越多个 composite box（如嵌套结构），统一绘制会在每个 composite box 内部独立完成，边界处仍可能被打断。但通常一行文本都在同一个 `concat_box` 内，可以接受
4. **性能**：统一绘制只需计算一次圆角顶点并调用一次 `polygon()`，性能优于逐 box 绘制
5. **与 `highlight_boxes.cpp` 的区别**：`highlight_box` 本身就是一个包裹完整内容的大 box，天然具备统一绘制条件；`text_box` 是碎片化的，本次方案通过提升绘制层级到 `composite_box`，本质上是把碎片化的 text_box 在渲染阶段重新聚合成连续的背景区域
