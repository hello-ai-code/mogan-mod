## 5 What

为 Chat Tab 实现自动滚动到底部功能，使用 TeXmacs 光标驱动机制（方案 v3）。

1. 在 `chat-tab-output` 中将 `(set-user-active #f)` 替换为 `(tree-go-to t :end)`
2. 在 `chat-tab-append-round!` 中将 `(set-user-active #f)` 替换为 `(tree-go-to doc :end)`
3. 撤销方案 v2 的全部 Qt 滚动条代码（`setup_auto_scroll`、`scroll_message_to_bottom`、`autoScroll` 字段）

## 6 Why

Chat Tab 聊天消息增多时，消息区域无法自动滚动到底部，用户需要手动滚动才能看到最新回复，体验不佳。

之前的尝试：
- **方案 v1**（Scheme→C++ glue 调用滚动函数）失败：`tree-pointer->tree` 错误，插件回调上下文冲突
- **方案 v2**（Qt `QScrollBar::rangeChanged` 信号驱动）失败：操控滚动条会重入 TeXmacs 渲染引擎，导致崩溃/黑屏

方案 v3 利用 TeXmacs 已有的 `user_active` + `cursor_visible()` 机制，不直接操作 Qt 控件，避免了前两个方案的问题。

## 7 How

### 7.1 核心原理

TeXmacs 内部已有的滚动机制：

```
tree-go-to t :end
  → C++ go_to() 自动 set_user_active(true)
  → interpose_handler() 每帧调用
    → apply_changes()
      → user_active == true → cursor_visible() → 自动滚动到光标
      → set_user_active(false)
```

commit 88fd902c9 和 86e98892b 在 session 环境下通过 `(set-user-active #f)` 禁止了插件输出时的自动滚动。Chat Tab 需要恢复滚动，所以改为 `(tree-go-to t :end)`——光标移到末尾，触发 `user_active`，TeXmacs 自动滚动。

### 7.2 Scheme 层改动

`chat-tab-session.scm` 两处改动，模式相同：

```scheme
;; 改前
(set-user-active #f)

;; 改后
(tree-go-to t :end)  ;; 或 (tree-go-to doc :end)
```

### 7.3 C++ 层改动（撤销方案 v2）

删除方案 v2 引入的全部 Qt 滚动条相关代码：

- `hpp`：删除 `setup_auto_scroll` 和 `scroll_message_to_bottom` 声明
- `cpp`：删除 `ChatConversationPanel` 的 `autoScroll` 字段、`create_conversation` 和 `restore_conversation` 中的 `setup_auto_scroll` 调用、`handle_send` 和 `notifyStateChanged` 中的 `autoScroll` 赋值、两个方法的完整实现
