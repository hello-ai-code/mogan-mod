# [0251] njson 异常兜底处理

## 如何测试
1. 构建：`xmake b stem`
2. 在 Mogan STEM 中执行 `(test_0251)`
3. 期望所有 check 通过，不崩溃

## 2026/06/05

### Why
Mogan 通过 nlohmann/json (njson) 库进行 JSON 操作。C++ 层 `goldfish.hpp` 中部分 `f_njson_*` 函数缺少 try-catch，未捕获的 C++ 异常可能逃逸出 s7 FFI 边界导致应用崩溃。非 ASCII 字符串（如 "Erwin Schrödinger"）和畸形 JSON 是潜在的崩溃触发器。

### What
为所有 njson C++ 函数添加 try-catch 兜底，在 Scheme 层添加 `catch #t` 安全网，确保异常不会导致崩溃。

### How

#### C++ 层 (`goldfish.hpp`)
为以下函数添加 `catch (const json::exception&)` 和 `catch (const std::exception&)`：
- P0: `f_njson_json_to_string`、`njson_run_structure_conversion`、`njson_run_update`、`f_njson_ref`
- P1: `f_njson_keys`、`f_njson_contains_key_p`
- P2: `f_njson_free`、`f_njson_size`、`f_njson_empty_p`

#### Scheme 层 (`njson.scm`)
为高风险 `g_njson-*` 调用添加 `catch #t` 包装。

### Test plan
- 非 ASCII 字符串解析和序列化（"Erwin Schrödinger"、中文、日文、希腊字母）
- 畸形 JSON（缺少括号、尾随逗号、单引号、空字符串）
- 无效 JSON 操作（不存在的 key、越界、类型不匹配）
- 结构转换中的非 ASCII 处理
- 边界情况（空对象/数组、深层嵌套）
