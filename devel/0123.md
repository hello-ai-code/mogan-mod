## 5 What

移除项目中未使用的截图功能及其依赖的 qhotkey 第三方库，精简代码并修复 Debian 13 上的 CI 构建失败。

1. 删除 `src/Plugins/Qt/screenshot_tool.hpp` 和 `screenshot_tool.cpp`
2. 删除整个 `3rdparty/qhotkey/` 目录
3. 从 `src/Mogan/Research/research.cpp` 中移除截图工具初始化代码和 QHotkey 头文件引用
4. 从 `xmake.lua` 中移除 `libqhotkey` target 定义及 `libmogan` 对该 target 的依赖

## 6 Why

- 截图功能当前未在 Mogan 中使用，属于无效代码
- qhotkey 依赖（尤其是 X11 相关代码）导致 GitHub Actions 上 Debian 13 的 CI 构建失败
- 移除这些代码可以减少项目体积、降低维护成本，并修复 CI

## 7 How

### 7.1 截图工具移除

截图工具仅在 Linux 平台下通过 `src/Mogan/Research/research.cpp` 初始化：

```cpp
#if defined(Q_OS_LINUX)
    ScreenshotTool* screenshotTool= new ScreenshotTool (nullptr);
    if (QHotkey::isPlatformSupported ()) {
      QHotkey* hotkey= new QHotkey (QKeySequence ("Ctrl+Alt+X"), true, qtmapp);
      QObject::connect (hotkey, &QHotkey::activated, qApp, [screenshotTool] () {
        screenshotTool->startCapture ();
      });
    }
#endif
```

直接删除该段代码及其头文件包含即可。

### 7.2 qhotkey 移除

qhotkey 是一个跨平台全局热键库，以静态库 `libqhotkey` 的形式集成在 `xmake.lua` 中：

```lua
target("libqhotkey") do
  set_kind("static")
  add_rules("qt.static")
  add_rules("qt.moc")
  add_packages("qt6widgets")
  if is_plat("linux") then
    add_packages("x11")
  end
  -- ... 源文件配置
end
```

删除整个 target 定义，并在 `libmogan` target 中移除 `add_deps("libqhotkey")` 和相关的 `add_includedirs("3rdparty", {public = true})` 注释。

### 7.3 验证无残留引用

删除后全局搜索 `screenshot_tool`、`qhotkey`、`QHotkey` 确认无残留引用，确保清理彻底。
