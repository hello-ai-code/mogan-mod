# [0413] 版本不一致对话框只留一个确认按钮

## 1 相关文档
- [0412.md](0412.md) - 帮助菜单新增版本信息按钮及按需跳转逻辑

## 2 任务相关的代码文件
- `TeXmacs/progs/doc/help-funcs.scm`

## 3 如何测试

### 3.1 手动更改版本（模拟测试环境）
在 `TeXmacs/progs/doc/help-funcs.scm` 的 `(mogan-version)` 中，修改 `cur-ver` 的绑定即可模拟不同版本状态：
- **版本一致测试**：保持 `cur-ver` 为 `(xmacs-version)` 或手动写死为 `"2026.2.5"`（与 `latest-ver` 相同）。
- **版本不一致测试**：将 `cur-ver` 临时改写为较低版本，如 `"2026.2.4"`：
  ```scheme
  (let* ((cur-ver "2026.2.4")
  ```

### 3.2 验证步骤
1. **版本一致**：点击 "帮助 -> 版本"，确认弹出普通信息对话框（只有确定按钮），无官网跳转逻辑。
2. **版本不一致**：点击 "帮助 -> 版本"，确认弹出信息对话框（只有确定按钮），点击 "确定" 可正常跳转到官网 `https://liiistem.cn`。
3. **关闭对话框**：版本不一致时，点击对话框右上角的 X，确认不会跳转官网。
4. **语言验证**：分别在简体中文、繁体中文和英文语言下运行测试，确保文案、标题完全正常。

## 4 如何提交

提交前执行以下最少步骤：
```bash
xmake b stem
```

## 5 What
优化了版本不一致时的交互逻辑，将原来的 "是/否" 询问对话框改为只有一个 "确定" 按钮的信息对话框：
- 如果版本不一致，弹出信息对话框（单确定按钮），点击确定后跳转至官网 `https://liiistem.cn` 下载最新稳定版。
- 如果用户不想跳转，点击对话框右上角的 X 关闭即可，不会触发跳转。
- 官网跳转 URL 增加了 utm 参数，用于追踪来源。

## 6 Why
1. **简化用户决策**：原来使用 `user-confirm` 提供 "是/否" 两个选项，但实际上用户看到版本不一致时，去官网下载是明确的主路径。减少一个选项可以降低用户的认知负担。
2. **保留关闭选项**：用户如果暂时不想升级，可以通过点击对话框右上角的 X 关闭，不会强制跳转。
3. **utm 追踪**：增加 utm 参数（`utm_source=mogan-app&utm_medium=referral&utm_campaign=version-check`）便于分析版本检查功能的实际转化效果。

## 7 How
1. **新增辅助函数**：在 `TeXmacs/progs/doc/help-funcs.scm` 中新增 `show-message-with-callback` 函数，基于 `dialogue-window` 和 `message-widget` 实现，点击 OK 后执行回调操作：
   ```scheme
   (tm-define (show-message-with-callback msg title callback)
     (:interactive #t)
     (dialogue-window (message-widget msg) callback title))
   ```
2. **修改 `mogan-version` 函数**：将版本不一致分支的 `user-confirm` 替换为 `show-message-with-callback`：
   - 文案从询问语气改为陈述+引导语气（如 "请点击确认前往官网下载最新稳定版"）。
   - 回调函数中执行 `(open-url url)` 跳转官网。
   - URL 增加 utm 参数：`https://liiistem.cn?utm_source=mogan-app&utm_medium=referral&utm_campaign=version-check`。
3. **保持版本一致逻辑不变**：版本一致时仍然使用 `show-message` 显示普通信息对话框。
