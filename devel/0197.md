## 5 What
创建 LLM 插件的 REPL 模块 `tm-llm.scm`，实现 `eval-and-print` 函数，将从 Mogan 插件获得的输入通过 `flush-scheme-u8` 放到 code 环境中。

1. 创建 `(texmacs tm-llm)` 库，提供 `llm-welcome` 和 `llm-repl`
2. 实现 `eval-and-print`，参数名使用 `data`（避免与 code 环境标记混淆）
3. 支持大输入（>1M）时自动写入临时文件并返回文件路径
4. 使用 `(liii path)` 的 `path-write-text`、`path-temp-dir`、`path-join` 处理文件操作

## 6 Why
LLM 插件需要一个独立的 REPL 来处理从 Mogan 插件中获得的输入。参数名使用 `data` 而非 `code`，是为了与 TeXmacs 的 code 环境标记名区分开，避免概念混淆。例如，当 code 环境里有三行内容（如 `(code (document "1" "22" "333"))`）时，`data` 这个参数名更清晰地表达"这是输入数据"而非"这是 code 环境本身"。

## 7 How

1. `eval-and-print` 接收 `data` 参数（从 Mogan 插件获得的输入字符串）
2. 小数据（<=1M）：直接通过 `flush-scheme-u8` 输出到 code 环境
3. 大数据（>1M）：使用 `(liii path)` 的 `path-write-text` 写入临时文件（路径由 `path-temp-dir` + `path-join` 生成），通过 `flush-scheme-u8` 返回 `(document "(temp-file <path>)")`
