## 5 What

为 Chat Tab 会话添加图片等非文本内容的兜底处理。当用户发送包含图片的消息时：

1. 不在 message buffer 中渲染图片本身
2. 将用户输入的文本部分保留显示在历史记录中
3. 在 output 区域显示中文提示，告知用户图片被过滤
4. 不将消息发送给 LLM 插件，避免 base64 垃圾数据进入 API 请求
5. 在提取历史上下文时，将图片元素替换为 `"filtered image"` 占位符，而非空字符串

## 6 Why

旧 session 模式（`make-session`）通过 `%scheme` 协议支持图片多模态上传：图片被检测后转为 `image-content :from-base64`，以结构化 JSON 发送给 VLM API。新的 Chat Tab 模式使用 `%context` 协议，通过 `chat-tab-build-context-input` 将消息历史转为纯文本 JSON，但存在以下问题：

- `chat-tab-tree->plain-text` 对 `(image ...)` 元素没有特殊处理，将其子元素（base64 数据、png、尺寸参数）递归拼接为纯文本，导致历史上下文中混入无意义的 base64 垃圾字符串
- 即使后续调用 LLM API，也是通过纯文本 `llm-message` 而非多模态 `vlm-message`，图片数据无法被正确解析
- 中文字符串直接通过 `stree->tree` 放入 TeXmacs 树时，由于 TeXmacs 内部使用 cork 编码，导致中文显示为乱码

本任务作为兜底方案，在 Chat Tab 层面拦截含图片的消息，给出明确反馈，避免无效 API 调用和数据污染。

## 7 How

修改 `TeXmacs/progs/dynamic/chat-tab-session.scm`，共三处改动：

### 7.1 `chat-tab-tree->plain-text` — 图片元素占位符

在 `cond` 分支中增加 `(image ...)` 的 case，返回 `"filtered image"` 而非空字符串：

```scheme
((eq? (car s) 'image) "filtered image")
```

这样当从历史记录提取上下文时，图片位置保留可读占位标记，而非完全消失或混入 base64 数据。

### 7.2 `chat-tab-tree-has-image?` — 新增图片检测函数

递归检测 tree/stree 中是否包含 `image` 元素：

```scheme
(define (chat-tab-tree-has-image? t)
  (let ((s (if (tree? t) (tree->stree t) t)))
    (cond ((string? s) #f)
          ((not (pair? s)) #f)
          ((eq? (car s) 'image) #t)
          (else
            (let loop ((rest (cdr s)))
              (if (null? rest)
                #f
                (or (chat-tab-tree-has-image? (car rest))
                  (loop (cdr rest)))))))))
```

使用命名循环 `loop` 而非 `any`（SRFI-1），避免 Guile 环境依赖问题。

### 7.3 `chat-tab-session-send` — 图片拦截与提示输出

将图片检测放在 `chat-tab-append-round!` 之前。若检测到图片：

1. 清空输入框 `chat-tab-clear-input!`
2. 用 `chat-tab-tree->plain-text` 过滤掉图片，得到纯文本
3. 用过滤后的纯文本调用 `chat-tab-append-round!`（图片不会出现在 message buffer）
4. 在 output 中输出灰色斜体提示，中文通过 `utf8->cork` 转换避免乱码：

```scheme
(chat-tab-output out
  (stree->tree `(document
    (with "color" "dark grey" "font-shape" "italic"
      ,(utf8->cork "AI 聊天暂不支持图片等非文本内容，相关内容已过滤。")))))
```

5. 不调用 `chat-tab-session-feed`，消息不发送给插件

若未检测到图片，走原有的正常发送流程不变。
