# [200_61] 修复 Apple Silicon (macOS ARM64) 上 Qt 6.8.3 编译失败的问题

## 如何测试

1. 在 Apple Silicon Mac (M1/M2/M3/M4/M5) 上，确保已安装 Xcode 和 xmake
2. 执行 `xmake build stem`
3. 预期结果：
   - 编译能够完整通过，不出现 `error: implicitly declaring library function '__yield'` 错误
   - 链接成功，生成 `MoganSTEM.app`

## 2026/05/06 实现说明

### What

修复在 Apple Silicon (arm64) macOS 上，使用 Qt 6.8.3 编译 Mogan STEM 时因 `qyieldcpu.h` 中隐式声明 `__yield()` 而导致构建失败的问题。

### Why

- Qt 6.8.3 的 `QtCore/qyieldcpu.h` 在 Apple Silicon 分支下直接调用了 `__yield()` 内建函数，但没有包含 `<arm_acle.h>` 头文件。
- 新版 Xcode/Clang 将 `-Wimplicit-function-declaration` 视为错误（`-Werror`），导致编译中断。
- 这是一个已知的 Qt Bug（[QTBUG-145239](https://bugreports.qt.io/browse/QTBUG-145239)），在 Qt 官方修复之前需要通过构建配置做兼容处理。
- 该问题仅影响 Apple Silicon (arm64)；Intel x86_64 走 `__builtin_ia32_pause` 分支，不受影响。

### How

**修改文件**

**`xmake.lua`**
- 在 macOS 平台配置段后新增条件编译标志：
  - 当平台为 `macosx` 且架构为 `arm64` 时：
    - `add_cxxflags("-Wno-error=implicit-function-declaration")` —— 对 C++ 源文件生效
    - `add_mxflags("-Wno-error=implicit-function-declaration")` —— 对 Objective-C++ (`.mm`) 源文件生效
- 这样仅把隐式函数声明的报错降级为警告，不影响其他代码的质量检查。

### 涉及文件

- `xmake.lua`

### 行为变化

- Apple Silicon Mac 上 `xmake build stem` 现在可以成功完成编译和链接
- 编译期间 `qyieldcpu.h:35` 仍会输出一次警告，但不再阻断构建
- Intel Mac 和其他平台不受任何影响

