## 2025/03/24 所有内嵌图片直接写入内存
### What
将内嵌图片（ramdisc URL）的加载路径从"临时文件"改为"直接内存读取"：

1. **Qt 图片处理层** (`qt_utilities.cpp`, `qt_picture.cpp`)
   - `qt_image_size`, `qt_native_image_size`, `qt_convert_image`, `get_image_for_real`
   - 对 ramdisc URL 使用 `loadFromData` 直接从内存加载

2. **MuPDF 渲染层** (`mupdf_renderer.cpp`)
   - `mupdf_read_from_url` 对 ramdisc URL 直接创建内存 buffer

3. **通用图片处理层** (`image_files.cpp`)
   - `image_size_sub`, `image_to_png` 对 ramdisc URL 跳过 `concretize`

4. **页眉页脚解码** (`pager.cpp`)
   - 新增 `decode_images_in_tree` 函数
   - 检测并解码页眉页脚中的 base64 图片数据

### Why
**优化对比：**

| 指标 | 原方案 | 新方案 |
|------|--------|--------|
| 100页文档内嵌图片渲染 | 300次磁盘 I/O | 0次磁盘 I/O |
| 内存占用 | 临时文件（磁盘） | 进程内存 |
| 代码复杂度 | 临时文件生命周期管理 | 直接内存访问 |

### 实现要点

**内存加载核心代码：**
```cpp
// qt_utilities.cpp
if (is_ramdisc(image)) {
    url    root_part = image[1];
    url    data_url  = root_part[2];
    string img_data  = data_url->t->label;
    im.loadFromData((const uchar*) img_data.begin(), N(img_data));
}
```

**跳过临时文件：**
```cpp
// image_files.cpp
url image = is_ramdisc(p_image) ? p_image : concretize(p_image);
```

**页眉页脚解码：**
```cpp
// pager.cpp
static tree decode_images_in_tree(tree t) {
    if (is_func(t, IMAGE) && is_func(t[0][0], RAW_DATA)) {
        string img_data = as_string(t[0][0][0]);
        char c = img_data[0];
        // base64 检测：首字符为 A-Za-z0-9+/
        if (is_base64_char(c)) {
            string decoded = decode_base64(img_data);
            return tree_with_new_raw_data(t, decoded);
        }
    }
    return t;
}
```

### 边界情况
- 外部链接图片保持原有行为（仍使用临时文件）
- 大图片（>10MB）：现代系统内存充足，不成问题
