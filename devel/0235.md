## 5 What
让 chat-tab 的消息区域（message buffer）变为只读，用户无法编辑消息内容，但可以复制和滚动。

1. 在 message widget 的 QTMWidget 上设置 `chat_message_readonly` 属性并安装 event filter
2. 将只读事件过滤逻辑提取为公开静态方法 `should_block_readonly_event`，便于单元测试
3. 采用白名单机制：仅放行 Ctrl/⌘+C（复制）、Ctrl/⌘+A（全选）、Ctrl/⌘+F（搜索），拦截粘贴、剪切、撤销等编辑操作

## 6 Why
消息区域展示的是历史对话内容，用户不应编辑它。但需要保留鼠标选择复制、滚动浏览，以及复制、全选、搜索等快捷键功能。

## 7 How
通过 Qt event filter 机制实现：

1. 创建 message widget 后，找到其内部的 `QTMWidget` 子控件，设置 `chat_message_readonly` 属性标记，并安装 event filter
2. 将只读事件过滤逻辑提取为公开静态方法 `should_block_readonly_event`，便于单元测试：
   - 拦截 `QEvent::InputMethod`（阻止 IME 输入）
   - 拦截无修饰键的 `QEvent::KeyPress`/`KeyRelease`（阻止普通文字输入）
   - 对带 Ctrl/Meta 修饰键的按键采用白名单：仅放行 C（复制）、A（全选）、F（搜索），拦截 V（粘贴）、X（剪切）、Z（撤销）等
3. 跨平台适配：Windows/Linux 用 Ctrl，macOS 用 ⌘（Command），两者分别映射为 `Qt::ControlModifier` 和 `Qt::MetaModifier`

### 7.1 方案选择

**`texmacs_output_widget` 不可用**：它将文档静态排版为 `box_widget`（固定大小的渲染结果），无滚动、无选择/复制、无法动态更新内容，不满足消息区域的需求。因此选择在 `texmacs_input_widget` 基础上通过 event filter 实现只读。
