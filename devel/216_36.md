# 216_36 启动页移除 Settings 功能及相关代码清理

## 如何测试

1. 编译：`xmake b stem`
2. 启动 Mogan，打开启动页。
3. 确认左侧导航栏仅显示 **3 个**按钮：File、Template、Open a Document、Quit（无 Settings 按钮）。
4. 确认右侧内容区仅堆叠 **2 个**页面：File（index 0）、Template（index 1）。
5. 点击 File / Template 导航按钮，确认页面切换正常。
6. 分别验证亮色/暗色主题下导航栏样式正常，无残留 Settings 相关样式异常。
7. 确认 `src/Plugins/Qt/qt_settings_page.cpp` 和 `qt_settings_page.hpp` 已删除。
8. 确认 `TeXmacs/progs/startup-tab/startup-tab-settings.scm` 已删除。

## 2026/04/30 实现说明

### What

从启动页（`QTStartupTabWidget`）完全移除 Settings 入口及其对应页面，同步删除 `QTSettingsPage` 实现、Scheme 侧 widget 定义以及 CSS 中 Settings 专属样式。

#### 修改文件

**src/Plugins/Qt/qt_startup_tab_widget.cpp**
- 移除 `#include "qt_settings_page.hpp"`。
- 构造函数初始化列表移除 `settingsPage_` 和 `navSettingsBtn_`。
- `setup_left_sidebar`：移除 Settings 导航按钮的创建、互斥按钮组注册、布局添加和信号连接；导航入口从 4 个减为 3 个。
- `setup_right_content`：移除 `create_settings_page()` 调用及 `Entry::Settings` 分支映射；堆叠页面从 3 个减为 2 个。
- 删除 `create_settings_page()` 函数。

**src/Plugins/Qt/qt_startup_tab_widget.hpp**
- 移除 `QTSettingsPage` 前向声明。
- `Entry` 枚举移除 `Settings` 值。
- 移除 `create_settings_page()` 声明。
- 移除 `navSettingsBtn_` 和 `settingsPage_` 成员变量。

**TeXmacs/misc/themes/liii.css / liii-night.css**
- 移除 `QTabWidget#settings-tab-widget` 样式块（pane、tab、hover、selected 规则）。

#### 删除文件

- **src/Plugins/Qt/qt_settings_page.cpp** — `QTSettingsPage` Qt 容器页实现。
- **src/Plugins/Qt/qt_settings_page.hpp** — `QTSettingsPage` 类声明。
- **TeXmacs/progs/startup-tab/startup-tab-settings.scm** — Scheme 侧 `startup-settings-widget` 定义。

### Why

1. **职责归位**：Settings 功能与首屏核心操作路径无关，用户更习惯通过菜单栏 `Edit -> Preferences` 或后续独立入口访问设置，启动页保留 File / Template 两个高频入口职责更清晰。
2. **减少维护面**：`QTSettingsPage` 与 `startup-settings-widget` 构成 Qt + Scheme 混合渲染链路，代码量不小但使用频率低；移除后减少跨层维护成本。
3. **首屏简化**：导航栏从 4 个入口减为 3 个，视觉和交互都更聚焦。
4. **样式清理**：Settings 专属 CSS 规则（`settings-tab-widget`）不再被任何控件使用，移除避免主题文件膨胀。

### How

- 从 `QTStartupTabWidget` 的侧边栏、堆叠控件、`Entry` 枚举三层彻底剥离 Settings。
- 直接删除 `qt_settings_page.cpp/hpp` 和 `startup-tab-settings.scm`，不再保留未引用文件。
- CSS 中搜索 `settings-tab-widget` 选择器并整段移除，确保主题文件无死代码。
