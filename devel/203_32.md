# [203_32] 修复 LaTeX 粘贴时 \setlength 解析导致多余 `}` 的问题

## 如何测试

1. 准备一段包含 `\setlength{\tabcolsep}{5pt}` 的 LaTeX 表格代码，例如：

```latex
\setlength{\tabcolsep}{5pt}
\begin{tabular}{|c|c|}
\hline
A & B \\
\hline
\end{tabular}
```

2. 在 Mogan 中使用**选择性粘贴 → LaTeX**（或 `Ctrl+V` 自动检测为 LaTeX）粘贴上述代码。
3. 确认粘贴后的表格**左侧没有多余的 `}` 符号**。
4. 确认去掉 `\setlength{\tabcolsep}{5pt}` 后粘贴行为与之前一致。
5. 确认 `\setlength` 的其他用法（如 `\setlength{\arraycolsep}{3pt}`）也不会产生多余字符。

## 2026/04/28

### What
- 仅影响 `\setlength` 命令的 LaTeX 解析逻辑，不影响其他 LaTeX 命令的解析行为。
- 修复了从 LaTeX 粘贴包含 `\setlength{...}{...}` 的代码时，解析器漏掉消费第二个参数后的 `}`，导致后续内容被插入一个多余的 `}` 字符的 bug。

### Why

`parsetex.cpp` 中 `parse_command` 函数对 `\setlength` 的处理分支调用了 `parse_argument(s, i)` 来解析 `{5pt}`，而 `parse_argument` 内部调用 `parse(s, i, "}")` 时，遇到 `}` 会**停止但不消费**它。

其他命令（如通用参数解析循环）在调用 `parse(s, i, "}")` 之后都会执行：

```cpp
if ((i < n) && (s[i] == '}')) i++;
```

来消费掉这个 `}`。但 `\setlength` 的特殊处理分支没有这一步，导致 `}` 留在输入流中，被后续解析当作普通字符输出。
