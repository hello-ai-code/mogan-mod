# [201_46] 非社区版安装后首次或者更新后打开软件，出现引导弹窗

## 如何测试
1. 社区版，不会出现引导弹窗
```
xmake config --is_community=true
xmake b && xmake r stem
```

2. 非社区版，未登录状态下，出现引导弹窗
- 进入非社区版
```
xmake config --is_community=false
xmake b && xmake r stem
```

3. 点击引导弹窗的登陆按钮，进入主界面后自动进入登陆流程

4. 点击引导弹窗的“退出软件”按钮，直接退出软件，不进入主界面

5. 把引导弹窗X掉，不进入主界面

6. 自动备份勾选框测试：
   - 勾选“开启自动备份”，点击“退出软件”，软件直接退出。
   - 勾选“开启自动备份”，点击登录并完成登录流程。进入软件后，运行(get-preference "autobackup")，查看输出是否为on。
   - 取消勾选“开启自动备份”，点击登录并完成登录流程。进入软件后，运行(get-preference "autobackup")，查看输出是否为off。

7. 登录弹窗不应阻塞软件启动

8. 主窗口遮罩测试：
   - 主窗口出现后，内容区应被半透明暗色遮罩覆盖。
   - 点击主窗口内容区，不应触发编辑、菜单或其他软件交互。
   - 登录窗关闭后，主窗口遮罩应立即消失。

9. 登录窗定位与置顶测试：
   - 登录窗应显示在软件主窗口区域中央，而不是屏幕中央。
   - 主窗口移动、缩放、最小化后恢复时，登录窗应继续保持相对主窗口居中。
   - 登录窗应保持在主窗口和遮罩之上，不应被软件内容覆盖。
   - 登录窗应仅保持在主窗口和遮罩之上，不应遮挡其他窗口

10. 登录窗拖动联动测试：
   - 拖动登录窗时，应带动主窗口一起移动。
   - 拖动后登录窗不应脱离主窗口中心。
   - Linux 下拖动时，优先使用系统级窗口拖动，避免明显拖影。

11. 异步登录成功关闭测试：
   - 非社区版未登录状态下启动软件，出现引导弹窗后点击登录。
   - 登录流程完成前，引导弹窗应显示“正在等待登录完成...”，提示“登录成功后自动关闭”，且登录、退出软件按钮都可再次点击触发对应操作。
   - 登录成功后，引导弹窗应自动关闭，不需要手动再次关闭窗口。

12. 离线启动测试：
   - 非社区版未登录状态下，在设备无网络连接时启动软件。
   - 启动过程中仍应显示登录引导弹窗，不直接进入主界面。
   - 引导弹窗上方黄色文案和下方说明区域保持不变，加载条下方状态文案改为离线模式相关提示，提示当前未检测到网络连接，会员功能暂不可用。
   - 登录按钮文案改为“进入软件”，点击后应直接关闭引导弹窗并进入软件；“退出软件”按钮行为保持不变。

## 2026/04/21 离线时在登录引导窗内显示离线提示

## What
+ 检测到当前设备无网络连接时，登录引导窗保留上方黄色文案和下方说明区域，仅替换加载条下方状态文案为离线模式提示
+ 离线场景下登录按钮文案改为“进入软件”，点击后直接进入软件；退出软件按钮行为不变

## 2026/04/21 登录成功后自动关闭异步引导弹窗

## What
+ 异步模式下点击登录后，引导弹窗不再立即关闭，而是进入“正在等待登录完成...”状态
+ 等待登录完成期间保留登录和退出软件按钮可用，允许用户再次触发登录流程或直接退出软件
+ 启动阶段增加登录成功状态监听，登录完成后自动通知引导弹窗关闭
+ 若登录状态在监听建立前已经完成，引导弹窗也会在检测到已登录后立即关闭

## 2026/04/20 异步模式下为主窗口增加遮罩并固定登录窗位置

## What
+ 异步模式下，登录窗显示后会为主窗口增加半透明暗色遮罩，阻挡对软件内容区的点击
+ 登录窗在异步模式下保持置顶，并始终相对主窗口居中显示
+ 主窗口移动、缩放或重新显示时，遮罩和登录窗位置会同步更新
+ 拖动登录窗时，优先触发主窗口系统级拖动，使登录窗跟随主窗口一起移动

## 2026/04/20 引导弹窗改为异步启动

## What
+ 引导弹窗不再阻塞启动流程，改为非模态异步显示
+ `StartupLoginDialog` 增加 `asyncStartupMode`，异步模式下等待 `texmacs_started` 后再启用交互
+ 点击登录后不再等待弹窗结束才进入登录流程，而是在主界面启动后触发登录
+ 点击“退出软件”或直接关闭弹窗时，异步模式下直接退出软件
+ 启动早期的通知栏刷新改为延迟重试，避免主窗口未就绪时调用 `update-menus`

## 2026/04/20 修正“退出软件”与自动备份勾选框行为

## What
+ “退出软件”按钮不再沿用旧的“跳过登录”逻辑，点击后直接退出软件
+ “开启自动备份”勾选框不再在点击时立即写入偏好，只有点击登录时才按当前勾选状态保存

## 2026/04/17 强制用户登录

## What
+ 用户未登录的情况下，引导窗口持续显示
+ 用户要么点击登录按钮，要么退出软件

## bug
目前引导窗口结束后软件才启动，因此，引导窗口出现时，scheme侧的登录流程无法使用，所以目前无法真正强制用户登录，需要后续将引导窗口改为异步方可实现。

## 2026/01/04 引导弹窗加入进度条
#### what
1. research.cpp调用show_startup_login_dialog()                                                                         
2. 显示引导弹窗，启动BootstrapTaskExecutor                                                                             
3. 主线程阻塞在QEventLoop中                                                                                            
4. BootstrapTaskExecutor在主线程中分步执行初始化：                                                                     
  - 执行一小段任务 → 更新进度 → 安排下一段执行                                                                        
  - 重复直到初始化完成                                                                                                
5. 初始化完成后启用弹窗按钮                                                                                           
6. 用户点击按钮 → QEventLoop退出 → 主线程继续                                                                          
7. research.cpp调用init_texmacs()（通过g_startup_login_executed防止重复）
