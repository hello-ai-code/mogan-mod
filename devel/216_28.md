# 216_28 优化启动页的设置页顶部的按钮

## 如何测试

1. 编译：`xmake b stem`
2. 打开启动页 `Settings` 页面，确认 Tab 分类按钮样式正确：
   - 按钮有圆角、padding / margin 正常，排列整齐。
   - 未选中文字灰色，选中背景深墨绿（light 主题 `#215a6a` / dark 主题 `#2791ad`）。
   - hover 时背景色变化正常（light `#F0F0F0` / dark `rgba(255,255,255,0.1)`）。
3. 验证 DPI 缩放：
   - 在 HiDPI 屏幕（macOS Retina、Windows 125%/150% 缩放、Linux 2x 缩放）上打开。
   - 确认 Tab 按钮的圆角、padding、margin 按 DPI 比例放大，不会显得过小或拥挤。
   - 对比普通 DPI 屏幕，视觉比例应保持一致。
4. 验证主题切换：
   - 切换 light / dark 主题。
   - 确认颜色（文字色、选中背景、hover 背景）跟随主题变化。
   - 确认尺寸（圆角、padding、margin）不受主题切换影响。
5. 验证跨平台一致性：
   - Windows / macOS / Linux 上 Tab 按钮样式应一致，不依赖平台原生 QTabWidget 样式。
6. 存在一个缺陷：在light主题下，图片颜色暂时不能修改为白色

## 2026/04/23 实现说明

### What

本次对 Settings 页面（`QTSettingsPage`）中 `QTabWidget` 的样式处理进行了重构，核心改动：

- **DpiUtils 动态缩放**：将 `QTabWidget` 样式中的硬编码尺寸改为通过 `DpiUtils::scaled()` 按屏幕 DPI 动态计算。
- **样式分离**：将颜色、背景、边框等视觉属性从 C++ 代码中移除，完全交由 CSS 主题文件控制。
- **CSS 清理**：删除 `liii.css` 和 `liii-night.css` 中已由代码动态管理的尺寸属性，避免重复定义。

#### 修改文件

**src/Plugins/Qt/qt_settings_page.cpp** (修改)
- 引入 `qt_dpi_utils.hpp`。
- 在匿名命名空间中定义常量：
  - `kTabPaneTop = 0`
  - `kTabBorderRadius = 12`
  - `kTabPaddingY = 6`
  - `kTabPaddingX = 14`
  - `kTabMarginY = 4`
  - `kTabMarginX = 2`
- `setStyleSheet()` 中使用 `QString::arg(DpiUtils::scaled(...))` 将上述常量按 DPI 缩放后拼接为样式字符串。
- 清理 C++ 层样式：移除 `border: none`、`background: transparent` 等视觉属性，仅保留必须通过代码动态计算的尺寸属性。

**TeXmacs/misc/themes/liii.css** (修改)
- `QTabWidget#settings-tab-widget::pane`：删除 `top: 0px`。
- `QTabWidget#settings-tab-widget QTabBar::tab`：删除 `border-radius: 12px`、`padding: 6px 14px`、`margin: 4px 2px`。
- 保留 `:hover`、`:selected` 颜色定义不变。

**TeXmacs/misc/themes/liii-night.css** (修改)
- 与 `liii.css` 做同样的尺寸属性清理。

### Why

1. **HiDPI 适配**：不同屏幕 DPI 差异大，硬编码像素值在 HiDPI 屏幕上会导致 Tab 按钮显得过小、拥挤。通过 `DpiUtils::scaled()` 统一处理，保证跨屏幕视觉一致性。
2. **职责分离**：
   - C++ 代码负责结构性尺寸（必须按 DPI 计算）。
   - CSS 主题文件负责视觉样式（颜色、背景等）。
   -两者各司其职，修改时互不干扰。
3. **避免重复定义**：`border: none`、`background: transparent` 等属性此前同时存在于 C++ 代码和 CSS 中，清理后只保留一份，降低维护不一致的风险。
4. **参考已有模式**：`qt_startup_tab_widget.cpp` 中导航按钮的 `padding` / `border-radius` 已通过 `DpiUtils::scaled()` 管理，Settings 页面的 Tab 按钮应遵循同一套 DPI 处理规范。

### How

1. **常量定义**：在 `qt_settings_page.cpp` 匿名命名空间中定义 6 个尺寸常量，对应 96 DPI 标准下的基准像素值。
2. **动态样式拼接**：使用 `QString` 多参数 `arg()` 将 `DpiUtils::scaled()` 计算后的整数值插入样式模板字符串：
   ```cpp
   tabWidget->setStyleSheet(
       QString("... top: %1px; ... border-radius: %2px; ...")
           .arg(DpiUtils::scaled(kTabPaneTop))
           .arg(DpiUtils::scaled(kTabBorderRadius))
           ...);
   ```
3. **CSS 层精简**：在 `liii.css` 和 `liii-night.css` 中，仅保留颜色、背景、边框状态等主题相关属性，删除所有具体像素值。这样主题文件只表达"视觉风格"，不表达"物理尺寸"。
4. **跨主题兼容**：由于尺寸已提取到 C++ 代码中，新增主题时只需复制颜色定义，无需再复制一套尺寸值。
