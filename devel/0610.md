# [0610] 修复 HTML 导出和导入时嵌套有序列表格式丢失与不匹配的问题

当 Mogan 导出包含 `<enumerate-numeric-paren>`（带有括号的数字列表）、`<enumerate-circle>`（带圈数字列表）或 `<enumerate-hanzi>`（汉字列表）等有序列表环境的文档为 HTML时，这些特殊的列表结构会被提前展开或丢失，导致生成的 HTML 不包含正确的嵌套 `<ol>` 结构，且项目样式和编号丢失。同样地，在导入包含嵌套 `<ol>` 的 HTML 文档时，应当能正确、优雅地被还原为 TeXmacs 的嵌套列表格式。

## 5 What
1. 修复了 HTML 导出时，`enumerate-numeric-paren`、`enumerate-circle`、`enumerate-hanzi` 宏因未被注册为 "identity" 宏而在预处理阶段提前展开，导致嵌套列表层级扁平化、编号丢失、样式混乱的系统缺陷。
2. 重构了 HTML 导出时自定义有序列表标号的 CSS 渲染机制。弃用存在换行遮挡漏洞的 `list-style-type: none` 搭配 `::before` 负 margin 方案，改用现代且 100% 兼容的 `::marker` 方案，实现了在列表项包含块级元素（如 `<p>`）时标号依然可以完美居中对齐、不错位且不隐藏的完美排版。
3. 修复了 HTML 导入列表解析器硬编码的问题。重构为 `htmltm-ol`、`htmltm-ul`、`htmltm-dl` 动态属性感知函数，使得在导入 HTML 时，能自动识别并无损还原括弧、带圈、汉字等全部新增的列表结构到对应的 Mogan 自定义列表中。
4. 随本 PR 提交了包含 128 项自动化 assertions 的 TDD Scheme 单元与集成测试用例：`0610.scm`。
5. 随本 PR 提交了用于手工/自动化测试的标准 TMU 测试样例（`0610.tmu`）和可供双向测试导入的网页样本（`0610_import_test.html`，存放于 `TeXmacs/tests/html/` 目录下）。

## 6 Why
* **提前展开与映射缺失原因**：由于历史遗漏，这三个有序列表宏未在 `tmhtml-expand.scm` 中作为 identity 宏进行豁免，导致被提前拆解为低级排版标记；同时 `tmhtml-stdmarkup%` 也缺少对应规则。
* **标号丢失原因**：原 `::before` 伪元素定位方案容易因为 `<li>` 内部包裹的 `<p>` 等块级换行标签而导致错位或被推到可视范围之外；同时 `list-style-type: none` 会导致部分现代浏览器直接关闭 `::marker` 标号槽进而使标号完全隐藏。而采用 `::marker` 改变 counter 字符可以无任何排版影响地原地渲染出 `(1)` 符号。
* **导入扁平化原因**：原导入器将所有的 `<ol>`、`<ul>`、`<dl>` 均硬编码处理为唯一的默认环境，造成样式在双向循环解析时丢失。

## 7 How
1. 将 `enumerate-numeric-paren`, `enumerate-circle`, `enumerate-hanzi` 补充注册到 `tmhtml-expand.scm` 的 `tmhtml-env-patch` 中。
2. 在 `tmhtml.scm` 的 `tmhtml-stdmarkup%` 的 `enumerate` 逻辑下添加这三者，并在其导出渲染时，通过在 `tmhtml-dispatch` 中添加 `tmhtml-current-tag` 状态机制，动态为其绑定正确的 `class` 样式。
3. 在 `tmhtml.scm` 的 `tmhtml-css-header` 中，利用 `::marker { content: ... }` 机制精准复现全部 14 种 ordered lists 及其自定义标号后缀。
4. 在 `htmltm.scm` 中新增 `htmltm-ol`、`htmltm-ul`、`htmltm-dl`，在导入时动态检索 class 并将其映射还原为对应的符号或自定义列表环境。
