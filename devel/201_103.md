# 201_103 添加统一 DPI 工具类 DpiUtils

## 如何测试

### 测试环境准备
准备以下至少一种环境：
- Windows 电脑，显示设置缩放为 125%、150% 或 200%
- macOS Retina 屏幕
- Linux

### 需要检查的具体位置

#### 1. 菜单栏高度 (`qt_tm_widget.cpp`)
- **检查方法**: 目测菜单栏是否被截断或过高

#### 2. 标签页栏按钮大小 (`qt_tm_widget.cpp`)
- **位置**: 主窗口标题栏的最小化/最大化/关闭按钮
- **检查点**: 按钮大小是否随 DPI 缩放（基准 46×32 像素，缩放后应增大）

#### 3. 登录对话框登出按钮 (`qt_tm_widget.cpp`)
- **位置**: 帮助菜单中的登录对话框
- **检查点**: 
  - macOS: 图标大小应为 30×30 像素（乘以缩放比例）
  - Windows/Linux: 图标大小应为 20×20 像素（乘以缩放比例）

#### 4. Retina 检测 (`QTMApplication.hpp`)
- **位置**: 应用启动时的 HiDPI 模式检测
- **检查方法**: 
  - 在 macOS Retina 屏幕上，启动时日志应显示 "Setting up HiDPI mode"
  - 界面元素应自动启用 Retina 渲染

#### 5. 标签页系统栏高度 (`QTMTabPage.cpp`)
- **位置**: 标签页顶部的系统按钮栏（macOS）
- **检查点**: 
  - 标签页部分是否居中显示
  - 标签页系统栏高度是否合适

#### 6. 图像悬浮菜单 (`QTMImagePopup.cpp`)
- **位置**: 右键点击图片后弹出的工具栏
- **检查点**: 
  - 整体比例协调，按钮不溢出边界

#### 7. 文本悬浮菜单 (`QTMTextToolbar.cpp`)
- **位置**: 选中文字后弹出的悬浮工具栏
- **检查点**:
  - 工具栏整体比例协调，按钮不溢出边界

### 回归测试
在标准 DPI（96 DPI，即 100% 缩放）下检查：
- 所有 UI 元素大小应与修改前保持一致
- 无缩小或放大异常

## 2026/04/02 添加统一 DPI 工具类 DpiUtils

### What

- 新增 `DpiUtils` 工具类（`qt_dpi_utils.hpp/cpp`），统一处理跨平台 DPI 缩放计算。
- 在 `qt_utilities.hpp` 中引入 `qt_dpi_utils.hpp`，使 `DpiUtils` 全局可用。
- 替换以下文件中手动实现的 DPI 检测代码：
  - `qt_tm_widget.cpp`：菜单栏高度、登录对话框按钮大小
  - `QTMApplication.hpp`：Retina 检测
  - `QTMTabPage.cpp`：标签页系统栏高度
  - `QTMImagePopup.cpp`：图像弹出框按钮大小
  - `QTMTextToolbar.cpp`：文本工具栏按钮大小

### Why

- 不同平台计算缩放的方式不同：
  - **Windows/Linux**：使用 `logicalDotsPerInch() / 96` 计算缩放比例
  - **macOS**：使用 `logicalDotsPerInch() / 72.0` 计算缩放比例（macOS 传统基准 DPI）
- 原代码中手动实现 DPI 检测的位置分散，且存在平台兼容性问题（macOS 上 `logicalDotsPerInch()` 始终返回 72，导致计算错误）。
- 统一封装后，调用方无需关心平台差异，代码更简洁可维护。

### How

- `DpiUtils::scaleFactor()`：根据平台自动选择正确的缩放计算方法
  - Windows：`screen->logicalDotsPerInch() / 96.0`
  - macOS：`logicalDotsPerInch() / 72.0`，Windows/Linux：`logicalDotsPerInch() / 96.0`
- `DpiUtils::scaled(int baseSize)`：将基准大小按当前屏幕 DPI 缩放并四舍五入
- `DpiUtils::toPhysicalRect/Point/Size()`：逻辑坐标 → 物理像素坐标转换
- `DpiUtils::toLogicalRect/Point/Size()`：物理像素坐标 → 逻辑坐标转换
- 所有使用位置统一替换为 `DpiUtils::scaleFactor()` 或 `DpiUtils::scaled()`，移除分散的 `#ifdef Q_OS_WIN` DPI 计算逻辑。

### 使用示例

```cpp
// 获取主屏幕缩放比例
qreal scale = DpiUtils::scaleFactor();

// 缩放一个基准大小为 40 的按钮（例如文本工具栏图标）
int buttonSize = DpiUtils::scaled(40);

// 坐标转换：逻辑坐标 → 物理像素
QRect logicalRect(10, 10, 100, 100);
QRect physicalRect = DpiUtils::toPhysicalRect(logicalRect);

// 坐标转换：物理像素 → 逻辑坐标
QRect logicalAgain = DpiUtils::toLogicalRect(physicalRect);
```

### 注意事项

- **像素对齐**：`DpiUtils::scaled()` 和坐标转换方法使用 `qRound` 进行四舍五入，确保缩放后的像素值为整数，避免亚像素渲染问题。
- **macOS 基准 DPI**：macOS 传统上使用 72 DPI 作为基准（而非 Windows/Linux 的 96 DPI），因此缩放计算采用 `logicalDotsPerInch() / 72.0`。
- **多屏幕支持**：所有方法都支持传入 `QScreen*` 参数以指定目标屏幕；传入 `nullptr` 时默认使用主屏幕。
- **向后兼容**：修改后的代码在标准 DPI（96 DPI，100% 缩放）下应保持与之前完全相同的外观尺寸。
