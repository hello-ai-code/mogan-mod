# [221_10] 为有序列表、无序列表和描述列表新增结构化向上/向下插入与删除

> 相关文档：[201_31](201_31.md)、[x_y.md](x_y.md)（模板）

## 如何测试

### 一、单元测试

运行集成测试：`xmake r 221_10`

### 快捷键说明

在列表上下文中：

- **非 macOS**：
  - `Alt + Up`：结构化向上插入
  - `Alt + Down`：结构化向下插入
  - `Alt + Backspace`：结构化向上删除（在列表上下文重映射，非列表中为向左删除）
  - `Alt + Delete`：结构化向下删除（在列表上下文重映射，非列表中为向右删除）
- **macOS**：
  - `Option + Up`：结构化向上插入
  - `Option + Down`：结构化向下插入
  - `Option + Backspace`：结构化向上删除（在列表上下文重映射，非列表中为向左删除）
  - `Option + Delete`：结构化向下删除（在列表上下文重映射，非列表中为向右删除）

### 二、无序列表（itemize）

1. 启动 Mogan，打开或新建一个正文文档。
2. 通过菜单 **Text -> Itemize -> Default** 插入无序列表，输入至少两个条目，例如：
   - 第一条
   - 第二条
3. 将光标放在**第二个条目内部**，按 **Alt + Up**（macOS 为 **Option + Up**），确认在当前条目**上方**插入一个新的空条目，且光标进入新条目内容区。
4. 将光标放在**第一个条目内部**，按 **Alt + Down**（macOS 为 **Option + Down**），确认在当前条目**下方**插入一个新的空条目，且光标进入新条目内容区。
5. 将光标放在列表条目内部，查看焦点工具栏，确认出现 **Insert above**、**Insert below**、**Remove upwards**、**Remove downwards** 四个结构化按钮，且**不出现**左右方向的插入/删除按钮。
6. 按 **Alt + Backspace**（macOS 为 **Option + Backspace**），确认删除当前 item 上方的同级逻辑 item。
7. 按 **Alt + Delete**（macOS 为 **Option + Delete**），确认删除当前 item 下方的同级逻辑 item。
8. 构造嵌套列表：让当前条目后面带有一个同类型子列表。
9. 在该条目内部执行结构化插入向下，确认新条目插入在嵌套子列表之后，仍处于外层同级列表，而不是插入到子列表内部。

### 三、有序列表（enumerate）

1. 通过菜单 **Text -> Enumerate -> Default** 插入有序列表，输入至少两个条目。
2. 重复无序列表测试中的步骤 3-9。
3. 确认插入新条目后，编号自动重新排列。
4. **结构化向下删除至空文档**：新建一个空白文档，仅插入一个有序列表并输入一个条目。将光标放在该条目内，按 **Alt + Delete** 删除唯一条目。确认：
   - Mogan **不崩溃**（不会触发 "Throwing empty composite box" 错误）。
   - 列表被自动移除，文档变为一个包含空字符串的普通段落，光标位于该段落内。
5. **结构化向下删除后仍有内容**：新建一个文档，先输入一段普通文字，再插入一个有序列表并输入一个条目。将光标放在列表条目内，按 **Alt + Delete** 删除唯一条目。确认：
   - 列表节点被移除，但前面的普通文字保留。
   - 光标位于列表原来的位置（文字之后）。

### 四、描述列表（description）

1. 通过菜单 **Text -> Description -> Default** 插入描述列表，输入至少两个条目，每个条目包含标签和描述内容。
2. 将光标放在**第二个条目内部**，按 **Alt + Up**（macOS 为 **Option + Up**），确认在当前条目**上方**插入一个新的空描述条目（`item* ""`），且光标位于 `item*` 子节点 `""` 的开始位置（`tree-go-to new-item 0 :start`）。
3. 将光标放在**第一个条目内部**，按 **Alt + Down**（macOS 为 **Option + Down**），确认在当前条目**下方**插入一个新的空描述条目（`item* ""`），且光标位于 `item*` 子节点 `""` 的开始位置（`tree-go-to new-item 0 :start`）。
4. 将光标放在描述列表条目内部，查看焦点工具栏，确认出现 **Insert above**、**Insert below**、**Remove upwards**、**Remove downwards** 四个结构化按钮，且**不出现**左右方向的插入/删除按钮。
5. 按 **Alt + Backspace**（macOS 为 **Option + Backspace**），确认删除当前 item 上方的同级逻辑 item（包括标签和描述内容）。
6. 按 **Alt + Delete**（macOS 为 **Option + Delete**），确认删除当前 item 下方的同级逻辑 item（包括标签和描述内容）。
7. 确认在描述列表中插入的新条目结构为 `item* ""`（而非普通列表的 `item`，也非无子节点的 `item*`）。

## 2026/05/08 修复描述列表插入结构与光标定位，修复空文档保护

### What

1. **描述列表插入结构修复**：`blank-list-item-stree` 对 `description` 类型返回 `(item* "")` 而非 `(item*)`，确保新插入的描述条目包含一个空的标签/描述内容子节点。
2. **描述列表光标定位修复**：`structured-insert-vertical` 在插入 `item*` 后，通过 `(tree-go-to new-item 0 :end)` 将光标放入 `item*` 的子节点内部，而不是停留在 `item*` 节点上。
3. **空文档保护修复**：`structured-remove-vertical` 在删除列表节点后，若父 `document` 的 arity 变为 0，则自动插入一个空字符串 `""` 作为占位，避免产生空的 document 触发 `TM_FAILED ("empty composite box")`。

### Why

1. 描述列表的 `item*` 必须包含一个子节点作为标签/描述内容，插入 `(item*)` 会导致结构不完整，后续编辑时行为异常。
2. 光标停留在 `item*` 节点上时，用户直接输入文字不会进入标签/描述内容区，需要额外的 Enter 或点击操作才能开始编辑，体验不符合预期。
3. 当整个文档仅包含一个有序/无序/描述列表时，结构化删除所有条目会导致列表节点被移除，根 `document` 变为空。空的 `document` 在排版阶段会创建没有任何子盒子的 composite box，触发 `composite_boxes.cpp:67` 的 `TM_FAILED ("empty composite box")`，导致 Mogan 崩溃。

### How

1. `blank-list-item-stree` 中 `description` 分支返回 `` `(item* "") ``。
2. `structured-insert-vertical` 的光标定位逻辑增加对 `item*` 的判断：
   ```scheme
   (if (tree-is? new-item 'item*)
       (tree-go-to new-item 0 :end)
       (tree-go-to new-list insert-pos :end))
   ```
3. `structured-remove-vertical` 在删除列表节点后增加空文档保护：
   ```scheme
   (when (and parent-doc list-index)
     (tree-remove parent-doc list-index 1)
     (if (== (tree-arity parent-doc) 0)
         (begin
           (tree-insert parent-doc 0 (list ""))
           (tree-go-to parent-doc 0 :end))
         (tree-go-to parent-doc list-index :end)))
   ```
4. 单元测试 `221_10.scm` 新增 `blank-list-item-stree` 的 `description` 类型断言、以及 `only-list-doc` 场景的结构化删除断言。

## 2026/05/07 新增列表结构化向上/向下插入与删除

### What

为 `itemize`、`enumerate` 和 `description` 系列列表补上结构化向上/向下插入和删除行为。

修改点：

- 在 `TeXmacs/progs/generic/generic-edit.scm` 中新增列表专用的 `structured-insert-vertical` 和 `structured-remove-vertical` 分派。
- 扩展 `list-structured-insert-context?`，让有序/无序/描述列表项上下文都能显示结构化向上/向下插入和删除菜单。
- 扩展 `structured-vertical?`，上述三种列表上下文均进入 vertical 分支。
- 调整 `focus-can-insert-remove?`，对原子节点 `item`/`item*` 放宽 `cursor-inside?` 限制，确保焦点工具栏按钮正常显示。
- 调整 `blank-list-item-stree`，根据列表类型返回 `(item)` 或 `(item*)`。
- 调整 `focus-insert-menu` 和 `focus-insert-icons`，列表上下文隐藏左右方向的插入/删除菜单项。
- 新增 `list-item-end-index`，用于计算向下插入时的同级插入位置。
- 新增 `list-item-insert-index`，统一封装向上/向下的插入位置计算。
- 新增 `list-item-remove-range` 和 `remove-list-item-range`，统一封装向上/向下删除的逻辑 item 范围。
- 新增 `current-list-item-marker-from-nearest-left`，支持光标位于 item marker 后续同级正文节点时，向左找到所属逻辑 item。
- 新增 `TeXmacs/tests/221_10.scm`，覆盖裸 `(item)` 与 `(concat (item) "...")` 混合结构、item 后续同级正文、嵌套列表跳过逻辑、向上/向下插入与删除位置计算和空列表项结构。

### Why

已有结构化插入主要服务于表格、数学结构和一般结构化节点。列表项虽然也属于结构化内容，但在有序/无序/描述列表中执行结构化向上或向下插入时，没有列表语义下的"插入同级条目"行为。

用户在编辑列表时，结构化向上/向下插入的合理预期是：

- 向上插入：在当前同级条目前插入一个空条目。
- 向下插入：在当前同级条目后插入一个空条目。
- 向上删除：删除当前条目上方的同级逻辑条目。
- 向下删除：删除当前条目下方的同级逻辑条目。
- 如果当前条目后面挂接了同类型嵌套子列表，向下插入应跳过该子列表，把新条目放回外层同级位置，避免误插到嵌套列表内部或打断当前条目的子结构。
- 列表没有左右方向的同级结构，因此菜单和焦点工具栏中不应该暴露左右插入/删除按钮。
- 列表上下文中不存在结构化向左/向右删除，因此 `Alt + Backspace`（macOS 为 `Option + Backspace`）和 `Alt + Delete`（macOS 为 `Option + Delete`）这两个原本对应"向左删除"和"向右删除"的快捷键，在列表上下文中被重映射为"向上删除"和"向下删除"。

### How

实现复用现有列表缩进逻辑中的结构识别能力：

- 使用 `current-list-item-marker` 找到当前逻辑列表项。
- 当光标不在 marker 或 marker wrapper 内部时，`current-list-item-marker` 会在同一 `document` 中向左查找最近的 item marker；这样 `(concat (item) "ddd")` 后面的同级字符串 `"ddd"` 仍属于该 item。
- 使用 `find-item-wrapper-and-list` 取得列表项外层 `concat` wrapper 和所在的 `document`。
- 使用 `list-family` / `same-list-family?` 判断 `itemize`、`itemize-dot`、`enumerate-numeric`、`description` 等变体是否属于同一类列表。
- 向上插入时直接使用当前 item 的同级索引。
- 向下插入时调用 `list-item-end-index`，跳过当前 item 后紧跟的同类子列表和正文续段，直到下一个 item marker 前再插入新的空 item。
- 向上删除时删除前一个 item marker 到当前 item marker 之前的逻辑范围。
- 向下删除时从当前 item 逻辑范围末尾开始，删除下一个 item marker 到其逻辑范围末尾。
- 插入后通过 `tree-go-to` 将光标移动到新条目的正文内容位置。
- `structured-vertical?` 在 `list-structured-insert-context?` 成立时返回真，使菜单栏和焦点菜单能够暴露 **Insert above** / **Insert below** / **Remove upwards** / **Remove downwards**。
- `focus-insert-menu` 和 `focus-insert-icons` 在列表上下文中只渲染上下方向的菜单项，避免显示不可用的左右插入/删除按钮。
