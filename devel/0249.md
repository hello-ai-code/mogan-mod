## 5 What
修复聊天会话 output 区域中 CJK 字符在自动换行时被截断的问题。

1. 在 `unfolded-io-text` 宏中为 output 参数添加 `<text|>` 包装
2. 在 `folded-io-text` 宏中为 output 参数添加 `<text|>` 包装

## 6 Why
聊天会话的 `unfolded-io-text` 宏只为 input 包裹了 `<text|>`，output 没有。
导致 output 区域渲染在 `program` 模式而非 `text` 模式下，CJK 字符在自动换行时被不当截断。

具体表现：
- 环境栏在 input 区域显示 `text sys-chinese 10`，在 output 区域显示 `program roman 10`
- 长中文段落中的汉字（如"力"）被物理切割

## 7 How
修改 `session.ts` 中 `unfolded-io-text` 和 `folded-io-text` 两个宏定义，
为 output 参数添加 `<text|>` 包装，与 input 的处理方式保持一致。

**修改前**:
```
<assign|unfolded-io-text|<\macro|prompt|in|out>
  <unfolded-io|<arg|prompt>|<text|<arg|in>>|<arg|out>>
</macro>>
```

**修改后**:
```
<assign|unfolded-io-text|<\macro|prompt|in|out>
  <unfolded-io|<arg|prompt>|<text|<arg|in>>|<text|<arg|out>>>
</macro>>
```
