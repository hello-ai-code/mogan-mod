## 5 What

1. `ChatConversationPanel` 构造函数新增 `msgBufUrl` 和 `inBufUrl` 参数
2. 新增 `msgBufferUrl_` 和 `inputBufferUrl_` 成员缓存 URL
3. `setup_ui` 使用缓存成员替代 `ChatSessionManager::messageBufferUrl()` / `inputBufferUrl()`
4. `readInputMessage` 使用缓存成员替代 `ChatSessionManager::inputBufferUrl()`
5. `QTChatTabWidget::createPanel` 中构造 URL 并传入 Panel

## 6 Why

Panel（View）直接调用 `ChatSessionManager::messageBufferUrl()` / `inputBufferUrl()` 静态方法构造 URL，感知了 Model 层的 URL 命名规则。Panel 作为 View 不应知道 URL 如何构造，URL 属于 Model 层细节。外部注入 URL 使 Panel 与 `ChatSessionManager` 实现解耦，便于测试时传入 mock URL。

## 7 How

### 构造函数注入
构造函数接收 `msgBufUrl` 和 `inBufUrl` 两个 `url` 参数，初始化到 `msgBufferUrl_` / `inputBufferUrl_` 成员。`setup_ui` 创建 TeXmacs widget 时使用缓存成员。`readInputMessage` 读取输入 buffer 时也使用缓存成员。

### 工厂方法负责 URL 构造
`QTChatTabWidget::createPanel` 作为 Panel 的工厂方法，在创建 Panel 前调用 `ChatSessionManager::messageBufferUrl()` / `inputBufferUrl()` 构造 URL，然后传入构造函数。URL 构造逻辑从 Panel 内部上移到工厂方法。
