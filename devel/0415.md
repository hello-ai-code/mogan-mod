## 3 如何测试

### 3.1 验证远程版本获取
1. 启动 Mogan STEM，点击 **帮助 -> 版本**。
2. 观察弹出的对话框，确认版本号能够正确展示：
   - 社区版应展示两个版本号（社区版和商业版）
   - 商业版应展示一个版本号（商业版）
3. 版本号应为具体的版本字符串（如 `2026.2.6`），而非空字符串或 URL 路径。

### 3.2 模拟版本不一致测试
在 `TeXmacs/progs/doc/help-funcs.scm` 的 `(mogan-version)` 中，临时修改 `cur-ver`：
```scheme
(let* ((cur-ver "2026.2.4")
```
点击 **帮助 -> 版本**，确认：
- 弹出的对话框中，远程版本号不为空
- 点击确认可以正常跳转官网

### 3.3 网络异常测试
临时修改 URL 为不存在的地址，确认失败时返回空字符串，不会导致崩溃。

### 3.4 验证消息框换行显示
1. 将界面语言切换为中文
2. 点击 **帮助 -> 版本**
3. 确认消息框中多行文本正确换行显示：
   - 社区版：
     ```
     您正在使用 v2026.2.4。
     Mogan STEM 的最新稳定版是 v2026.2.6，Liii STEM 的最新稳定版是 v2026.2.6。
     请点击确认前往官网下载最新稳定版。
     ```
   - 商业版：
     ```
     您正在使用 v2026.2.4，Liii STEM 的最新稳定版是 v2026.2.6。
     请点击确认前往官网下载最新稳定版。
     ```

## 5 What
修复了帮助菜单"版本"功能无法正确获取远程版本号的问题。原先使用 `system-url->string` 和 `eval-system` 调用外部命令（`curl`/`wget`）获取远程内容，但在 Mogan 运行环境中返回空字符串，导致版本号无法展示。现改用 Mogan 内置的 `string-load` + `string->url` 机制，利用框架自动下载远程 `.tm` 文件并读取内容，稳定可靠。

同时优化了版本检查消息框的显示效果：
1. 消息文本按句子换行，提升可读性
2. `message-widget` 支持 `\n` 多行显示
3. 中英文翻译同步更新，按语义自然分段

## 6 Why
1. **`system-url->string` 不适用于 HTTP**：该函数底层对应 `sys_concretize`，仅做本地路径转换，无法发起 HTTP 请求获取远程内容。
2. **`eval-system` 调用 `curl` 在运行时失效**：虽然系统终端中可以正常执行 `curl`，但 Mogan 运行时的 `eval-system` 返回空字符串，可能与沙箱环境或路径问题有关。
3. **复用框架内置能力**：Mogan 本身已有完善的 web 文件处理机制（`get_from_web` → `lolly::io::download`），通过 `string-load` + `string->url` 可以自动下载远程文件到本地缓存，然后读取内容，无需依赖外部命令。
4. **统一跨平台行为**：使用框架内置机制可以避免不同平台（Windows/macOS/Linux）上 `curl`/`wget` 可用性差异的问题。
5. **消息框文本过长**：原版本检查消息为单行长文本，在对话框中显示拥挤，按句子换行后更清晰易读。

## 7 How
1. **新增版本解析函数**：在 `TeXmacs/progs/doc/help-funcs.scm` 中新增 `extract-version-from-tm-content`，参考 C++ 侧的 `parseVersionFromTM` 实现，使用 `string-search-forwards` 定位 `<\body>` 和 `</body>` 标签，提取中间内容并 `tm-string-trim-both`：
   ```scheme
   (define (extract-version-from-tm-content content)
     (if (not (string? content))
         ""
         (let* ((body-start (string-search-forwards "<\body>" 0 content))
                (body-end (string-search-forwards "</body>" 0 content)))
            (if (and (!= body-start -1) (!= body-end -1) (> body-end body-start))
                (tm-string-trim-both 
                 (substring content (+ body-start 7) body-end))
               ""))))
   ```
2. **重写 `fetch-latest-version` 函数**：从 `eval-system` + `curl`/`wget` 改为 `string-load` + `string->url`：
   ```scheme
   ;; 修改前
   (define (fetch-latest-version url)
     (with content (eval-system (string-append "curl -sL \"" url "\""))
       (if (and (string? content) (!= content ""))
           (extract-version-from-tm-content content)
           (with content2 (eval-system (string-append "wget -qO- \"" url "\""))
             ...))))
   
   ;; 修改后
   (define (fetch-latest-version url)
     (with content (string-load (string->url url))
       (if (and (string? content) (!= content ""))
           (extract-version-from-tm-content content)
           "")))
   ```
3. **调用链路说明**：
   - `string-load` → `url-concretize` → `concretize_url` → `get_from_web` → `lolly::io::download`
   - 远程 `.tm` 文件被自动下载到本地临时目录（如 `/tmp/jiadong/.lolly/{pid}/...`）
   - `string-load` 读取本地文件内容返回字符串
   - 解析函数从 TeXmacs 格式的 `<\body>` 标签中提取版本号
4. **消息框换行支持**：修改 `TeXmacs/progs/kernel/gui/menu-widget.scm` 中的 `message-widget`，使用 `string-decompose` 按 `\n` 拆分文本，每行生成独立的 `(text x)` 组件，用 `vlist` 垂直排列：
   ```scheme
   (tm-widget ((message-widget msg) done)
     (padded (centered (vlist (for (x (string-decompose msg "\n")) (text x))))
             ===
             (centered (explicit-buttons ("Ok" (done)))))
   ) ;tm-widget
   ```
5. **消息文本按句子分段**：在 `help-funcs.scm` 中，版本检查消息按句子加入 `\n`：
   - 社区版：3 行（`You are using...` / `The latest stable...` / `Please click OK...`）
   - 商业版：2 行（`You are using...` / `Please click OK...`）
6. **翻译同步更新**：在 `zh_CN.scm` 和 `zh_TW.scm` 中，对应翻译条目加入 `\n`，按中文语义自然分段，与英文断句保持一致。
