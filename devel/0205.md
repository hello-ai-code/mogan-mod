## What

修复 Chat Tab 会话持久化的三个 bug：

1. **archived 字段始终为 true**：C++ 通过 `call()` 传递字符串 `"false"` 给 Scheme，但 Scheme 的 `(if archived ...)` 将非空字符串视为真值，导致所有会话的 `archived` 都被写为 `"true"`。
2. **重启后 sidebar 不加载历史会话**：
   - `QTChatTabWidget` 构造函数已调用 `loadSessions()`，但 Scheme→C++ 的 `as_tree()` 无法将嵌套列表转为 tree，导致数据传递失败。
   - 改为 Scheme 遍历 sessions 并逐个调用 C++ 回调，绕过 tree 转换问题。
3. **归档/恢复状态未持久化到磁盘**：归档或恢复会话后，只更新了内存中的 `session.archived` 标志，没有调用 `saveOneSession()` 写入 manifest。导致重启后已归档会话仍出现在活跃列表中。

## Why

### archived 始终为 true
C++ `saveOneSession()` 将 `archived` 以字符串 `"true"/"false"` 传给 Scheme。Scheme 的 `chat-persist-make-entry` 中：
```scheme
(njson-set! entry "archived" (if archived "true" "false"))
```
在 Scheme 中，只有 `#f` 是假值，字符串 `"false"` 是真值，所以 `(if "false" "true" "false")` 返回 `"true"`。

### sidebar 不加载历史会话
`loadSessions()` 通过 `as_tree(call("chat-persist-load-all"))` 获取 Scheme 返回的列表。但 `as_tree()` 无法将 Scheme 嵌套列表转为 TeXmacs tree，返回空原子 tree。

解决方案：Scheme 端遍历 sessions 时直接调用 `qt-chat-tab-restore-session` C++ 回调，每个参数都是简单字符串，完全避开 tree 转换。

## How

### 1. 修复 archived 真值判断

**文件**：`chat-session-persist.scm`

```scheme
;; 修复前（buggy）：
(njson-set! entry "archived" (if archived "true" "false"))

;; 修复后：
(njson-set! entry "archived" (if (or (not archived) (== archived "false")) "false" "true"))
```

同时兼容 C++ 传入的字符串 `"false"/"true"` 和 Scheme 传入的布尔值 `#f`/`#t`。

### 2. Scheme→C++ 逐会话回调

**新增 C++ 函数** `qt_chat_tab_restore_session(sid, title, model, archived)`：
- 四个 string 参数，通过 `findChild` 找到 `QTChatTabWidget`
- 调用 `restore_conversation()` 创建 panel，调用 `addConversation()` 加入列表

**改造 `chat-persist-load-all`**：
- 不再返回列表，改为遍历 sessions 时直接调用 `(qt-chat-tab-restore-session ...)`
- 仍然负责加载消息内容到 buffer

**简化 `loadSessions()`**：
- 只调用 `call("chat-persist-load-all")`，不再解析 tree
- Scheme 端回调完成后，执行激活和模型恢复逻辑

### 3. 归档/恢复后持久化状态

**问题**：用户归档或恢复会话后，内存中 `session.archived` 已变更，但 manifest.json 中仍保存旧值。

**修复**：在三个归档/恢复入口调用 `saveOneSession()` 立即写入磁盘：

- **单个归档/恢复**：右键菜单 → 归档/恢复 操作后添加 `saveOneSession(panel->sessionId)`
- **批量归档（顶部按钮）**：`batchArchiveBtn_` 点击回调中，循环归档后逐个调用 `saveOneSession()`
- **批量归档（右键菜单）**：多选模式的右键菜单批量归档操作后同样逐个调用

### 数据流

```
构造函数 QTChatTabWidget()
  → loadSessions()
    → call("chat-persist-load-all")
      → Scheme 遍历 manifest sessions
        → buffer-load + buffer-set-body（恢复消息内容）
        → qt-chat-tab-restore-session(sid, title, model, archived)
          → restore_conversation()（创建 panel + 插入 session 元数据）
          → addConversation()（加入 conversations_ 列表）
    → activate_conversation()（激活首个非归档会话）
    → call("chat-tab-session-select-model")（恢复全局模型）
```

### 持久化文件结构

```
~/Library/Application Support/liiilabs/system/ai-chat-sessions/
  manifest.json                                    # 会话元数据
  <sessionId>/
    message.tm                                     # 会话消息内容（tmu 格式）
```

manifest.json 格式：
```json
{
  "version": 1,
  "sessions": [
    {
      "sessionId": "UUID",
      "title": "会话标题",
      "model": "模型名称",
      "archived": "true"
    }
  ]
}
```
