## 3 如何测试

### 3.1 手动更改版本（模拟测试环境）
在 `TeXmacs/progs/doc/help-funcs.scm` 的 `(mogan-version)` 中，修改 `cur-ver` 的绑定即可模拟不同版本状态：
- **版本一致测试**：保持 `cur-ver` 为 `(xmacs-version)` 或手动写死为 `"2026.2.5"`（与 `latest-ver` 相同）。
- **版本不一致测试**：将 `cur-ver` 临时改写为较低版本，如 `"2026.2.4"`：
  ```scheme
  (let* ((cur-ver "2026.2.4")
  ```

### 3.2 验证步骤
1. **版本一致**：点击 “帮助 -> 版本”，确认仅弹出普通信息对话框（只有确定按钮），无官网跳转链接与询问。
2. **版本不一致**：点击 “帮助 -> 版本”，确认弹出询问对话框（带是/否选项），点击 “是” 可正常跳转到官网 `https://liiistem.cn`。
3. **语言验证**：分别在简体中文、繁体中文和英文语言下运行测试，确保文案、标题和选项翻译完全正常。

## 5 What
在“帮助”菜单下增加了一个“版本”选项，点击后根据当前本地版本与最新稳定版本的对比，按需显示不同的提示信息与逻辑：
- 如果版本一致，仅通过普通信息对话框进行展示。
- 如果版本不一致，通过询问对话框引导用户，支持一键点击直接跳转至官网 `https://liiistem.cn` 下载最新稳定版。
- 该菜单项命名为 "Version" (翻译为 "版本")，没有多余的省略号。

## 6 Why
1. **引导用户升级**：当存在最新稳定版且用户当前使用的版本与之不一致时，跳转按钮非常重要，可以协助用户最快速度地去官网下载最新版本。
2. **优化用户体验，避免过度打扰**：如果用户已经在使用最新稳定版，则不需要展示官网跳转按钮或多余的询问选项，用最简单的 OK 信息框提示即可。
3. **去除省略号**：在 Mogan (TeXmacs) 菜单配置中，如果菜单项动作被配置为带有交互修饰符（如声明为交互式命令等），菜单系统会自动在项后面拼接 "..."。通过在菜单配置中直接绑定 `("Version" (mogan-version))`，无任何交互声明，排除了非必要的省略号。
4. **完整的本地化支持**：为了让不同语境的用户都能获得优良的使用体验，支持了简体中文（chinese）、繁体中文（taiwanese）以及英文（默认 fallback）的文本及提示对话框国际化。

## 7 How
1. **注册菜单项**：在 `TeXmacs/progs/doc/help-menu.scm` 中直接绑定不含交互修饰的动作：
   ```scheme
   ("Version" (mogan-version))
   ```
2. **多语言词条注册**：在 `english-chinese-miss.scm` 和 `english-taiwanese-miss.scm` 中分别增加 `"Version"` 的翻译词条 `"版本"`。
3. **核心逻辑实现**：在 `TeXmacs/progs/doc/help-funcs.scm` 中实现 `(mogan-version)`：
   - 提取本地版本 `(cur-ver (xmacs-version))` 配合最新稳定版 `"2026.2.5"`（`latest-ver`）进行判断。
   - 使用 `(get-output-language)` 获取当前的输出语言，并使用 `cond` 分流到 `"chinese"`、`"taiwanese"` 和英文分支。
   - 内部条件逻辑：
     - 若 `latest?` 为真，调用 `show-message`（单 OK 按钮信息框）。
     - 若 `latest?` 为假，调用 `user-confirm`（带是/否选项的确认框），若用户选择“是”，则调用 `(open-url "https://liiistem.cn")` 在浏览器中打开官网页面。
