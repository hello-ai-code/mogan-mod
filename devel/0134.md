## What
将 chat-tab-session.scm 中自主拼接的输入输出格式，改为使用 TeXmacs 标准的 `unfolded-io-text` 结构。

1. 修改 `chat-tab-append-round!`，将当前手动拼接的 `(with "font-series" "bold" "User:")` / `(with "font-series" "bold" "Assistant:")` + `output` 节点，替换为标准的 `unfolded-io-text` 节点。
2. `unfolded-io-text` 的子节点依次为：prompt（`document`）、input（`document`）、output（`document`）。
3. prompt 默认使用模型简称（如 `"V4> "`），由新增的 `chat-tab-model-prompt` 从模型名提取。
4. `chat-tab-message-document` 兼容读取 `session` 节点内部的 `document`，为后续顶层结构迁移预留空间。

规范格式示例：
```scheme
(document
  (unfolded-io-text
    (document "V4> ") (document "hello")
    (document "Hello! How can I help you today? "))
  (unfolded-io-text
    (document "V4> ") (document "world")
    (document "Hello! If you have any questions..."))
)
```

## Why
当前 chat-tab-session 采用自主拼接的 document 结构（手动插入 bold "User:"、bold "Assistant:" 和 output 节点），这导致：
- 格式与 TeXmacs 原生 session 机制不兼容；
- 无法复用 session-edit.scm 中已有的字段折叠、prompt 管理、输出处理等基础设施；
- 未来若需支持导出或渲染自定义样式，维护成本高。

通过迁移到标准的 `unfolded-io-text`，可以复用现有的 session 框架，降低维护成本，并确保格式一致性。

## How
在 `chat-tab-session.scm` 中进行如下改造：

1. **新增 `chat-tab-model-prompt`**：
   - 从 `chat-tab-current-model` 提取 prompt 前缀。
   - 将模型名按 `-` 切分，查找包含数字的部分（如 `DeepSeek-V4-Pro` 取 `V4`），否则取最后一部分。
   - 拼接为 `"V4> "` 格式返回。

2. **修改 `chat-tab-append-round!`**：
   - 构建 `unfolded-io-text` 节点，三个子节点分别为 prompt、input、output 的 `document`。
   - prompt 来自 `chat-tab-model-prompt`。
   - input 通过 `chat-tab-body-children` + `tree->stree` 转换为 Scheme 数据后嵌入。
   - output 初始化为空 `(document "")`。
   - 将 `unfolded-io-text` 作为一个整体节点插入到 message-buffer 的 document 末尾。
   - 返回值改为 `(tree-ref last-node 2)`，即 `unfolded-io-text` 的第三个子节点（output document），供 `chat-tab-output` 追加内容。

3. **调整 `chat-tab-message-document`**：
   - 原来只识别 `document` 节点，现在增加对 `session` 节点的兼容：若顶层是 `session`，返回其第三个子节点（内部的 `document`）。
   - 这使得后续若将顶层结构迁移为 `session`，读取逻辑无需再改。

4. **保持现有回调机制不变**：
   - `chat-tab-session-do`、`chat-tab-session-next`、`chat-tab-session-notify`、`chat-tab-session-cancel` 的回调逻辑无需改动。
   - `chat-tab-output` 仍负责将模型返回内容追加到 output document 中。
