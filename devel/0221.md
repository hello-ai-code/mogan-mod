## 5 What

1. LLM 插件端：reasoning delta 不再缓冲，改为实时增量输出 `reasoning-delta` 标记节点
2. LLM 插件端：reasoning 开始时输出 `unfolded-explain` 创建展开结构
3. LLM 插件端：reasoning 结束时输出 `fold-explain-reasoning` 折叠指令
4. LLM 插件端：删除 `emit-folded-reasoning!`，新增 `emit-reasoning-begin!`、`emit-reasoning-delta!`
5. 编辑器端：新增 `tree-contains-label?`，递归检测 tree 中是否包含指定 label
6. 编辑器端：新增 `tree-remove-label-from-children!`，从 document 子节点和 concat 子节点中移除指定 label
7. 编辑器端：新增 `tree-extract-reasoning-delta!`，提取所有 `reasoning-delta` 文本并清除
8. 编辑器端：新增 `chat-tab-find-last-unfolded-explain`，向后搜索 `unfolded-explain`（支持 concat 包裹）
9. 编辑器端：新增 `chat-tab-append-reasoning!`，将 reasoning 文本追加到 `unfolded-explain` body（按 `\n` 拆分为段落）
10. 编辑器端：新增 `chat-tab-fold-last-explain!`，将 `unfolded-explain` 转为 `folded-explain`
11. 编辑器端：修改 `chat-tab-session-notify` output handler，检测 `reasoning-delta` 和 `fold-explain-reasoning` 标记

## 6 Why

原有 reasoning 机制在生成过程中完全缓冲，用户无法实时看到 LLM 的思考过程。改进后 reasoning 以展开态流式展示，生成结束后自动折叠，既让用户了解思考过程，又不占用过多视觉空间。

## 7 How

### 数据流

```
LLM 插件端                          编辑器端
─────────                          ─────────
reasoning 开始:
  flush-scheme-u8                   ch="output"
  (unfolded-explain ...)         →  t 不含 reasoning-delta
                                    → chat-tab-output 正常插入

reasoning delta:
  flush-scheme-u8                   ch="output"
  (reasoning-delta "新增")       →  t 含 reasoning-delta（与 unfolded-explain 混在同一个 document）
                                    → tree-extract-reasoning-delta! 提取文本
                                    → chat-tab-output 输出 t 剩余内容
                                    → chat-tab-append-reasoning! 追加到 unfolded-explain body
                                    → 按 \n 拆分为段落，首段追加到末尾，后续段落新增

reasoning 结束:
  flush-scheme-u8                   ch="output"
  (fold-explain-reasoning)       →  t 可能同时含 reasoning-delta 和 fold-explain-reasoning
                                    → reasoning-delta 分支优先处理
                                    → tree-remove-label-from-children! 移除 fold-explain-reasoning
                                    → chat-tab-fold-last-explain! 折叠
```

### 关键设计

1. **递归标签检测**：`reasoning-delta` 和 `fold-explain-reasoning` 不是 expand 节点，而是嵌套在 document > concat 中的 tree label。使用 `tree-contains-label?` 递归检测
2. **concat 包裹处理**：`unfolded-explain` 被 `concat` 包裹，`chat-tab-find-last-unfolded-explain` 向后搜索所有位置，支持 concat 内查找
3. **混合处理**：`t` 可能同时包含 `reasoning-delta` 和 `fold-explain-reasoning`，reasoning-delta 分支优先处理，一并检测并处理 fold 命令
4. **换行处理**：reasoning 文本不经过 markdown 管线，`chat-tab-append-reasoning!` 按 `\n` 拆分内容，首段追加到 doc 末尾子节点，后续段落作为新子节点插入
5. **增量追加**：LLM 端每个 delta 只输出当前 chunk，编辑器端追加到 `unfolded-explain` 的 body document 中
6. **折叠指令**：`fold-explain-reasoning` 是指令标记，不输出到文档，只触发 `variant-set` 将 `unfolded-explain` 转为 `folded-explain`
7. **上下文隔离**：`chat-tab-tree->plain-text` 对 `folded-explain`/`unfolded-explain` 返回空字符串，确保 reasoning 内容不参与后续对话的上下文构建
