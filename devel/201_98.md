# 201_98 有序列表中实现Tab键缩进和Shift+Tab反缩进功能

## 如何测试
1. 打开`TeXmacs/tests/tmu/201_98.tmu`
2. 在测试文档中可以随意对各级列表项执行 `Tab` 和 `Shift+Tab`
3. 验证 `Tab` 时，当前 item 是否会缩进成为前一个 item 的子列表项
4. 验证前一个 item 已有子列表时，`Tab` 是否会复用并合并到已有子列表，而不是新建相邻子列表
5. 验证当前 item 后面带有同类型子列表时，`Tab` 后是否会一并合并过去
6. 验证 `concat` 包装（就是有内容的列表项）的 item 在 `Tab` 后结构不丢失，光标落点正确
7. 验证子列表中的 item 执行 `Shift+Tab` 后，是否会提升到外层列表
8. 验证 `Shift+Tab` 到最外层列表时，会直接止住，不再继续反缩进
9. 验证 `concat` 包装的 item 在 `Shift+Tab` 后光标位置仍然正确
10. 验证光标在列表项正文内容中间时，`Tab` 和 `Shift+Tab` 仍然可以触发
11. 验证 `Tab` 和 `Shift+Tab` 后，光标会恢复到原列表项内容中的对应位置
12. 验证在行首输入 `- `、`+ ` 可以创建无序列表，输入 `1. ` 可以创建有序列表
13. 验证行首前面只有空格时也可以创建列表，并保留这些空格
14. 验证行首前面已有正文内容时，`- `、`+ `、`1. ` 不会创建列表，只保留普通输入
15. 验证删除当前行前面的列表项标记后，按回车是普通换行，不会创建新的列表项

## 2026/04/13 改进列表快捷键、回车和光标处理

### What
1. 新增 `+ Space` 创建无序列表
2. `- Space`、`+ Space`、`1. Space` 只在行首前面没有内容时触发列表创建
3. 行首前面全是空格时也允许触发列表创建，并保留这些空格
4. 当前光标不在真实列表项中时，列表环境里的回车不再创建新 item，而是普通换行
5. `Tab` 和 `Shift+Tab` 允许在列表项正文内容中触发，不再要求光标正好位于 item marker 上
6. `Tab` 和 `Shift+Tab` 移动列表项后，会优先恢复光标在原 item 内部的相对位置

### Why
之前列表快捷键和列表编辑操作主要依赖光标刚好处在特定结构位置。这样会带来几个问题：在正文中输入 `- ` 或 `1. ` 容易误触发列表；删除 item marker 后，回车仍可能被列表环境接管；光标在列表项正文内容里时，`Tab` 和 `Shift+Tab` 不能稳定命中当前 item，或者移动后光标回到 item marker。

### How
在 `TeXmacs/progs/text/text-edit.scm` 中做了这些调整：

1. 新增空白判断辅助函数
   - `blank-text?`
   - `blank-tree?`
   - `left-siblings-blank-until?`

2. 改进 `line-start-empty-text?`
   - 不再只要求当前文本节点为空和在开头
   - 会检查当前光标左侧的文本前缀是否只包含空白
   - 会继续向上检查当前段落中光标左侧的兄弟节点是否都为空白

3. 改进列表回车判断
   - 新增 `list-item-marker?`
   - 新增 `list-item-wrapper?`
   - 新增 `list-item-logical-tree`
   - 新增 `current-list-item-tree`
   - `kbd-enter` 的列表分支增加 `(current-list-item-tree)` 要求，避免删除 item marker 后仍然创建新 item

在 `TeXmacs/progs/text/text-kbd.scm` 中做了这些调整：

1. `- Space` 改为 `(make-tmlist-if-line-start 'itemize "- ")`
2. 新增 `+ Space`，使用 `(make-tmlist-if-line-start 'itemize "+ ")`
3. `1. Space` 继续使用 `(make-tmlist-if-line-start 'enumerate "1. ")`
4. 当不满足行首条件时，通过 fallback 保留普通输入

在 `TeXmacs/progs/generic/generic-edit.scm` 中做了这些调整：

1. 新增列表项定位辅助函数
   - `list-item-marker-node?`
   - `list-item-wrapper-node?`
   - `current-list-item-marker`
   - `list-tab-context-match?`

2. `current-list-item-marker` 优先从 `cursor-tree` 向上找当前列表项
   - 光标在列表项正文内容中时，可以找到外层 `concat`
   - 如果找到 `concat` 包装，则取第 0 个子节点作为真正的 item marker
   - 如果 `cursor-tree` 找不到，再回退到 `focus-tree`

3. 新增光标恢复辅助函数
   - `list-item-cursor-state`
   - `go-to-list-item-relative-path`
   - `go-to-moved-list-item`

4. 在移动 item 前，通过 `tree-cursor-path` 保存光标相对当前 item 或 `concat` wrapper 的路径
5. 在移动 item 后，优先把同一个相对路径应用到新的 `moved-item` 上
6. 如果相对路径无法恢复，再回退到内容末尾或 item marker

## 2026/04/03 改进列表Tab键缩进功能

### What
1. `Tab` 缩进现在同时支持 `enumerate`、`itemize`、`description`
2. 当前 item 前一个 item 已经有同类型子列表时，会直接复用这个子列表
3. 当前 item 后面如果紧跟同类型子列表，缩进时会一并合并到目标子列表
4. `concat` 包装的 item 也按完整 item 处理，并修正光标位置

### Why
之前的 `Tab` 缩进实现是直接新建一个子列表，再把当前 item 放进去。这样在“前一个 item 已有子列表”或者“当前 item 后面本来还挂着子列表”的情况下，会产生相邻的碎片列表，结构不够自然。

### How
在 `TeXmacs/progs/generic/generic-edit.scm` 中做了这些调整：

1. 新增辅助函数
   - `list-item-node?`
   - `list-node?`
   - `find-previous-item-index`
   - `find-following-list-index`
   - `append-strees-to-document`

2. `kbd-variant` 的 `forwards? = #t` 分支中：
   - 先通过 `find-item-wrapper-and-list` 识别当前逻辑 item
   - 向左查找前一个逻辑 item
   - 优先查找并复用前一个 item 后面已有的同类型子列表
   - 如果当前 item 后面紧跟同类型子列表，则把这部分内容一起并入目标子列表
   - 最后删除旧位置，并在 `concat` 场景下把光标落到 `tree-go-to moved-item 0 :end`

## 2026/04/02 新增列表Shift+Tab键反缩进功能

### What
在 `TeXmacs/progs/generic/generic-edit.scm` 中新增了列表的 `Shift+Tab` 反缩进功能。当用户在子列表的 item 上按 `Shift+Tab` 键时，该 item 会提升为外层列表的同级项。

### Why
在实现 `Tab` 键缩进功能后，还需要相应的反缩进功能来提供更完整的列表编辑体验。当用户误操作缩进或者需要调整列表结构时，`Shift+Tab` 可以方便地将子列表项提升为父列表的同级项。

### How
在 `generic-edit.scm` 中新增了 `kbd-variant` 的 `forwards? = #f` 分支，主要逻辑如下：

1. 处理当前 item 的 `concat` 包装
2. 识别当前 `sublist`、外层 `parent-list`，以及更高一层的 `outer-list`
3. 只有当前子列表外面还能找到另一层列表时，才允许继续反缩进
4. 提前保存当前 item 后面的兄弟节点，用于后续重建 trailing sublist
5. 分两种情况处理：
   - 当前 item 不是第一个 item：保留原 `sublist`，将当前 item 插到 `sublist` 后面
   - 当前 item 是第一个 item：删除原 `sublist`，将当前 item 插回原 `sublist` 位置
6. 如果当前 item 后面原本还有兄弟节点，则在新位置后面重建同类型子列表
7. 最后根据是否为 `concat` item 修正光标位置

## 2026/02/13 新增有序列表Tab键缩进功能

### What
在 `TeXmacs/progs/generic/generic-edit.scm` 中新增了有序列表的 `Tab` 键缩进功能。当用户在有序列表的 item 上按 `Tab` 键时，该 item 会缩进成为前一个 item 的子列表项。

### Why
在 LaTeX 等文档编辑器中，有序列表支持通过 `Tab` 键将列表项缩进为子列表，这是常见的编辑功能。TeXmacs 此前缺少这一功能，用户体验不够友好。通过添加此功能，用户可以更方便地创建嵌套的有序列表结构。

### How
最初版本在 `generic-edit.scm` 中新增了以下内容：

1. 辅助函数
   - `in-enumerate-context?`
   - `find-item-wrapper-and-list`
   - `extract-item-content`
   - `remove-item-from-list`

2. `kbd-variant` 的 `forwards? = #t` 分支
   - 检查当前是否在有序列表环境中，并且光标位于 item 上
   - 找到当前 item 的包装和所属列表
   - 检查是否存在前一个 item
   - 在前一个 item 末尾创建子列表
   - 将当前 item 移动到新创建的子列表中
   - 从原列表中移除当前 item
