## 5 What

1. 将 Chat Tab 的 C++ 代码重构为 MVC 架构，拆分出 ChatController（逻辑层）、ChatSessionManager（数据层）和 QTChatTabWidget/ChatSidebar（视图层）
2. 修复 `chat-tab-session->input-buffer` 未绑定错误（`define` → `tm-define`）
3. 新建会话插入列表顶部而非底部
4. 多选模式执行动作后自动退出多选模式
5. 归档/删除激活会话后自动转移激活到第一个非归档会话，无可用会话时自动新建
6. 修复归档区计数因 `isVisible()` 循环依赖导致归档会话不显示的问题
7. 修复 `chat-persist-extract-title` 参数类型错误（字符串 `"20"` → 整数 `20`）
8. 重命名 `lastSessions_` → `sessionCache_`、`lastActiveId_` → `activeSessionId_`

## 6 Why

原有的 Chat Tab C++ 代码将所有逻辑集中在 `qt_chat_tab_widget.cpp` 中（超过 2500 行），导致维护困难、职责不清。重构为 MVC 后，Controller 集中管理 Scheme 交互和状态变更，View 只负责 UI 渲染，便于后续功能扩展和 bug 修复。

同时修复了多个 sidebar 交互问题：新建会话位置、多选模式退出、计数显示、归档会话激活等。

## 7 How

- **MVC 拆分**：从 `qt_chat_tab_widget.cpp` 中提取出 `qt_chat_controller.cpp`（信号处理、Scheme 调用、会话生命周期）和 `qt_chat_session.cpp`（会话数据管理），View 只保留 UI 构建和渲染逻辑
- **sidebar 计数修复**：`updateCountLabels` 去掉 `isVisible()` 检查，改为按 `isArchived` 数据标记直接统计，避免归档区折叠后子 widget 不可见导致计数归零的循环依赖
- **激活转移**：在 `onDeleteRequested` 和 `onArchiveRequested` 中，操作完成后查找第一个非归档会话并激活，找不到则调用 `ensureNewConversation` 新建
