## 3 如何测试

### 3.1 确定性测试（单元测试）
运行单元测试验证 PDF 十六进制数据解析和 Base64 路径不生成无效 MIME type：
```bash
xmake r 0416
```

### 3.2 非确定性测试（文档验证）
1. 在 Mogan 中打开包含嵌入式 PDF 图片的文档（如 Gnuplot 教程 `.tmu` 文件）。
2. 选择 **文件 -> 导出 -> HTML**。
3. 确保导出时启用了 Base64 选项（`tmhtml-base64?` 为 true）。
4. **检查 HTML 输出**：
   - 搜索 `<img>` 标签，确认 `src` 属性为 `data:image/png;base64,...` 格式，**而非** `data:image/pdf;base64,...`。
   - 用浏览器打开 HTML 文件，确认图片能正常显示。

## 5 What
修复了 `tmhtml-image` 函数在 Base64 模式下对嵌入式 PDF/PS/EPS/TIF 数据的处理逻辑：原先直接对这些非原生图片格式进行 Base64 编码，生成浏览器无法识别的 `data:image/pdf;base64,...` URI；现在改为先写入临时文件，调用 `tmhtml-png` 转换为 PNG，再利用 `tmhtml-png` 内部的 Base64 逻辑生成有效的 `data:image/png;base64,...` URI。

具体改动包括：
- 在 `tmhtml-image` 函数的嵌入式图片 Base64 分支中，增加 `ext` 格式判断：
  - **原生格式**（如 `png`、`jpg`）：保持直接 Base64 编码路径。
  - **需转换格式**（`ps`/`eps`/`pdf`/`tif`）：先写入临时文件，调用 `tmhtml-png` 转换为 PNG，`tmhtml-png` 内部会自动读取 PNG 字节并生成 `data:image/png;base64,...` URI，最后清理临时文件。
- 在 `TeXmacs/tests/0416.scm` 中新增 PDF 十六进制数据解析测试和 Base64 路径验证。

## 6 Why
1. **修复无效 MIME type**：`data:image/pdf;base64,...` 不是浏览器支持的有效 data URI 格式，导致导出的 HTML 中所有嵌入式 PDF 图片都无法显示。
2. **复用现有转换能力**：`tmhtml-png` 已经具备将 PDF 渲染为 PNG 的能力，并且内部已集成 Base64 编码逻辑，复用此函数风险最低。
3. **保持代码简洁**：仅在 Base64 分支中增加一层 `in?` 判断，非 Base64 模式完全不受影响。

## 7 How
1. **格式判断**：使用 `(in? ext (list "ps" "eps" "pdf" "tif"))` 判断是否需要格式转换。
2. **临时文件写入**：对需转换格式，使用 `tmhtml-image-names` 生成临时文件名，`tmhtml-write-binary-file` 写入原始二进制数据。
3. **调用转换**：`(tmhtml-png (cons 'image (cons abs-string (cdr l))))` 将临时文件交给 `tmhtml-png` 处理，`tmhtml-png` 会：
   - 渲染为 PNG 文件
   - 读取 PNG 字节并 Base64 编码
   - 生成 `data:image/png;base64,...` URI
   - 删除 PNG 文件
4. **清理临时文件**：在 `tmhtml-png` 返回后，使用 `(url-remove abs-url)` 删除临时的 PDF/PS/EPS/TIF 文件。
5. **测试覆盖**：新增测试用例验证：
   - `tmhtml-extract-embedded` 能正确解析 PDF 十六进制数据（`<#255044...#3e` 格式）
   - 提取后的字节数组以 `%PDF` 开头
   - PDF 数据不会直接生成 `data:image/pdf;base64,...` URI
