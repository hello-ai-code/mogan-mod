# 222_78 Revert 工具栏位置调整记忆

## 如何测试

1. 编译：`xmake b stem`
2. 启动 Mogan，拖动工具栏到不同位置（如上下左右或悬浮）。
3. 关闭窗口后重新打开。
4. 确认工具栏位置**不再被记忆**，每次启动均恢复为默认布局。
5. 确认窗口大小和位置**不再被记忆**。

## 2026/04/29 实现说明

### What

Revert commit `f852a011`（!273 工具栏位置调整记忆），移除窗口状态与几何信息的自动保存/恢复功能。

#### 修改文件

**src/Plugins/Qt/QTMWindow.cpp**
- 移除 `#include "tm_sys_utils.hpp"`、`#include "tm_url.hpp"`、`<QSettings>`。
- 移除 `saveSettings ()` 函数实现。
- 移除 `closeEvent` 中对 `saveSettings ()` 的调用。

**src/Plugins/Qt/QTMWindow.hpp**
- 移除 `saveSettings ()` 声明。

**src/Plugins/Qt/qt_tm_widget.cpp**
- 移除 `<QSettings>` include。
- 移除 `restoreSettings ()` 函数实现。
- 移除构造函数中对 `restoreSettings ()` 的调用。
- 保留 `#include "tm_sys_utils.hpp"` 和 `#include "tm_url.hpp"`（后续功能依赖）。

**src/Plugins/Qt/qt_tm_widget.hpp**
- 移除 `restoreSettings ()` 声明。

### Why

工具栏位置记忆功能在当前版本不再需要，revert 以保持行为简单一致，避免状态文件残留和潜在的窗口恢复问题。

### How

- 直接删除 `saveSettings` / `restoreSettings` 函数及其调用点。
- 清理相关的 `<QSettings>` 和仅被该功能使用的 include。
- `qt_tm_widget.cpp` 中的 `tm_sys_utils.hpp` 和 `tm_url.hpp` 因后续提交已广泛使用（`is_community_stem ()`、`url` 等），予以保留。
