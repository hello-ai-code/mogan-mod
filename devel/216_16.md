# 216_16 启动页 Recent 与“文件->最近使用”对齐并支持实时刷新

## 如何测试
1. 编译：`xmake b stem`
2. 打开启动页 `File` 页面，确认 Recent 列表正常显示。
3. 从 `File -> Recent used` 与启动页 Recent 对比顺序与可见项，确认一致。
4. 保持停留在启动页 `File` 页面，通过其他入口打开文档（如 Template / Open a Document / 外部触发），确认 Recent 在页面可见时会自动刷新，不需要手动切页。
5. 切到 `Template` 或 `Settings` 再切回 `File`，确认 Recent 立即刷新到最新状态。

## 2026/04/16 实现说明

### What
本次补齐两项能力：
1. 启动页 Recent 的数据口径与 `文件 -> 最近使用` 完全对齐（同源、同筛选、同数量限制）。
2. 启动页 `File` 页面可见期间自动轮询刷新 Recent，解决“没有实时刷新”的问题。

#### 修改文件

**TeXmacs/progs/startup-tab/startup-tab-file.scm** (修改)
- 引入 `(texmacs menus file-menu)`。
- `startup-tab-get-recent-docs` 改为基于 `recent-file-list` 返回结果，直接复用 File 菜单的过滤与排序逻辑。
- 读取并应用偏好项 `startup-tab:max-recent`（异常值回退为 10，最小值 1）。

**src/Plugins/Qt/qt_file_page.cpp** (修改)
- Recent 读取以 `(startup-tab-get-recent-docs)` 返回的路径列表为主，确保与 File 菜单一致。
- 从 `recent-files.json` 仅补充名称/时间等元数据，不再由 Qt 侧自行决定可见集合。
- 新增 `QTimer` 周期刷新（1s）：
  - `showEvent`：页面显示时立即刷新并启动定时器；
  - `hideEvent`：页面隐藏时停止定时器。
- 保持原有手动刷新入口 `refreshRecentDocs()`，与自动刷新机制并存。

**src/Plugins/Qt/qt_file_page.hpp** (修改)
- 增加 `showEvent/hideEvent` 声明。
- 增加 `QTimer* recentRefreshTimer_` 成员用于可见期自动刷新。

### Why
1. 之前启动页 Recent 与 File 菜单虽共享同一底层数据，但读取口径不一致，导致展示集合和顺序可能偏差。
2. 仅在切页时刷新不能覆盖“当前页面停留期间”的文档打开行为，用户感知为不实时。

### How
1. 在 Scheme 层收敛数据口径：启动页读取逻辑直接复用 `recent-file-list`。
2. 在 Qt 层将“路径集合”和“元数据展示”职责分离：路径以 Scheme 为准，JSON 仅做展示补全。
3. 通过 `show/hide + QTimer` 形成轻量实时刷新机制，保证页面可见时状态持续同步。
