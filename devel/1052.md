## 5 What
修复深色主题（liii-night.css）下 AI Chat Dock 模式中输入框、发送按钮、思考按钮等组件缺少样式的问题。

1. **input-frame 添加后备选择器** — `QWidget#chat-tab-input-frame` 与带 `#centralWidget` 前缀的选择器并列
2. **thinking-btn 添加后备选择器** — `QToolButton#chat-tab-thinking-btn` 与带 `#centralWidget` 前缀的选择器并列
3. **send-btn 添加后备选择器** — `QPushButton#chat-tab-send-btn` 与带 `#centralWidget` 前缀的选择器并列
4. **collapse-btn 添加后备选择器** — `QPushButton#chat-tab-collapse-btn` 与带 `#centralWidget` 前缀的选择器并列（dock 模式关闭按钮共用此组件）
5. **content / conversation-page 添加后备选择器** — 背景色在 dock 模式下生效
6. **model-label 添加后备选择器** — 文字颜色在 dock 模式下生效
7. **message-frame 添加后备选择器** — 消息区背景色在 dock 模式下生效

## 6 Why
`liii-night.css` 中 chat 组件的样式选择器使用了 `QWidget#centralWidget` 祖先限定：

```css
QWidget#centralWidget QWidget#chat-tab-input-frame { ... }
```

在 Chat Tab 全屏模式下，`QTChatTabWidget` 作为 `centralWidget()` 的子控件，上述选择器正常匹配。

但在 Dock 模式下（`qt_tm_widget_rep::sync_chat_sidebar_mode()`），`chatContentWidget` 被移动到 `QDockWidget` 中：

```cpp
chatContentWidget->setParent(chatSideDock);
chatSideDock->setWidget(chatContentWidget);
```

此时 `chatContentWidget` 不再是 `QWidget#centralWidget` 的后代，所有带 `QWidget#centralWidget` 前缀的选择器全部失效，组件回退到 Qt 默认样式（无边框、无背景）。

相比之下，`liii.css`（浅色主题）的选择器没有祖先限制，所有模式下都能正常匹配：

```css
QWidget#chat-tab-input-frame { ... }
```

## 7 How
为 dock 模式下可见的每个组件，在 `liii-night.css` 中添加不带 `QWidget#centralWidget` 前缀的后备选择器，与原选择器用逗号并列：

```css
/* 之前：仅 centralWidget 内生效 */
QWidget#centralWidget QWidget#chat-tab-input-frame {
  background-color: #404040;
  border: 1px solid #4c4c4c;
}

/* 之后：dock 和全屏模式均生效 */
QWidget#chat-tab-input-frame,
QWidget#centralWidget QWidget#chat-tab-input-frame {
  background-color: #404040;
  border: 1px solid #4c4c4c;
}
```

由于 `QWidget#centralWidget Foo#bar` 的特异性（0,2,0,2）高于 `Foo#bar`（0,1,0,1），在同一规则组中并列不会产生特异性冲突。侧边栏专属组件（搜索框、多选按钮、浮球容器、会话列表等）保留 `#centralWidget` 前缀，因为它们在 dock 模式下被 `setSidebarVisible(false)` 隐藏，无需修改。
