## 5 What

重启应用后，归档会话恢复时内容丢失。

1. Scheme 端将 `chat-persist-save-one` 拆分为两个独立函数：
   - `chat-persist-export-buffer`：只导出 buffer 到磁盘
   - `chat-persist-update-manifest`：只更新 manifest 元数据
2. C++ 端将 `saveOneSession` 拆分为 `exportBuffer` + `updateManifest`
3. 各场景按需组合：buffer 变动时 export，元数据变动时 update manifest
4. 空白会话不持久化、不归档

## 6 Why

`onRestoreRequested` 中先调用 `saveOneSession`（会 export buffer），但此时 buffer 内容尚未加载（空的），空内容覆盖了磁盘上持久化的对话消息。之后 `activateSession` 从磁盘加载时文件已为空。

同样的问题也存在于 `onArchiveRequested` 中——重启后归档未加载内容的 session 也会覆盖磁盘文件。

## 7 How

### 持久化调用场景

| 场景 | export buffer | update manifest |
|------|:---:|:---:|
| `onSendRequested` 发送消息 | ✓ | ✓ |
| `notifyStateChanged` 生成完成 | ✓ | ✗ |
| `onArchiveRequested` 归档（非空白） | ✗ | ✓ |
| `onRestoreRequested` 恢复 | ✗ | ✓ |
| `ensureNewConversation` 新建 | ✗ | ✗ |
| `onDeleteRequested` 删除 | `chat-persist-delete-one` | - |
