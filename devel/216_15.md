# 216_15 修复启动页标签点击时取消选中的问题和拖拽普通页无法编辑问题

## 如何测试
1. **启动页标签点击测试**：启动软件后，确保当前处于启动页，点击启动页标签，观察标签应保持选中状态（文字底部有高亮），不应被取消选中
2. **切换标签后返回测试**：先切换到其他普通文档标签，然后再次点击启动页标签，启动页标签应正确选中并显示启动页界面
3. **普通标签页测试**：打开多个普通文档标签页，点击已选中的标签页，确认不会出现类似问题
4. **启动页标签拖拽测试**：尝试拖拽启动页标签，确认不会被移动（启动页标签应始终固定在第一个位置）
5. **普通标签页拖出测试**：**打开多个普通文档标签页，拖拽其中一个标签页到新窗口，确认新窗口能正常编辑，不会出现全白无法编辑的情况**

## 2026/04/16 修复启动页标签点击取消选中问题

### What
1. 修复点击已选中的启动页标签时，标签被取消选中的问题
2. 修复拖拽普通标签页到新窗口后，新窗口全白无法编辑的问题

#### 新增文件：

无新增文件

#### 修改文件：

**src/Plugins/Qt/QTMTabPage.cpp** (修改)
- 修改 `mousePressEvent` 函数中启动页标签的处理逻辑：
  - 原代码：
    ```cpp
    if (is_startup_tab_view (m_viewUrl)) {
      return QToolButton::mousePressEvent (e);
    }
    ```
  - 新代码：
    ```cpp
    if (is_startup_tab_view (m_viewUrl)) {
      // 如果启动页标签已经是当前视图，不处理点击事件，避免取消选中状态
      url currentView= get_current_view_safe ();
      if (!is_none (currentView) && currentView == m_viewUrl) {
        return;
      }
      return QToolButton::mousePressEvent (e);
    }
    ```

**src/Texmacs/Data/new_view.cpp** (修改)
- 在 `view_set_new_window` 函数中添加 `set_current_view` 和 `set_window_url` 调用：
  - 原代码：
    ```cpp
    detach_view (view_u);
    attach_view (win_u, view_u);
    view->win_tabpage= win;
    ```
  - 新代码：
    ```cpp
    detach_view (view_u);
    attach_view (win_u, view_u);
    set_current_view (view_u);
    win->set_window_url (view->buf->buf->name);
    view->win_tabpage= win;
    ```

### Why

#### 修复 1：启动页标签点击取消选中问题
1. **问题的根本原因**：当点击已经选中的启动页标签时，`window_set_view` 会因为 `new_vw->win == win` 条件成立而提前返回，不做任何操作；但 `QToolButton::mousePressEvent` 仍然会执行，Qt 会改变按钮的 checked 状态
2. **启动页的特殊性**：启动页 (`tmfs://startup-tab`) 是特殊的虚拟 buffer，不能拖拽移动，也不能关闭，需要特殊处理
3. **用户体验**：用户点击已选中的标签时，预期是保持当前状态不变，而不是取消选中

#### 修复 2：标签页拖出新窗口全白问题
1. **缺少关键调用**：`view_set_new_window` 函数中只调用了 `attach_view`，没有调用 `set_current_view` 和 `set_window_url`
2. **与启动页相关**：`set_window_url` 会触发 `SLOT_FILE`，进而设置 `startupTabMode`。缺少这个调用会导致新窗口处于不一致状态
3. **与正常流程对比**：`window_set_view` 中会调用这两个函数，而 `view_set_new_window` 缺少它们导致问题

### How

#### 修复 1：启动页标签点击问题
1. 在 `mousePressEvent` 中，首先判断是否为启动页标签（`is_startup_tab_view`）
2. 如果是启动页标签，获取当前视图（`get_current_view_safe`）
3. 如果当前视图就是该启动页标签对应的视图，直接返回，不调用 `QToolButton::mousePressEvent`，从而避免 Qt 改变按钮状态
4. 如果当前不是启动页标签，或者启动页标签不是当前视图（例如从其他标签切换到启动页），则正常处理点击事件

#### 修复 2：标签页拖出问题
1. 在 `view_set_new_window` 中，`attach_view` 之后添加 `set_current_view(view_u)` 调用
2. 添加 `win->set_window_url(view->buf->buf->name)` 调用，触发 `SLOT_FILE` 并正确设置 `startupTabMode`
3. 确保新窗口的状态与正常 `window_set_view` 流程一致
