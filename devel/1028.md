## 5 What

1. 将 C++ 中内联的硬编码颜色值（`#ffffff`、`#f0f0f0`、`#cccccc`、`#d9d9d9`、`#e0e0e0`、`#888888` 等）移除，改为由 CSS 主题文件统一管理
2. 修复归档区背景色偏深：给 `scrollContent` 设置 `objectName("chat-tab-scroll-content")` 匹配 CSS 选择器
3. 为会话按钮 `chat-tab-conversation-btn` 添加 `:hover` 悬停效果
4. 亮色主题补全侧边栏子控件选择器（`chat-tab-sidebar-normal`、`chat-tab-header`、`chat-tab-nav-title`、`chat-tab-conversation-count`、`chat-tab-multi-select-bar`、`chat-tab-session-item QWidget`）
5. 清理 CSS 中冗余的 `border-color`、`background: transparent` 等属性

## 6 Why

0219 重构时为了快速实现功能，在 C++ 内联样式中硬编码了大量颜色值。这导致两个问题：
- 暗色主题无法覆盖这些内联样式（内联优先级高于 CSS），控件在暗色模式下显示异常
- 修改样式需要改动 C++ 代码并重新编译，无法通过主题文件调整

同时新增的 `scrollContent` widget 缺少 `objectName`，CSS 中已有的 `QWidget#chat-tab-scroll-content` 选择器无法匹配，导致归档区使用 Qt 默认灰色背景。

## 7 How

- **去内联**：逐一移除 `qt_chat_tab_widget.cpp` 中的 `setStyleSheet` 内硬编码颜色（输入框 border、viewport 背景、发送按钮 hover、模型标签颜色、滚动条 handle、多选操作栏按钮 border 等），只保留布局相关属性（border-radius、padding）
- **CSS 补全**：在 `liii.css` 的侧边栏选择器组中补充缺失的子控件选择器，统一设为 `background-color: #f5f5f5`；在 `liii-night.css` 中同步清理对应的 `border-color`
- **scrollContent 修复**：添加 `setObjectName("chat-tab-scroll-content")`，复用 CSS 中已有选择器
- **hover 效果**：在两个主题文件中为 `QPushButton#chat-tab-conversation-btn` 添加 `:hover` 伪状态
