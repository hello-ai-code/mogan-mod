## 2026/04/19 修复 Fedora 平台构建及中文字体识别问题

### What
1. 修复了 Fedora 平台上中文字体（Noto CJK）无法被正确识别的问题
2. 修复了 Fedora 平台构建时使用 apt 包导致的依赖问题
3. 修复了 EC 字体空字符警告问题
4. 修复了 small-caps 字体对 CJK 字符的错误处理
5. 添加了 Fedora CI 支持

### Why
1. **字体问题**：Fedora 将 Noto CJK 字体安装在 `/usr/share/fonts/google-noto-serif-cjk-fonts/` 路径下，而软件原本只搜索 `/usr/share/fonts/noto-cjk`（Arch Linux 路径）和 fontconfig 返回的路径。由于 Fedora 的 fontconfig 配置问题，导致字体未被正确扫描，使得中文支持不可用。

2. **构建问题**：`xmake.lua` 中强制使用 `apt::` 包管理，在非 Debian/Ubuntu 系统（如 Fedora）上无法找到依赖。

3. **警告问题**：当 EC 字体无法翻译特殊字符（如 `<#9501>`）时，`special_translate` 返回空字符串，导致产生大量警告。

4. **small-caps 问题**：`poor_smallcaps_font` 尝试对中文字符进行大小写转换，导致错误。

### How
`xmake.lua` :243-251 修复非 apt 系统的依赖处理
- 对 Fedora/RHEL/CentOS/Rocky/AlmaLinux/OL 系统使用系统包
- 对其他 Linux 发行版使用 xmake 源码构建依赖

`src/Plugins/Freetype/tt_file.cpp` :141-151 添加 Fedora 特有的字体搜索路径
- `/usr/share/fonts/google-noto-serif-cjk-fonts`
- `/usr/share/fonts/google-noto-sans-cjk-fonts`
- `/usr/share/fonts/google-noto`

`src/Plugins/Metafont/tex_font.cpp` :1033-1036 修复 EC 字体空字符警告
- 当 `special_translate` 无法翻译特殊字符时，直接返回 -1 而不是继续处理空字符串

`src/Graphics/Fonts/poor_smallcaps.cpp` :15,63-67,79-84 修复 small-caps 字体对 CJK 字符的处理
- 添加 `is_cjk_unified_ideographs` 检测，避免对中文字符进行大小写转换

`bin/format` :10-11,18-19 添加 Fedora 支持
- 添加 `clang-format-19` 函数定义
- 添加 `/usr/local/bin` 和 `$E:HOME/.local/bin` 到搜索路径

`.github/workflows/ci-fedora.yml` 新增 Fedora CI 工作流
- 使用 `fedora:latest` 容器
- 安装 `dnf` 依赖包（包括 `google-noto-sans-cjk-fonts`, `google-noto-serif-cjk-fonts`, `libpng-devel`, `libcurl-devel`, `harfbuzz-devel` 等）
- 添加 `git add safe directory` 步骤
- 设置 `XMAKE_GLOBALDIR` 环境变量
- 使用 `xmake repo --update` 更新 xmake 仓库
- 添加 `actions/cache` 缓存 xrepo 包和 build cache
- 简化 config 命令（移除 `--png=false` 和 `--require=freetype`）
