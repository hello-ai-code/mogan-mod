### 非确定性测试（功能验证）
1. 启动 Mogan，进入 Template Center
2. 点击一个**已缓存**的模板卡片 → 应正常打开，且 Documents 文件夹下生成对应 `.tmu` 文件（如 `TemplateName.tmu`）

## What

1. **新建 `qt_template_utils` 工具模块**：提取模板拷贝和文档加载的公共逻辑，供 Home 页和 Template Center 页复用。
2. **本地已缓存模板**：打开时先拷贝到用户 Documents 文件夹，再执行 `load-document`，避免直接编辑缓存文件。
3. **首次下载模板**：下载完成后同样拷贝到 Documents 再打开，保持行为一致。
4. **自动编号去重**：Documents 中已存在同名文件时，自动生成 `Name(1).tmu`、`Name(2).tmu` 等序号文件名。
5. **修复异步下载导致的 crash**：将 Home 页和 Template Center 页的模板下载从异步回调改为同步阻塞（`QEventLoop`），确保 `load-document` 在窗口状态稳定时执行，彻底消除 `no window attached to view` 崩溃。
7. **错误提示统一为浮动 Toast**：下载失败、模板元数据缺失等场景统一使用 `QtFloatingToast::Error`，避免 `QMessageBox` 阻塞用户。

## Why

- 用户直接打开模板时，原先会直接加载缓存目录下的 `.tmu` 文件，导致用户编辑的内容污染缓存。
- 需要保证每次打开模板都是一份全新的副本，同时保留原始缓存不被修改。
- 两个页面（Home 页、Template Center 页）存在大量重复的"拷贝+加载"代码，需要提取复用。
- 异步下载完成后在信号槽中调用 `eval_scheme("(load-document ...)")` 时，Scheme 层 `concrete_window()` 断言 `no window attached to view` 崩溃，因为此时模态对话框关闭后的窗口切换导致当前 view 为空。同步阻塞确保下载和打开在同一个调用栈中执行，状态稳定。

## How

### 1. 工具函数提取

新增 `qt_template_utils.hpp/cpp`，提供四个无状态工具函数：

- `qt_generate_document_save_path`：基于 `QStandardPaths::DocumentsLocation` 生成唯一文件名，自动过滤非法字符、自动编号。
- `qt_copy_template_to_documents`：调用生成函数得到目标路径，执行 `QFile::copy`。
- `qt_load_document_path`：封装 `eval_scheme("(load-document ...)")`。
- `qt_copy_template_and_load`：组合拷贝+加载，失败时通过 `QtFloatingToast` 提示用户。

### 2. TemplateManager 新增同步下载接口

`template_manager.hpp/cpp` 新增 `downloadTemplateSync`：

```cpp
QString downloadTemplateSync(
    const QString& templateId,
    int timeoutMs = 30000,
    QString* errorMessage = nullptr
);
```

- 本地已缓存时直接短路返回 `localTemplatePath`
- 否则启动异步下载，通过 `QEventLoop` 阻塞等待 `downloadCompleted` / `downloadFailed` 信号
- 超时（默认 30s）自动 `cancelDownload` 并返回空路径
- 失败时通过 `errorMessage` 输出参数返回错误信息

### 3. 使用方改造

`qt_home_page.cpp` 和 `qt_template_page.cpp` 中统一替换为同步流程：

```cpp
// 本地已有缓存
if (mgr->isTemplateAvailableLocally(templateId)) {
    auto meta = mgr->templateById(templateId);
    // ... 校验 ...
    qt_copy_template_and_load(this, localPath, meta->name);
    return;
}

// 需要下载
QProgressDialog dialog(...);
dialog.setWindowModality(Qt::WindowModal);
// ... 连接 cancel 和 downloadProgress ...
dialog.show();

QString errorMsg;
QString localPath = mgr->downloadTemplateSync(templateId, 30000, &errorMsg);
dialog.hide();

if (localPath.isEmpty()) {
    if (!cancelledByUser) {
        QtFloatingToast::showToast(
            this, errorMsg.isEmpty() ? tr("Download failed") : errorMsg,
            3000, QtFloatingToast::Error);
    }
    return;
}

auto meta = mgr->templateById(templateId);
if (!meta) {
    QtFloatingToast::showToast(
        this, tr("Template metadata not found"), 3000,
        QtFloatingToast::Error);
    return;
}
qt_copy_template_and_load(this, localPath, meta->name);
```

### 4. 清理冗余代码

- `qt_template_page.hpp`：移除 `progressDialog_` 成员、`downloadCancelledByUser_` 标志、`onDownloadProgress` / `onDownloadCompleted` / `onDownloadFailed` 三个槽、`QNetworkAccessManager` / `QNetworkReply` / `QProgressDialog` 前向声明
- `qt_template_page.cpp`：移除 `initialize()` 中对下载信号的类级别连接
- `qt_home_page.cpp`：移除 `DownloadCtx` 结构体、`#include <memory>` / `#include <QMessageBox>`
- `qt_template_page.cpp`：移除 `#include <QMessageBox>`
- `qt_startup_tab_widget.cpp`：移除 `templateOpened` 信号连接（模板页内部已直接调用 `load-document`）

### 5. 历史问题

原异步方案中，Home 页 `elegantbook`/`nsfc-ysf-c` 固定模板在首次下载完成后，从 `downloadCompleted` 信号槽中调用 `eval_scheme` 会触发 Mogan 内部断言 `no window attached to view`。原因：模态 `QProgressDialog` 关闭后，Scheme 层 `concrete_window()` 找不到当前激活窗口。通过同步重构彻底修复。
