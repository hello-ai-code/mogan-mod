## 如何测试
1. 编译时先输入 `xmake config -vD --startup_tab=true`
2. 启动 Mogan STEM，进入 `Mogan STEM -> Settings`
3. 验证 Settings 首页签：
   - **Startup / General / Keyboard / Mathematics / Convert / Other** 均可切换
   - 每个页签内容可正常显示、交互可用（toggle/enum/输入项）
4. 验证 Convert 子标签：
   - **Html / LaTeX / BibTeX / Verbatim / Pdf / Image / Mogan Scheme** 正常显示与切换
5. 在 `icon-tabs` 上连续切换标签，不应出现窗口被动自动缩放抖动
6. 中文界面下确认 `"Startup"` 文案显示为 `"启动页"`

## 2026/04/09 启动页 Settings 分支实现

### What
本分支完成了启动页 `Settings` 的真正落地：从“占位页面”升级为“由 Scheme 定义并渲染的完整设置页”，并补齐了标签布局一致性与点击切换时的尺寸稳定性问题。

#### 新增文件：

**src/Plugins/Qt/qt_settings_page.hpp** (新增)
- 新增 `QTSettingsPage` 类声明，用于承载“Scheme 驱动”的设置页 QWidget。

**src/Plugins/Qt/qt_settings_page.cpp** (新增)
- 构建 `QTSettingsPage`：
  - 加载模块 `(startup-tab startup-tab-settings)`
  - 通过 `(startup-settings-widget)` 获取 Scheme widget 定义
  - 使用 `make_menu_widget` 渲染并嵌入 Qt 布局
- 设置 `tm_no_tab_auto_resize` 属性，抑制标签切换导致的窗口级自动缩放。

### Why
1. 启动页 `Settings` 之前是占位实现，无法承载真实用户配置。  
2. 设置页面配置项多，采用 Scheme 统一定义更利于后续扩展与维护。  
3. 标签切换引起的窗口自动缩放会影响体验，需要在容器层明确禁止。  
4. 顶层各页签布局策略不一致，需统一视觉与交互行为。  

### How
1. **渲染链路**：Qt 容器页 (`QTSettingsPage`) + Scheme widget (`startup-settings-widget`) 组合，实现“Qt 承载、Scheme 定义”。  
2. **集成方式**：启动页右侧 `QStackedWidget` 的 Settings 页替换为 `QTSettingsPage` 实例。  
3. **尺寸控制**：通过 `tm_no_tab_auto_resize` 属性 + `resizeOthers` 早退逻辑，限制 tab 切换尺寸传播范围。  
4. **布局统一**：把居中责任收敛到顶层 `icon-tab`，减少子组件重复布局控制。
