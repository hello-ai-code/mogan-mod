# 216_35 File 页面新增快捷模板入口

## 如何测试

1. 编译：`xmake b stem`
2. 启动 Mogan，打开启动页 **File** 页面。
3. 确认显示 **3 个**卡片：New Blank Document、ElegantBook Notes Template、NSFC Young Scientists Fund。
4. 确认 ElegantBook 和 NSFC 卡片显示缩略图（首次可能显示 "Loading..."，随后加载完成）。
5. 点击 ElegantBook / NSFC 卡片，若本地无缓存应弹出下载进度条；下载完成后自动打开模板。再次点击应直接打开（无下载弹窗）。
6. 切换到 **Template** 页面，确认同一套模板列表加载正常，缩略图显示一致。
7. 重启 Mogan，再次进入 File 页面，确认缩略图优先从本地缓存读取，无重复网络请求。
8. 分别验证亮色/暗色主题下模板卡片边框、Hover 效果、缩略图背景色正常。
9. 切换中文，确认显示 "优美的笔记模板" 和 "国自然基金青年基金申请模板"。
10. 测试下载过程中点击 Cancel，确认无 crash。

## 2026/04/27 实现说明

### What

在 File 页面原有 "New Blank Document" 基础上，新增两个快捷模板入口（ElegantBook、NSFC），支持缩略图展示、点击下载/打开、本地缓存复用。同时提取 `ThumbnailLoader` 单例，供 FilePage 与 TemplatePage 共享缩略图加载逻辑。

#### 新增文件

**src/Mogan/Cache/thumbnail_loader.hpp / cpp**
- `ThumbnailLoader` 单例，全局管理缩略图加载队列。
- 集成 `ThumbnailCache`：优先读缓存，命中后若未验证则发 `If-None-Match` 条件请求；未缓存则直接下载。
- 全局 `validatedUrls_`：同一会话内每个 URL 仅验证一次。
- 全局并发限制 `MAX_CONCURRENT = 6`。
- 统一缩略图裁剪策略：`KeepAspectRatioByExpanding` + 居中裁剪。

#### 修改文件

**src/Plugins/Qt/qt_file_page.cpp / hpp**
- `DocStyle` 结构体扩展 `templateId` 字段，用于关联 `TemplateManager` 中的模板。
- `styles_` 从 1 项扩展为 3 项（generic / elegantbook / nsfc-ysf-c），模板名称、描述、缩略图 URL 均通过 `templateId` 从 `TemplateManager` 动态获取。
- `StyleCard` 新增 `isTemplate_` 标志和 `setupThumbnailMode()`：模板卡片显示缩略图 + 标题标签；空白文档卡片保持原有图标模式。
- `StyleCard::loadThumbnail()` 调用 `ThumbnailLoader::instance()->load()` 加载网络缩略图。
- 新增 `QtFilePage::refreshTemplateThumbnails()`，在 `showEvent` 中连接 `TemplateManager::templatesLoaded` 信号，确保模板元数据就绪后自动加载缩略图。
- `createDocumentFromTemplate()` 复用 `TemplateManager` 的下载逻辑：本地存在则直接打开，不存在则弹 `QProgressDialog` 下载。
- 修复下载对话框生命周期问题：`QProgressDialog*` 改为 `QPointer<QProgressDialog>`，显式保存并断开 `downloadProgress` 连接后再 `deleteLater`，避免下载完成或取消后访问悬空指针导致 crash。

**src/Plugins/Qt/qt_template_page.cpp / hpp**
- 移除本地 `ThumbnailRequest` 结构体、`loadThumbnail()` / `processThumbnailQueue()`（约 140 行）。
- 移除 `networkManager_`、`thumbnailQueue_`、`validatedUrls_`、`activeThumbnailRequests_` 等重复状态。
- `createTemplateCard()` 中缩略图加载改为 `ThumbnailLoader::instance()->load()`。

**TeXmacs/misc/themes/liii.css / liii-night.css**
- 新增 `QFrame#style-card-frame` 边框（亮色 `#cfcfcf` / 暗色 `#3c3c3c`）和 Hover 效果（`1px` `#2791ad`）。
- 新增 `QLabel#style-card-thumbnail` 背景色（亮色 `#f5f5f5` / 暗色 `#4a4f57`）。
- 新增 `QLabel#style-card-title` 背景色和文字色。
- 修复 `QLabel#startup-tab-template-thumbnail`：去掉 `border` 和 `[thumbnailLoaded="true"]` 动态状态切换，统一使用 `border-radius: 4px`；移除 C++ 中的 `applyThumbnailFrameStyle` `setStyleSheet` 覆盖。

**TeXmacs/plugins/lang/dic/en_US/zh_CN.scm**
- 新增 ElegantBook 和 NSFC 相关 4 条中文翻译。

### Why

1. **快速访问高频模板**：ElegantBook 和 NSFC 是用户使用频率最高的两个模板，放在 File 页面首屏可减少操作路径（无需切换到 Template 页面）。
2. **与 Template Center 数据同源**：`templateId` 映射到 `TemplateManager` 的模板元数据，缩略图 URL、下载 URL、版本更新检测全部复用现有基础设施，无需额外维护。
3. **提取公共加载器**：FilePage 和 TemplatePage 都需要网络缩略图加载，各自维护一套队列和缓存验证逻辑会导致代码冗余和状态不一致；`ThumbnailLoader` 单例统一了并发控制、ETag 验证和 `ThumbnailCache` 交互。
4. **生命周期安全**：`QProgressDialog` 在异步下载场景下容易被 `deleteLater` 后仍被 lambda 访问，使用 `QPointer` 和显式断开连接可以避免 segfault。

### How

- `DocStyle` 仅保留 `id`、`name`、`description`、`templateId`；`setupStyleCards` / `refreshTemplateThumbnails` 通过 `TemplateManager::templateById()` 动态解析 `thumbnailUrl`，再交给 `ThumbnailLoader` 加载到 `QLabel`。
- `ThumbnailLoader::load(label, url, size)` 内部查 `ThumbnailCache`：命中则立即显示 pixmap；若该 URL 本会话未验证，则入队发 `If-None-Match` 请求；未缓存则直接下载并写入缓存。
- `createDocumentFromTemplate` 调用 `TemplateManager::isTemplateAvailableLocally()` 判断本地缓存，不存在时通过 `TemplateManager::downloadTemplate()` 下载，进度条使用 `QPointer` 包装，`cleanup` lambda 中先 `disconnect(progressConn)` 再销毁对话框。
- CSS 通过 `#style-card-frame` / `#style-card-thumbnail` / `#style-card-title` 控制模板卡片视觉，C++ 不再硬编码样式。
