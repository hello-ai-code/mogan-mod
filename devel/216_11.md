# 216_11 启动标签页标题根据版本类型显示 Mogan STEM 或 Liii STEM

## 如何测试
1. **社区版测试**：构建社区版（`xmake config --is_community=true`），首次启动时观察启动标签页标题应显示为 "Mogan STEM"
2. **商业版测试**：构建商业版（`xmake config --is_community=false`），首次启动时观察启动标签页标题应显示为 "Liii STEM"
3. **切换主题测试**：在启动标签页打开的情况下，通过偏好设置切换主题（如从 Light 切换到 Dark），观察启动标签页标题是否保持正确显示，不应变回 URL 或 "startup-tab"
4. **标题持久性测试**：在多个标签页之间切换，确认启动标签页标题始终显示正确

## 2026/04/16 启动标签页标题正确性实现

### What
解决启动标签页标题在不同版本类型下显示不正确的问题，以及切换主题后标题显示为 URL 的问题：

- `new_window.cpp`：在创建启动标签页时，根据 `is_community_stem()` 设置正确的标题
- `tabpage-menu.scm`：在渲染标签页标题时，对启动标签页进行特殊处理，确保即使 buffer title 为空也能显示正确标题

#### 新增文件：

无新增文件

#### 修改文件：

**src/Texmacs/Data/new_window.cpp** (修改)
- 添加头文件包含：`#include "tm_sys_utils.hpp"`
- 在 `ensure_window()` 函数中修改启动标签页标题设置逻辑：
  - 原代码：`set_title_buffer (name, "Mogan STEM");`
  - 新代码：
    ```cpp
    string title= is_community_stem () ? "Mogan STEM" : "Liii STEM";
    set_title_buffer (name, title);
    ```

**TeXmacs/progs/texmacs/menus/tabpage-menu.scm** (修改)
- 在 `texmacs-tab-pages` 菜单定义中，添加启动标签页的特殊标题处理：
  ```scheme
  ;; 特殊处理启动标签页标题
  (title* (if (== (url->system buf) "tmfs://startup-tab")
              (if (community-stem?) "Mogan STEM" "Liii STEM")
              title*))
  ```
- 使用 `url->system` 将 URL 对象转换为字符串后进行比较
- 通过 `(community-stem?)` Scheme 函数检测版本类型

### Why
1. **版本区分**：社区版和商业版需要显示不同的产品名称，提升品牌识别度
2. **主题切换稳定性**：切换主题时会触发 UI 刷新，buffer 的 title 字段可能被重置，需要在显示层进行保护
3. **双层保护**：C++ 层确保数据正确性，Scheme 层确保显示正确性，形成防御性编程
4. **用户体验**：避免用户看到内部 URL（如 "tmfs://startup-tab"）或文件名（"startup-tab"）

### How
1. 在 C++ 层 `ensure_window()` 中，创建启动标签页 buffer 时根据 `is_community_stem()` 返回值设置对应标题
2. 在 Scheme 层 `texmacs-tab-pages` 中，每次渲染标签页时检测是否为启动标签页（通过比较 URL 字符串）
3. 如果是启动标签页，则使用 `(community-stem?)` 动态决定显示 "Mogan STEM" 或 "Liii STEM"
4. 保留原有的标题回退逻辑（当 title 为空时使用 URL 尾部），确保非启动标签页的行为不变
