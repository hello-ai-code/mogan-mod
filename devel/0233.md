# [200_63] 支持深度推理（Deep Reasoning）会话选项

## 如何测试

1. 新建 AI 聊天会话 → thinking 按钮默认未按下
2. 点击 thinking 按钮 → 按钮切换为按下状态，manifest.json 中对应会话 thinking 字段变为 "enabled"
3. 再次点击 → 按钮恢复，thinking 字段变为 "disabled"
4. 关闭会话后重新打开 → thinking 按钮状态正确恢复
5. 开启 thinking 发送消息 → 推理内容以灰色斜体折叠显示，然后显示正式回答

## 2026/05/27 会话级 thinking 开关

### What

1. `ChatSession` 数据结构新增 `bool thinking` 字段，默认 `false`
2. `ChatConversationPanel` 新增 thinking 切换按钮（QToolButton），位于输入框上方工具栏
3. 按钮 toggled 信号通过 `thinkingToggled` 信号传递给 `ChatController`
4. `ChatController::onThinkingToggled` 更新 C++ 侧状态并调用 Scheme 层 `chat-tab-session-set-thinking`
5. manifest 持久化读写支持 thinking 字段
6. 恢复会话时正确还原 thinking 按钮状态

### Why

Kimi k2.6 模型支持 thinking（推理）模式，需要在会话级别提供开关让用户控制。thinking 状态应与每个会话绑定并持久化到 manifest，以便重新打开会话时恢复。

### How

**C++ 数据层**：
- `qt_chat_session.hpp/cpp`：`ChatSession` 新增 `bool thinking` 字段，`setThinking`/`getThinking` 方法

**UI 层**：
- `qt_chat_tab_widget.hpp/cpp`：`ChatConversationPanel` 新增 `QToolButton* thinkingButton_`，`thinkingToggled` 信号，使用 `thinking.svg` 图标
- 按钮位于发送按钮左侧，checkable 模式

**控制器层**：
- `qt_chat_controller.hpp/cpp`：新增 `onThinkingToggled` 方法，连接信号，更新 manifest
- `restoreSessionMeta` 接收 thinking 参数，恢复按钮状态

**Scheme 持久化层**：
- `chat-session-persist.scm`：manifest 读写支持 thinking 字段
- `chat-tab-session.scm`：session state 扩展 thinking，`chat-tab-rounds->json` 将 thinking 写入 %context JSON

**Scheme LLM 插件层**：
- `llm-data.scm`：thinking 参数显式传给服务器（enabled/disabled），temperature 为 #f 时不传
- `llm-chat.scm`：从 %context 提取 thinking-override，传给 payload

**胶水层**：
- `glue_widget.lua`：`qt-chat-tab-restore-session` 从 6 参数扩展为 7 参数（新增 thinking）
