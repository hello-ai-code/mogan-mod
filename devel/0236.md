## 5 What

为普通 Session（非 Chat Tab）增加 reasoning 推理过程的支持，使普通 Session 能像 Chat Tab 一样展示和折叠 thinking 内容。

修改内容：
1. `session-edit.scm`：新增 6 个辅助函数：
   - `tree-contains-label?` - 递归检查 tree 是否包含指定 label
   - `tree-remove-label-from-children!` - 从 document/concat 子节点中移除指定 label
   - `tree-extract-reasoning-delta!` - 提取 reasoning-delta 文本并清除节点
   - `session-find-last-unfolded-explain` - 向前搜索最后一个 unfolded-explain
   - `session-append-reasoning!` - 追加 reasoning 文本到 unfolded-explain 的 document
   - `session-fold-last-explain!` - 折叠最后一个 unfolded-explain
2. `session-edit.scm`：修改 `session-notify` 的 `"output"` 分支，检测 `reasoning-delta` 和 `fold-explain-reasoning` 标记
3. `session-edit-test.scm`：新增回归测试，覆盖 4 个辅助函数（共 11 个测试用例）

## 6 Why

此前 reasoning 支持仅在 Chat Tab 中实现（0233），普通 Session 无法显示推理过程。用户需要在普通 Session 模式下也能查看 LLM 的 thinking 内容。


## 7 How

普通 Session 的 reasoning 处理逻辑与 Chat Tab 保持一致，但无需 `with-buffer` 包装（普通 Session 直接操作当前 buffer）：

1. LLM 插件通过 `flush-scheme-u8` 发送 `(document (reasoning-delta ,content))` 和 `(document (fold-explain-reasoning))` 标记（主分支已有）
2. `session-notify` 检测到 `reasoning-delta` 标记时：
   - 提取 reasoning 文本，从输出树中移除标记节点
   - 将剩余非 reasoning 内容正常输出
   - 将 reasoning 文本追加到 out 中最后一个 `unfolded-explain` 的 document 里（按换行分段）
   - 如有 `fold-explain-reasoning` 标记，折叠为 `folded-explain`
3. 仅包含 `fold-explain-reasoning` 时，直接折叠最后一个 `unfolded-explain`
