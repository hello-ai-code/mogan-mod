## 2026/04/15

### What

修复 HTML5 MathML 导入时，文字上下标或 `max/min` 下标被拆成多个 MathML sibling 后触发 `bad msub`、`bad msup`、`bad msubsup` 的问题。

本次涉及：

- `TeXmacs/plugins/html/progs/convert/html/htmltm.scm`
- `TeXmacs/plugins/html/progs/convert/mathml/mathtm.scm`
- `TeXmacs/plugins/html/progs/convert/mathml/mathtm-test.scm`

### Why

原来的 `mathtm-msub`、`mathtm-msup`、`mathtm-msubsup` 严格要求子节点数量分别为 2、2、3。规范 MathML 中脚标内容通常会被包进 `mrow`，例如：

``` xml
<msub>
  <mi>a</mi>
  <mrow>
    <mi>max</mi>
    <mo>(</mo>
    <mi>k</mi>
    <mo>)</mo>
  </mrow>
</msub>
```

但真实网页或 HTML5 MathML 导入中，脚标内容可能被拆成多个同级节点：

``` xml
<msub>
  <mi>a</mi>
  <mi>max</mi>
  <mo>(</mo>
  <mi>k</mi>
  <mo>)</mo>
</msub>
```

此时 `msub` 的内容长度大于 2，旧实现会直接返回红色错误文本 `bad msub`。

另外，HTML5 的 `<math>` 标签在 HTML 导入路径中会先走 `htmltm-math`。旧实现构造 `m:math` 时把整个子节点列表作为一个节点插入，导致 MathML 内容多包了一层 list，也会影响后续分发和转换。

### How

#### HTML5 MathML 入口修复

`htmltm-math` 中将 MathML 子节点展开插入：

``` scheme
`(m:math (@ ,@a) ,@(replace-nsprefix-in-stree c "h:" "m:"))
```

避免把 `c` 整体作为一个子节点塞进 `m:math`。

#### `msub` / `msup` 容错

`mathtm-msub` 和 `mathtm-msup` 不再要求刚好两个子节点，而是要求至少两个子节点：

- 第一个子节点作为 base
- 剩余所有子节点整体序列化为 sub 或 sup

这样 `a_{max(k)}` 被拆成 `a max ( k )` 时，也能得到一个完整的下标内容。

#### `msubsup` 容错

`mathtm-msubsup` 不再要求刚好三个子节点，而是要求至少三个子节点，并加入简单拆分规则：

- 如果只有两个脚标节点，保持原语义：第一个为 sub，第二个为 sup。
- 如果第二段脚标以 `(`、`[`、`{` 开头，并以 `)`、`]`、`}` 结尾，则把这段括号内容整体作为 sup。
- 否则默认将最后一个节点作为 sup，其余脚标节点作为 sub。

这可以覆盖常见的 `a_{max}^{(k)}`、`f_{max}^{[a,b]}` 等 HTML5 MathML 拆分形式。
