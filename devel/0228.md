## 5 What
在 Chat Tab 页面按搜索快捷键（macOS Cmd+F、Linux Ctrl+F、Emacs C-s 等）会导致 crash。原因是搜索系统对虚拟 `tmfs://chat-` buffer 执行了不兼容操作（如 `buffer-set-master`）。

1. 在 `interactive-search` 中添加 `unless` 守卫，检测当前 buffer 是否以 `tmfs://chat-` 开头
2. 在 `interactive-replace` 中添加同样的守卫
3. 使用 `unless` 包裹整个函数体，chat buffer 下直接跳过搜索/替换逻辑

## 6 Why
Chat Tab 使用虚拟 tmfs:// buffer（`tmfs://chat-tab`、`tmfs://chat-message-{id}`、`tmfs://chat-input-{id}`），这些 buffer 不支持完整的 buffer 操作。搜索功能调用 `buffer-set-master` 时对虚拟 buffer 触发 crash。

不同平台的搜索快捷键不同（macOS: Cmd+F、Linux: Ctrl+F、Emacs: C-s/C-r、其他: Find 键），但最终都调用 `interactive-search`。在 Scheme 层拦截比在 C++ 层逐个拦截快捷键更统一、更简洁。

## 7 How
在 `search-widgets.scm` 的 `interactive-search` 和 `interactive-replace` 函数中，用 `unless` 守卫包裹函数体：

```scheme
(tm-define (interactive-search)
  (:interactive #t)
  (unless (string-starts? (url->system (current-buffer)) "tmfs://chat-")
    ;; ... 原有搜索逻辑 ...
  ) ;unless
) ;tm-define
```

当 `(current-buffer)` 返回以 `tmfs://chat-` 开头的 URL 时，整个搜索/替换流程被跳过，避免对虚拟 buffer 执行不兼容操作。
