# [0208] Chat Tab Sidebar Session 搜索匹配功能

## 如何测试

### 编译
```bash
xmake b stem
```

### 功能测试

#### 基本搜索过滤（Active Sessions）
1. 启动 Mogan STEM，打开 Chat Tab
2. 创建 3-5 个会话，分别发送不同内容使标题自动生成（如 "Python 问题"、"JavaScript 问题"、"代码 review"）
3. 在左侧 sidebar 的搜索框中输入 "Python"
4. 观察：只有标题包含 "Python" 的会话显示在 Active Sessions 列表中
5. 观察：Conversations (N) 的计数变为匹配的数量
6. 清空搜索框 → 所有会话重新显示，计数恢复总数

#### 搜索过滤（Archived Sessions）
1. 先归档 2-3 个会话（右键 → 归档）
2. 展开归档区（点击 "Archived (N)"）
3. 在搜索框中输入归档会话的标题关键字
4. 观察：归档区也只显示匹配的归档会话
5. Archived (N) 计数随过滤结果实时变化

#### 大小写不敏感匹配
1. 在搜索框输入 "python"（小写）
2. 标题为 "Python 问题" 的会话应被匹配并显示
3. 输入 "PYT" 也应匹配 "Python"

#### 中文搜索
1. 创建标题含中文的会话（如 "算法讨论"、"前端方案"）
2. 搜索框输入 "算法" → 只显示 "算法讨论"
3. 搜索框输入 "前端" → 只显示 "前端方案"

#### 清除按钮
1. 在搜索框输入任意文字
2. 点击搜索框右侧的 "x" 清除按钮
3. 搜索框清空，所有会话重新显示

#### 新会话的默认标题搜索
1. 创建一个新会话但不发送消息（标题显示 "新会话"）
2. 搜索框输入 "新会话" → 该会话应被匹配
3. 搜索框输入 "不存在" → 该会话应被隐藏

### 边界测试
- 搜索框为空 → 显示所有会话，计数为总数
- 输入不匹配任何会话的关键字 → Active Sessions 和 Archived Sessions 均为空，计数为 0
- 删除会话时带有搜索过滤 → 删除后刷新 sidebar，过滤条件仍然生效
- 归档/恢复会话时带有搜索过滤 → 操作后刷新 sidebar，过滤条件仍然生效
- 重命名会话时带有搜索过滤 → 若新标题不匹配过滤条件，重命名后该会话立即隐藏
- 快速连续输入搜索关键字 → 每次输入变化都触发刷新，无延迟或卡顿

## How

### 1. UI 布局

在 `setup_left_sidebar()` 中，于 `conversationCountLabel_`（"Conversations (N)" 标签）和 `multiSelectBar_`（多选操作栏）之间插入一个 `QLineEdit` 搜索框：

```
Chat
[New chat]
Conversations (N)
[搜索会话...          x]   ← 新增搜索框
[取消] [全选] [归档] [删除]  ← 多选操作栏（隐藏）
[会话 1]
[会话 2]
...
Archived (M)
[归档会话 1]
...
```

搜索框样式：
- 边框：`1px solid #cccccc`，圆角 `4px`
- 背景：`#ffffff`
- 字体大小：与收缩按钮一致（`kCollapseFontPx`）
- 内边距：与收缩按钮一致
- 清除按钮：启用 `setClearButtonEnabled(true)`
- 焦点策略：`Qt::ClickFocus`（避免抢占编辑器焦点）

### 2. 实时过滤机制

连接 `QLineEdit::textChanged` 信号到 `refresh_sidebar()`：

```cpp
connect (searchEdit, &QLineEdit::textChanged, this,
         [this] () { refresh_sidebar (); });
```

每次输入变化即时触发 sidebar 刷新，无需回车确认。

### 3. 过滤逻辑

在 `refresh_sidebar()` 中：

1. **获取过滤文本**：将搜索框文本转小写存储为 `filterText`
2. **标题去重**：仍基于**全部会话**计算 `titleCounts`，保证序号稳定性（过滤不改变去重序号）
3. **遍历匹配**：对每个会话计算 `displayText`（含去重序号后的显示文本），执行大小写不敏感包含匹配：
   ```cpp
   bool matchesFilter= filterText.isEmpty () ||
                       displayText.toLower ().contains (filterText);
   ```
4. **不匹配处理**：
   - 调用 `panel->itemWidget->setVisible(false)` 隐藏整个条目
   - `continue` 跳过，不加入任何列表布局
5. **匹配处理**：
   - 显示条目，加入对应布局（active 或 archive）
   - 根据归档状态分别增加 `activeCount` 或 `archivedCount`
6. **计数更新**：循环结束后更新 `Conversations (N)` 和 `Archived (M)` 标签

### 4. 计数显示策略

搜索过滤时，计数标签显示的是**过滤后的数量**而非总数：

- 空搜索框：`Conversations (5)` / `Archived (3)`
- 输入 "Python"：`Conversations (1)` / `Archived (0)`

这样用户可以直观看到匹配结果的数量。

### 5. 与其他功能的共存

- **多选模式**：搜索过滤不影响多选模式的内部状态。隐藏的会话其 checkbox 仍可能被 "全选" 选中（因为只是视觉隐藏）。这是有意为之——过滤只影响显示，不影响操作语义。
- **归档/恢复/删除/重命名**：这些操作都会调用 `refresh_sidebar()`，因此搜索过滤条件在操作后仍然生效。
- **持久化加载**：加载持久化会话后调用 `refresh_sidebar()`，若搜索框已有内容，新加载的会话也会被过滤。
