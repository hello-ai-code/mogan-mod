## 5 What
修复 Debian 13 CI 中 xmake 在安装 Qt 时自行编译 Python 3.14 的问题。

1. 在 apt 安装列表中增加 `python3.13-venv` 和 `python3-pip` 包
2. 通过 `update-alternatives` 将 `python3` 指向 `python3.13`，使 xmake 优先使用系统 Python
3. 预先通过 `pip install aqtinstall` 安装 aqtinstall，避免 xmake 因找不到而自行编译 Python

## 6 Why
Debian 13 CI 在运行 `xmake config` 安装 Qt 依赖时，xmake 会调用 aqtinstall。由于系统未安装 pip 且 `python3` 命令未绑定到已安装的 Python 3.13，xmake 会自行下载并编译 Python 3.14 源码来满足 aqtinstall 的需求。从源码编译 Python 耗时极长（约 20+ 分钟），且常常因缺少编译依赖而失败，导致 CI 超时或报错。

## 7 How
- `update-alternatives --install /usr/bin/python3 python3 /usr/bin/python3.13 1`：让 `python3` 命令指向系统已安装的 3.13，xmake 检测到可用 Python 后不再自行编译
- `python3.13-venv`：提供 venv 模块，部分 pip/xmake 工作流需要
- `python3-pip`：提供 pip 基础设施
- `pip install --break-system-packages aqtinstall`：提前安装 aqtinstall，xmake 检测到已安装后直接使用，跳过 Python 编译步骤
