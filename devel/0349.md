## 5 What
在魔法粘贴（Ctrl+Shift+V）和选择性粘贴（Ctrl+Alt+V）操作前调用后端 API 检查使用次数，非会员每日限制 10 次。

1. 新增 C++ 函数 `check_magic_paste()`，使用 QNetworkAccessManager + QEventLoop 同步 POST `/api/v1/oauth2/magicPaste/check`，3 秒超时，返回状态字符串
2. 新增 Scheme 函数 `with-magic-paste-check`（continuation 风格），统一处理 API 检查结果和用户提示
3. 改造 `kbd-magic-paste`：OCR 分支不检查，非 OCR 分支走次数限制
4. 改造 `interactive-paste-special`：image/verbatim/internal 分支不检查，其余分支走次数限制
5. 添加 `zh_CN.scm` 中文翻译

## 6 Why
需要对非会员用户的魔法粘贴和选择性粘贴功能进行每日使用次数限制，引导用户升级会员。社区版不受限。

## 7 How
C++ 层使用 `QEventLoop` 将异步 HTTP POST 转为同步调用，避免侵入 Scheme 的同步执行模型。根据 HTTP 状态码返回不同结果字符串（200→allowed, 401→not-logged-in, 403→limit-exceeded, 超时→error:timeout, 其他→error:状态码）。

Scheme 层通过 `with-magic-paste-check` 封装检查逻辑，采用 continuation 风格（接受 thunk 参数），未登录时弹出 user-ask 对话框引导登录，超限时引导升级，网络错误显示错误码。
