## 5 What

1. 新增 `chat-tab-buffer-empty?` 函数，判断 message buffer 是否为空（无对话历史）
2. 新增 `chat-tab-tree->plain-text` 函数，将 TeXmacs 树转为纯文本（处理 document、concat、new-line、with、folded-explain 等节点类型）
3. 新增 `chat-tab-join-nonempty` 辅助函数，用分隔符连接非空字符串列表
4. 新增 `chat-tab-find-session` 函数，在 body 树中查找 session 节点（处理 TMU 加载后 body 为 document 包裹的情况）
5. 新增 `chat-tab-extract-rounds` 函数，从 message buffer 提取所有已完成对话轮次的 `(role . text)` 列表
6. 新增 `chat-tab-rounds->json` 函数，将轮次列表转为 JSON 字符串（使用 `(liii njson)` 库）
7. 新增 `chat-tab-build-context-input` 函数，构建 `%context` 命令树
8. 修改 `chat-tab-session-feed`，在发送前将原始输入替换为包含完整上下文的输入
9. 修改 session 初始化逻辑，仅在空缓冲时创建新 session 结构（避免覆盖已有历史）
10. 修改 `chat-tab-message-document`，支持 body 为 document 包裹 session 的情况

## 6 Why

Chat Tab 之前每次发送消息时只传递当前用户输入，不包含历史对话记录。LLM 无法看到之前的对话内容，导致无法进行连贯的多轮对话。需要将完整对话历史打包为 `%context` 命令传给底层 LLM 插件。

此外，从已保存的 tmu 文件加载 Chat Tab 时，body 结构可能与新建时不同（document 包裹 session），原有代码无法正确处理这种情况。

## 7 How

### 数据流

```
用户输入 → chat-tab-session-feed
         → chat-tab-build-context-input
           → chat-tab-extract-rounds（提取历史轮次）
           → chat-tab-rounds->json（转为 JSON）
           → 拼接 "%context " + JSON
         → 发送给 LLM 插件
```

### 关键设计

1. **树转纯文本**：`chat-tab-tree->plain-text` 递归处理 TeXmacs 树，将 document、concat、with 等结构节点展开为纯文本字符串，使用 `cork->utf8` 确保 Unicode（中文等）正确编码

2. **轮次提取**：`chat-tab-extract-rounds` 遍历 session document 的子节点，识别 `unfolded-io-text` 结构，分别提取用户输入和助手回复

3. **njson 构造**：使用 `(liii njson)` 库构建 JSON 对象和数组，避免手工拼接字符串

4. **空缓冲判断**：`chat-tab-buffer-empty?` 检查多种空状态（nil、空 document、仅含空节点的 document、空 session），确保只在真正为空时初始化 session 结构

5. **session 查找**：`chat-tab-find-session` 处理 TMU 加载后 body 可能为 `(document (session ...))` 的情况，在 document 子节点中搜索 session
