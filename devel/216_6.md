# 216_6 启动页 File 页视觉与 Recent 读写链路完善

## 如何测试
1. 编译时先输入 `xmake config -vD --startup_tab=true`
2. 启动 Mogan STEM，进入 `Startup -> File`
3. 验证样式卡片区：
   - `Generic / Beamer / Book / Exam / Letter / Article` 卡片可见
   - 悬停切换选中边框正常，`Default` 徽标显示正常
4. 验证 Recent 区：
   - 列表为空时显示 `No recent documents`
   - 打开已有文档后，Recent 立刻出现条目（文件名+路径两行）
   - 点击 recent 条目可打开文档并刷新其最近时间顺序
   - 右键 `Remove from list` 可移除条目
5. 验证跨入口行为：
   - 从 `Template` 页面打开模板后，recent 自动增加对应文档
   - 从 `Open a Document` 入口打开文档后，应用可稳定打开文档（无 crash）
6. 切换亮色/暗色主题，确认 File 页卡片、recent 列表、分隔线样式均正常

## 2026/04/10 启动页 File 页分支实现

### What
本分支围绕启动页 `File` 页面完成三类工作：  
1) 视觉样式升级（亮/暗主题补全）；  
2) Recent 文档读写链路从“占位”改为“与全局 recent 状态联动”；  
3) 从 Template / Open a Document 进入文档时，统一补齐 recent 同步行为。

#### 修改文件：

**TeXmacs/progs/startup-tab/startup-tab-file.scm** (修改)
- 引入 `(kernel texmacs tm-dialogue)`，接入全局 recent 相关能力。
- `startup-tab-get-recent-docs` 改为读取 `learned-interactive 'recent-buffer`。
- `startup-tab-add-recent-doc` 改为通过 `learn-interactive` 写入 recent-buffer。
- `startup-tab-clear-recent-doc` 改为调用 `recent-files-remove-by-path`。

**src/Plugins/Qt/qt_file_page.hpp** (修改)
- `addRecentDoc(const QString& path)` 提升为 public，供启动页其他入口复用。
- 新增 `renderRecentDocs()`，将“读取数据”和“渲染列表”拆分，提升职责清晰度。

**src/Plugins/Qt/qt_file_page.cpp** (修改，核心)
- Recent 存储路径对齐到 `TEXMACS_HOME_PATH/system/recent-files.json`。
- 读取兼容两种结构：旧版 `recent_documents` 与新版 `files`（含 `show/last_open/open_count`）。
- 保存策略升级：在保留历史元数据基础上更新可见项，并限制总量（`MAX_GLOBAL_RECENT_DOCS`）。
- `refreshRecentDocs()` 改为统一走 `loadRecentDocs()`，并由 `renderRecentDocs()` 专责渲染。
- Recent 列表项改为双行自定义行组件（名称 + 路径），并增加 DPI 友好的尺寸常量。
- 点击 recent 项时先 `addRecentDoc(path)`，再 `load-document`，确保打开行为会刷新 recent 顺序。
- `addRecentDoc/removeRecentDoc` 在 Qt 本地更新后，同步调用 Scheme：
  - `(startup-tab-add-recent-doc "...")`
  - `(startup-tab-clear-recent-doc "...")`
- 样式卡片与 Recent 区字体/间距统一改为 DpiUtils 常量化，减少硬编码尺寸。

**src/Plugins/Qt/qt_startup_tab_widget.cpp** (修改)
- Template 打开文档信号处理改为捕获 `this`，并在加载文档前调用 `filePage_->addRecentDoc(filePath)`。
- Open a Document 入口保持稳定优先：当前版本不引入额外时序刷新钩子，避免异步链路带来的不确定行为。

**TeXmacs/misc/themes/liii.css** (修改)
- 新增 Startup File 页样式：分隔线、分区标题、recent 列表、卡片边框、默认徽标、选中态等。

**TeXmacs/misc/themes/liii-night.css** (修改)
- 新增与亮色主题对应的暗色版 Startup File 页样式，保证两套主题一致体验。

### Why
1. 原有 startup-tab-file 的 recent 逻辑存在占位实现，无法与全局 recent 数据源一致。  
2. File 页 UI 结构已成型，但缺少完整主题样式和 DPI 细节，视觉一致性不足。  
3. 文档可能从多个入口打开（Recent/Template/Open a Document），需要统一 recent 更新策略，避免入口间行为不一致。  

### How
1. **数据层**：Scheme recent API 与 Qt recent 文件读写双向联动，保证状态一致。  
2. **渲染层**：拆分 `load`/`render`，让解析与绘制解耦，便于后续维护。  
3. **交互层**：在 Template/Open a Document/Recent 点击三条路径上统一补 recent 更新。  
4. **样式层**：亮色与暗色主题同步补齐 Startup File 页组件样式。
