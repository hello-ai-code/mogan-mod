## 5 What

优化 Chat Tab 的整体 UI 布局和交互体验。

1. **左侧边栏滚动**：会话列表和归档区放入 QScrollArea，支持滚动，隐藏滚动条
2. **去掉内容区缩进**：去掉左右 24px 边距，内容区占满宽度
3. **输入框和输出区布局重构**：
   - 移除垂直 QSplitter，改用 QVBoxLayout
   - 输出区占满剩余空间（stretch=1）
   - 输入区位于底部，高度自适应（stretch=0）
4. **输入框 UI 优化**：
   - 添加 send.svg 图标替换文本按钮
   - 发送按钮移入输入框内部右下角
   - 输入框边框改为无边框纯白背景
   - 输入框内 texmacs 控件 viewport 背景设为白色，去掉灰色边缘
   - 输入框滚动条美化（6px 细圆角滚动条）
5. **消息输出区 UI**：无边框、透明背景、100% 宽度
6. **sidebarButton 宽度优化**：水平内边距从 12px 改为 8px
7. **右键多选菜单修复**：只建一个菜单、只 exec 一次

## 6 Why

1. 左侧会话过多时会挤压底部按钮，无法滚动
2. 输入框和输出区用 QSplitter 可手动调节，不符合聊天界面习惯
3. texmacs 控件默认 viewport 灰色背景导致输入框左右出现灰边
4. 发送按钮用文本不够美观，且位置在输入框外部
5. 右键多选菜单会连续弹出两次（先归档、后删除各一次 exec）

## 7 How

1. 用 QScrollArea 包装会话列表和归档区，设置 `setVerticalScrollBarPolicy(Qt::ScrollBarAlwaysOff)` 隐藏滚动条但保留滚动功能
2. 去掉垂直 QSplitter，将 messageFrame 和 inputArea 直接放入 QVBoxLayout，分别设置 stretch=1 和 stretch=0
3. 通过 `findChildren<QAbstractScrollArea>` 找到 texmacs 内部滚动区，给 viewport 设置白色背景
4. send.svg 放入 `TeXmacs/misc/images/llm-chat/send.svg` 并在 images.qrc 注册，用 `QIcon(":llm-chat/send.svg")` 加载
5. 多选菜单改为一次性 addAction 所有选项，只 exec 一次，通过 text() 判断选中项
