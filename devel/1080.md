## 5 What

移除 ChatSidebar 的 `sessionCache_` 成员，消除 Model 影子副本：
1. `addItem` 签名改为接收 `SessionDisplayInfo`，移除内部缓存写入
2. `updateItemTitle` 移除 sessionCache_ 同步循环
3. `moveToArchive` / `moveFromArchive` 移除 sessionCache_ 同步循环
4. `removeItem` 移除 sessionCache_ 同步循环
5. 构造函数移除 `sessionCache_ = sessions`
6. selectAll 按钮改用 `SidebarItem::isArchived` 判断
7. "..." 菜单改用 `SidebarItem::isArchived` 获取归档状态
8. Controller 的 `registerSession` 适配新 `addItem` 签名
9. 新增 15 个 Sidebar 单元测试覆盖 addItem/removeItem/moveToArchive/moveFromArchive/updateCountLabels/exitMultiSelectMode

## 6 Why

Sidebar 自行维护 `QList<SessionDisplayInfo> sessionCache_`，与 Model 状态双源。每次业务状态变更（归档、恢复、删除、重命名）都需要手动同步缓存，容易遗漏。`SidebarItem` 已包含 `isArchived` 字段，`items_`（QMap）可遍历获取全部信息，sessionCache_ 是冗余的。

## 7 How

### 单源状态
移除 sessionCache_ 后，Sidebar 只维护 UI 状态（SidebarItem::isArchived）。业务状态的唯一来源是 Controller 通过方法调用（moveToArchive/moveFromArchive 等）传入的指令，Sidebar 执行后更新本地 UI 状态即可。

### addItem 接口变更
从 `(string, string)` 改为 `(SessionDisplayInfo)`，与构造函数的接口风格一致。Controller 在 registerSession 中构造完整的 SessionDisplayInfo 传入。

### "..." 菜单和 selectAll
从 items_ 查找 SidebarItem 读取 isArchived，替代遍历 sessionCache_。代码更简洁，且 items_ 查找是 O(1) 的 QMap 查找，优于原来的 O(n) 线性遍历。
