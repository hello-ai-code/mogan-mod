# 216_9 非当前标签关闭刷新 + 针对启动页的标签页宽度 DPI 适配

## 如何测试
1. **在启动页关闭非当前标签页，确认标签栏会立即刷新，不会残留刚关闭的标签**
2. 在不同 DPI 屏幕（100%、125%、150%、200%）下启动 Mogan STEM
3. 打开多个标签页，观察普通标签页宽度是否随 DPI 正常缩放
4. 验证普通标签页最大宽度限制是否生效（宽屏、少标签情况下）
5. 验证 startup tab 宽度是否受到独立上限限制，不会过宽
6. 在标签很多时验证最小宽度下限是否仍可用，避免布局异常

## 2026/04/14 标签页宽度缩放与标签栏刷新实现

### What
将标签页宽度缩放与非当前标签关闭后的标签栏刷新问题一起处理：
- `QTMTabPage.cpp`：宽度常量改为基于 `DpiUtils` 计算，避免固定像素在高 DPI 下显示不一致。
- `new_view.cpp`：补充非当前标签关闭后的强制刷新逻辑，保证标签栏立即更新。

#### 新增文件：

无新增文件

#### 修改文件：

**src/Plugins/Qt/QTMTabPage.cpp** (修改)
- 将标签页宽度常量改为 96 DPI 基准值：
  - `MAX_TAB_PAGE_WIDTH_BASE = 150`
  - `STARTUP_TAB_MAX_WIDTH_BASE = 100`
- 新增缩放函数：
  - `getScaledMaxTabPageWidth()`
  - `getScaledStartupTabMaxWidth()`
- 标签页宽度计算改为使用 DPI 缩放值：
  - 普通标签宽度上限使用 `getScaledMaxTabPageWidth()`
  - startup tab 上限使用 `getScaledStartupTabMaxWidth()`
- 保留最小宽度下限逻辑，并建议后续统一为 `DpiUtils::scaled(25)` 形式

**src/Texmacs/Data/new_view.cpp** (修改)
- 在 `kill_tabpage()` 中新增 `refresh_tabbar_for_non_current` 标记，识别“关闭的是否为当前标签”
- 当关闭非当前标签时，针对同一 tabpage window 的当前 editor 执行 `resume -> suspend -> resume`
- 通过该流程触发 UI 立即刷新，避免标签栏延迟更新

### Why
1. **高 DPI 一致性**：固定像素在不同缩放比例下视觉尺寸不一致，需统一走 DPI 工具。
2. **布局稳定性**：普通标签与 startup tab 需要不同的上限约束，避免 startup tab 占用过多空间。
3. **交互一致性**：关闭非当前标签时也应立即反映到标签栏，避免“已关闭标签短暂残留”。
4. **维护性提升**：以 base 常量 + 缩放函数组织宽度参数，后续调参更清晰。

### How
1. 将宽度参数定义为 96 DPI 基准常量。
2. 通过 `DpiUtils::scaled()` 在运行时计算实际像素宽度。
3. 在 `arrangeTabPages()` 中分层应用约束：
   - 先对所有标签应用通用上下限；
   - 再对 startup tab 应用专用最大宽度限制。
4. 在 `kill_tabpage()` 中针对“关闭非当前标签”补充一次 editor 状态切换，主动触发 tab bar 重绘。
