## 如何测试

1. 启动 Mogan STEM，点击顶部工具栏的 AI 图标（LLM 按钮）
2. 观察标签栏是否出现固定位置的 "Chat" 标签页
3. 确认侧边栏显示 `Conversations (1)`，列表中存在初始会话项 `Chat 1`
4. 点击 `New chat` 两次，确认侧边栏数量递增为 `Conversations (3)`，列表中依次出现 `Chat 1`、`Chat 2`、`Chat 3`
5. 在 `Chat 1` 输入消息并发送，确认右侧进入会话态并显示对应对话
6. 切换到 `Chat 2`，确认右侧恢复为独立的空会话，不继承 `Chat 1` 的消息历史
7. 在 `Chat 2` 输入另一条消息并发送，确认回复只追加到 `Chat 2` 自己的消息区
8. 在 `Chat 1` 与 `Chat 2` 之间来回切换，确认各自消息历史和输入区草稿彼此独立
9. 点击 Chat 页面输入区，使焦点落在嵌入的 `texmacs_input_widget` 上，并输入任意内容
10. 不切换到其他标签页，直接点击 Chat 标签页关闭按钮，确认第一次点击就立即关闭
11. 再次点击 AI 图标重新打开 Chat 标签页，确认仍可正常打开
12. 确认启动标签页和 Chat 标签页可以正常切换
13. 确认普通文档标签页关闭行为不受影响
14. 确认 Chat 页面左侧有浅灰色侧边栏，顶部有 "Chat" 标题和"收缩"按钮，"New chat" 按钮位于其下
15. 确认 Chat 页面右侧显示当前激活会话对应的欢迎态或会话态界面
16. 确认输入区可正常输入、编辑 TeXmacs 内容（公式、结构化文本等）

## 2026/05/18 恢复 Chat 会话从 message list 到 session/tree-pointer 的改造

### What

- 恢复 `chat-tab-session.scm` 中以 `message-buffer` document tree 为真源的会话实现
- 发送时重新直接向消息区追加 `User:` / `Assistant:` 结构和 `(output (document ""))` 占位节点
- 恢复基于 `pending-ref + tree-pointer` 的 `output` / `error` / `cancel` 回写路径
- 删除误恢复回来的 `chat-tab-session-clear-input`、`chat-tab-session-append-user-message`、`chat-tab-session-append-assistant-message` 旧入口

### Why

- 旧的 message-list 实现会维护第二份消息真源，再整体重绘 `message-buffer`，已经偏离当前多会话 Chat 页的 buffer 驱动设计
- 在单页多会话场景下，插件异步回包需要直接命中当前轮次的 `output` 节点，不能再靠“替换最后一条 assistant message”这种重渲染语义兜底
- 这次是恢复之前已验证过的改造，避免误撤销后又回到旧模型

### How

- `TeXmacs/progs/dynamic/chat-tab-session.scm`：
  - 去掉 `messages` 状态、`chat-tab-session-owners` 和整套 `render-state!` / `append-message!` / `replace-last-assistant!` 逻辑
  - 恢复 `chat-tab-message-document`、`chat-tab-output`、`chat-tab-errput`、`chat-tab-append-round!`
  - 恢复 `chat-tab-session-encode` / `decode` / `detach` 的 `tree-pointer` 路由
  - 恢复 `chat-tab-session-feed`，并让 `notify` / `next` / `cancel` 直接回写当前轮次的 `output` tree

### What

### How

  - 将右侧内容区重构为 `QStackedWidget` 管理的多会话面板

- `TeXmacs/progs/dynamic/chat-adapter.scm`：
  - 只保留 `open-llm-chat-tab` 和 `chat-tab-send`
  - 删除未使用的 `chat-tab-clear-input` / `chat-tab-append-*` 公开 wrapper

## 2026/05/15 修正 Chat Tab 嵌入焦点下的关闭判定

### What

- 在 `kill_tabpage()` 中补充 Chat tab 的当前页判定分支
- 当 Chat 页输入框或消息区这类嵌入 `texmacs_input_widget` 持有焦点时，关闭外层 Chat 标签页仍按“关闭当前 tab”处理

### Why

- Chat tab 的输入区和消息区是独立的嵌入 editor view，获得焦点后会把全局 `current_view` 切到嵌入 view
- 原有关闭逻辑仅通过 `get_current_view() == u` 判断是否关闭当前标签页，导致 Chat tab 在输入区有焦点时被误判为“关闭非当前 tab”
- 误判后不会走当前 tab 的 detach/attach 路径，因此表现为第一次点击关闭只触发状态变化，没有立即关闭

### How

- `src/Texmacs/Data/new_view.cpp`：
  - 新增 `is_chat_tab_buffer()`，按 `tmfs://chat-tab` 前缀识别 Chat 顶层 buffer
  - 在 `kill_tabpage()` 中先使用 `get_current_view_safe()` 判断当前 view
  - 当目标 tab 是 Chat tab 且当前 view 不是外层 view 时，额外检查当前 editor 的 `mvw` 是否回指目标 Chat tab view
  - 若命中该分支，则将本次关闭视为“关闭当前 tab”，沿用原有 detach/attach/kill window 路径
