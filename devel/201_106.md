## 2026/04/10 重构悬浮框系统

### What

#### 1. 提取 QTMBasePopup 基类

新增文件：
- `src/Plugins/Qt/QTMBasePopup.hpp`
- `src/Plugins/Qt/QTMBasePopup.cpp`

将图片和文本悬浮框的公共逻辑抽取到基类：
- 位置缓存和计算 (`cachePosition`, `getCachedPosition`)
- 悬浮框位置更新 (`updatePosition`)
- 滚动时位置调整 (`scrollBy`)
- 按钮大小自动调整 (`autoSize`，支持 DPI 缩放)
- 阴影效果初始化 (`initCommonUI`)
- 选区可见性检查 (`selectionInView`)

#### 2. QTMTextToolbar 重命名为 QTMTextPopup

重命名文件：
- `QTMTextToolbar.cpp/hpp` → `QTMTextPopup.cpp/hpp`

统一命名规范：
- `show_text_toolbar()` → `show_text_popup()`
- `hide_text_toolbar()` → `hide_text_popup()`
- `is_point_in_text_toolbar()` → `is_point_in_text_popup()`
- `update_text_toolbar()` → `update_text_popup()`

测试文件同步重命名：
- `text_toolbar_test.cpp` → `text_popup_test.cpp`

#### 3. QTMImagePopup 和 QTMTextPopup 继承 QTMBasePopup

两个类现在都继承自 `QTMBasePopup`：
```cpp
class QTMImagePopup : public QTMBasePopup {
    // 只需实现 showPopup 和特定按钮逻辑
};

class QTMTextPopup : public QTMBasePopup {
    // 只需实现 showPopup 和特定按钮逻辑
};
```

#### 4. 扩大图片悬浮框识别区域 10%

修改 `edit_mouse.cpp`：
- 添加 `hover_image_rect` 和 `hover_image_path` 成员变量
- 鼠标在图片上时缓存区域和路径
- 鼠标移出图片但在扩大区域内时继续使用缓存显示悬浮框

### Why

#### 代码结构问题
- 原代码中 `QTMImagePopup` 和 `QTMTextToolbar` 有大量重复逻辑
- 位置计算、阴影效果、DPI 缩放处理等代码分散且难以维护

#### 命名不一致
- 图片悬浮框叫 `ImagePopup`，文本悬浮框叫 `TextToolbar`
- 命名混乱导致代码难以理解和维护

#### 用户体验问题
- 图片悬浮框在鼠标稍微移出边缘时就消失，操作不便
- 需要扩大识别区域提升体验

### How

#### 基类设计

```cpp
class QTMBasePopup : public QWidget {
protected:
    rectangle cached_rect;      // 选区缓存
    double    cached_magf;      // 缩放因子缓存
    int       cached_scroll_x, cached_scroll_y;  // 滚动位置缓存
    int       cached_canvas_x, cached_canvas_y;  // 画布位置缓存
    int       cached_width, cached_height;       // 悬浮框尺寸缓存
    
public:
    virtual void showPopup(qt_renderer_rep* ren, rectangle selr, 
                          double magf, int scroll_x, int scroll_y,
                          int canvas_x, int canvas_y) = 0;
    
    void updatePosition(qt_renderer_rep* ren);   // 更新位置（通用实现）
    void autoSize();                              // 调整按钮大小（通用实现）
    bool selectionInView() const;                 // 检查选区可见性
};
```

#### 扩大识别区域实现

```cpp
// 鼠标在图片上时：扩大检测并缓存
if (is_func(current_tree, IMAGE)) {
    selr = ...;  // 获取图片区域
    SI w = selr->x2 - selr->x1;
    SI h = selr->y2 - selr->y1;
    SI margin_x = (SI) (w * 0.1);  // 10% 边距
    SI margin_y = (SI) (h * 0.1);
    
    if (鼠标在扩大区域内) {
        hovering_image = true;
        hover_image_rect = selr;      // 缓存区域
        hover_image_path = current_path;  // 缓存路径
    }
}
// 鼠标移出图片但在扩大区域内：使用缓存
else if (!is_zero(hover_image_rect)) {
    // 检查是否在扩大区域内
    if (在扩大区域内) {
        hovering_image = true;
        current_path = hover_image_path;  // 恢复路径
        selr = hover_image_rect;          // 恢复区域
    } else {
        // 清空缓存
        hover_image_rect = rectangle(0, 0, 0, 0);
        hover_image_path = path();
    }
}
```

### 注意事项

1. **向后兼容**: 所有函数重命名都在 C++ 层完成，对外部 Scheme 代码无影响
2. **状态隔离**: `hover_image_rect` 独立于 `last_image_brec`，确保 resize handles 不受影响
3. **性能**: 使用缓存避免重复计算图片路径和区域
4. **清理**: 鼠标移出扩大区域时及时清空缓存
