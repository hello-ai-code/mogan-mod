## 5 What

1. 将 `chat-tab-session.scm`（1300行）拆分为四个职责清晰的模块：
   - `chat-protocol.scm`：会话协议层（回调、编码解码、发送、桥接、buffer 类型检测）
   - `chat-tree-ops.scm`：无状态树操作（文档处理、推理渲染、输出渲染）
   - `chat-persist.scm`：持久化（manifest 管理、buffer 导出/导入）
   - `chat-loader.scm`：模块加载入口（不含 import，解决 S7 环境冲突）
2. C++ 接管会话状态（model/thinking），移除 Scheme 侧 ahash-table 状态管理
3. 协议简化为单轮 `%chat`，移除多轮 `%context` 上下文构建
4. Plugin session ID 简化为 `chat-tab:{sid}`，不再包含 model 前缀
5. 持久化改用 `buffer-export` 直接写文件，移除 tmfs handler 机制
6. `chat-tab-init-session!` 改为从 C++ `getOrCreatePanel` 调用，移除幂等保护
7. Buffer 类型检测统一用 `chat-input-buffer?` / `chat-message-buffer?`（`tm-define` 导出），替换散落的 `string-ends?` 检查
8. 删除冗余的 `chat-persist-save-one`（C++ 已分别调用 `exportBuffer` + `updateManifest`）
9. 删除 C++ 侧未使用的 `sessionIdFromUrl` 函数及相关测试

## 6 Why

`chat-tab-session.scm` 是一个 1300 行的单体模块，混合了协议逻辑、树操作、持久化和状态管理。拆分后每个模块职责单一，便于维护和测试。同时将 Scheme 侧的 ahash-table 状态管理移到 C++ 侧（ChatSessionManager 已有完整的 CRUD），消除 Scheme/C++ 双重状态同步问题。

S7 的 `(import ...)` 与 texmacs `(use-modules ...)` 存在环境冲突：当被 `use-modules` 直接加载的模块包含 `import` 时，`tm-define` 注册到 rootlet 的函数运行时找不到 `define-expansion` 宏（如 `with-buffer`）。通过 chat-loader 无 import 入口解决，子模块可直接使用 `with-buffer`。

## 7 How

### 模块拆分策略
- `chat-tree-ops.scm`：纯函数，无 `(import ...)`，无副作用
- `chat-protocol.scm`：有 `(import (liii njson))`，包含回调、编码解码、发送逻辑、buffer 类型检测
- `chat-persist.scm`：有 `(import (liii njson) (liii os))`，包含持久化和路径管理
- `chat-loader.scm`：无 import 的薄入口，被 `init-research.scm` 直接加载

### 去状态化
- 移除 `chat-tab-session-states`、`chat-tab-style-loaded` ahash-table
- 移除 `chat-tab-state` 构造器及访问器
- `chat-tab-send` 签名从 `(session-id)` 改为 `(session-id model thinking)`，参数由 C++ 传入
- C++ `onThinkingToggled` 直接调用 `sessionManager_.setThinking`，不再调用 Scheme

### 持久化修复
- tmfs handler 机制（`tmfs-save-handler` / `tmfs-load-handler`）未被 `buffer-save` 正确路由，导致保存失败
- 改用 main 分支的 `buffer-export` 方案：直接将 buffer 内容导出为 TMU 文件
- C++ `exportBuffer` 从 `buffer-save` 改为调用 `chat-persist-export-buffer`
- 加载时用 `tree-import` + `buffer-set-body`，不经过 tmfs

### Buffer 检测统一
- `chat-input-buffer?`、`chat-message-buffer?`、`chat-buffer-session-id` 以 `tm-define` 导出在 `chat-protocol.scm`
- 用 `string-starts?` + `string-contains?` 替代 `string-ends?`，更鲁棒
- 6 个文件中的内联检查统一改为调用这些导出函数
- `search-widgets.scm` 中的本地 `define` 定义删除，改用 `chat-protocol` 的 `tm-define` 版本
