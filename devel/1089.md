## 2026/06/15 PDF 阅览器新增右键"另存为"功能

### What

在 PDF 阅览器组件 `PDFReaderWidget` 中添加右键菜单，支持"另存为"功能：
1. 新增 `showContextMenu` 函数，在右键事件中弹出 QMenu
2. 菜单含 "Save as..." 选项，点击后弹出文件保存对话框
3. 默认保存路径和文件名为当前 PDF 路径，筛选器为 `*.pdf`
4. 保存操作使用 `QFile::copy` 将当前 PDF 复制到用户选择的位置

### Why

此前 PDF 阅览器无右键菜单，用户无法方便地保存正在浏览的 PDF 文件。

尝试过沿用 TeXmacs 原有的 Scheme 菜单机制（`texmacs-popup-menu` / `menu-bind` / `popup_window_widget`），但最终选择直接在 C++ 中使用 `QMenu + QFileDialog`，原因如下：

1. **PDF 阅览器不是 TeXmacs 文档 canvas**：`PDFReaderWidget` 是独立的 Qt 组件，没有绑定 `edit_interface_rep`，无法复用 `edit_mouse.cpp` 中 `SERVER(menu_widget(...))` + `popup_window_widget` 的弹出路径（该路径依赖 `get_window(this)` 返回 `edit_interface_rep` 的窗口句柄）。
2. **现有先例一致**：项目中其他独立 Qt 组件如 `QTMHomePage::onRecentDocContextMenu`、`QTChatTabWidget` 的会话操作菜单，均直接在 C++ 中创建 `QMenu`，不经过 Scheme 层。
3. **保存操作为纯文件 I/O**：PDF 的"另存为"本质是 `QFile::copy`，不需要 TeXmacs 的 buffer / tree / converter 体系参与，直接使用 Qt 的 `QFileDialog` 即可，无需绕到 Scheme。

### How

在 `qt_pdf_reader_widget.hpp` 中：
- 添加 `showContextMenu(const QPoint& pos)` 公开函数
- 添加 `QString pdfFilePath_` 成员变量，记录当前打开的 PDF 路径

在 `qt_pdf_reader_widget.cpp` 中：
- `loadFromFile` 中保存文件路径到 `pdfFilePath_`
- `clear` 中清空 `pdfFilePath_`
- `eventFilter` 中捕获 `QEvent::ContextMenu`，调用 `showContextMenu`
- `showContextMenu` 中创建 QMenu，添加 "Save as..." 动作，执行 `QFileDialog::getSaveFileName` + `QFile::copy`
