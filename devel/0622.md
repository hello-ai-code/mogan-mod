# [0622] 修复 HTML 和 MathML 导出时 <dfrac>, <tfrac>, <cfrac> 被静默丢弃的问题

当 Mogan 导出包含 `<dfrac>`（显示分数）、`<tfrac>`（小分数）或 `<cfrac>`（连分数）的数学文档为 HTML/MathML 时，这些分数在导出的 MathML 文档中会被静默丢弃，只保留周围的算符和空白。

## 5 What
1. 修复了包含 `<dfrac>`, `<tfrac>`, `<cfrac>` 的数学公式在导出为 HTML/MathML 时被静默丢弃，导致公式内容残缺不全的问题。
2. 修复了右键选区 "Copy to" -> "Html" 到系统剪贴板（`key == "clipboard"`）时完全不进行 exec_html 预展开的系统缺陷，从而支持了右键局部公式的正确剪切板拷贝导出。
3. 实现了自动检测并无痕清洗 LaTeX 遗留行距（如 `[6pt]`, `[2pt]`）的机制，确保导出的 HTML 代码纯净度。
4. 随 PR 提交了标准测试用例：`0622_infinity_sizes.tmu` 到 `TeXmacs/tests/tmu/` 目录下。
5. 编写了完整的 Scheme 单元与集成测试 `TeXmacs/tests/0622.scm`。

## 6 Why
* **dfrac 丢失原因**：在 `tmhtml-expand.scm` 的 `tmhtml-env-patch` 中，`<dfrac>`, `<tfrac>`, `<cfrac>` 等宏被错误地注册为 "identity" 宏（使用 `tmhtml-env-macro` 转换）。这导致它们在导出预处理阶段不会被展开为基础的 `<frac>` 标签，而是保持字面上的 `dfrac`, `tfrac`, `cfrac`，从而作为未知标签被导出引擎静默丢弃。
* **选区拷贝失效原因**：在 `edit_select.cpp` 中，宏展开仅针对 X11 的 `"primary"` 和 `"mouse"` 选择区进行了逻辑覆盖，导致 macOS/Windows 系统默认的 `"clipboard"` 选区拷贝被完全跳过，绕过了宏展开。
* **行距残留原因**：Mogan 的 LaTeX 导入机制在处理 `\\[6pt]` 时，会把行距额外参数错误解析为下一个单元格首部的 `<around|[|6*p*t|]>` 字符。

## 7 How
1. 将 `tfrac`, `dfrac`, `cfrac` 从 `tmhtml-expand.scm` 的 `tmhtml-env-patch` 环境补丁列表中移除，使它们正常展开为带 `math-display` 属性的 `<frac>`。
2. 在 `edit_select.cpp` 中，扩充 key 的判定，将 `"clipboard"` 选区复制也纳入宏展开预处理：
   ```cpp
   if (key == "primary" || key == "mouse" || key == "clipboard")
   ```
3. 在 `tmhtml.scm` 中，增加 `strip-latex-spacing-artifacts` 深度遍历过滤器，全自动剥离所有不合规 of LaTeX 导入行距残留。
