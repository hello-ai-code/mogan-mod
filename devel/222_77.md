# 222_77 阻止 AppKit 覆盖 macOS 系统按钮自定义位置

## 如何测试

1. 编译：`xmake b stem`
2. 启动 Mogan，确认 macOS 系统按钮（traffic lights）在无边框窗口中位置正确。
3. 快速拖动窗口改变大小，确认系统按钮不被重置到默认位置。
4. 进入/退出全屏多次，确认按钮位置始终正确。
5. 打开 2+ 文档窗口，确认各窗口系统按钮独立且位置正确。
6. 关闭窗口后重新打开，确认无崩溃或异常行为。

## 2026/04/29 实现说明

### What

修改 `3rdparty/qwindowkitty/src/core/contexts/cocoawindowcontext.mm`，通过 method swizzling 拦截 `NSButton` 的 `setFrame:` / `setFrameOrigin:`，阻止 AppKit 内部 layout pass 覆盖自定义的系统按钮位置。另外微调 `qt_tm_widget.cpp` 中系统按钮的垂直偏移。

#### 修改文件

**3rdparty/qwindowkitty/src/core/contexts/cocoawindowcontext.mm**
- 新增 `proxyForButton` / `shouldBlockFrameChange` 辅助函数，提取 `setFrame` / `setFrameOrigin` 的公共逻辑。`shouldBlockFrameChange` 直接通过 `standardWindowButton:` 实时查询当前窗口的系统按钮进行指针比对，避免缓存带来的悬垂指针风险。
- `updateSystemButtonRect` 引入 RAII `UpdatingGuard`，确保 `isUpdatingSystemButtonRect` 在异常时正确恢复。
- `swizzleNSButtonSetFrame` / `swizzleNSButtonSetFrameOrigin` 增加 `class_addMethod` 返回值检查，失败时回退到 `method_setImplementation`。
- `restoreImplementations` 中给 `class_getInstanceMethod` 加 `nil` 保护，防止极端情况下崩溃。

**src/Plugins/Qt/qt_tm_widget.cpp**
- 系统按钮区域垂直偏移 `yOffset` 从 `(titleBarHeight - nativeTitleBarHeight) / 2` 改为 `DpiUtils::scaled(1)`，使按钮更加居中。

### Why

1. **AppKit 顽固覆盖**：222_75 通过 `setSystemButtonArea` + next-runloop defer 解决初始化时位置错误，但 AppKit 在窗口生命周期内任意时刻（如 resize、全屏切换后的动画）会再次重置按钮 frame。
2. **异常安全**：`updateSystemButtonRect` 内部调用 `setFrameOrigin:`，若中间抛出异常，`isUpdatingSystemButtonRect` 将永远为 true，导致后续所有 AppKit 内部修改被静默拦截。
3. **悬垂指针风险**：任何缓存 `NSButton *` 的方案在 AppKit 重建按钮实例后都可能失效，向已释放对象发消息会导致崩溃。改为每次实时查询 `standardWindowButton:` 可彻底消除该风险。
4. **防御性编程**：`class_addMethod` 在极端并发场景下可能失败，需有回退路径避免 old implementation 丢失；恢复 swizzle 时也对 `class_getInstanceMethod` 做 `nil` 保护。
5. **视觉微调**：原 `yOffset` 让按钮在 42px 标题栏中偏上一些，改为固定 1 个缩放单位后更接近原生 macOS 标题栏按钮位置。

### How

- Swizzle `NSButton` 的 `setFrame:` / `setFrameOrigin:`，在 `shouldBlockFrameChange` 中判断：若按钮等于当前窗口通过 `standardWindowButton:` 实时获取到的系统按钮，且 proxy 未处于 `updateSystemButtonRect` 主动布局状态，则忽略 AppKit 的 frame 重置。
- 使用 `UpdatingGuard` RAII 对象管理 `isUpdatingSystemButtonRect` 状态，确保无论正常返回或异常跳出均正确恢复。
- `systemButtons()` 每次直接查询 `standardWindowButton:`，不维护任何缓存，彻底消除悬垂指针和缓存同步问题。
