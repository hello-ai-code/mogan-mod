## What
修复通过文件管理器双击或右键 "Open with Mogan" 打开 tmu 文档时，标签栏没有启动页的问题，并清理死代码。

1. 修改 `TeXmacs_main`，无条件调用 `ensure_window()` 创建启动页后再处理命令行传入的文件
2. 第一个命令行文件改为在当前窗口打开（`new_buffer_in_this_window`），而非新建窗口
3. 移除不再使用的 `has_initial_file` 变量
4. 修复命令行打开文件未记录到最近打开列表的问题，加载后调用 `buffer-notify-recent`
5. 修复 `xmake run stem` 无法传递文件参数的问题，支持 `xmake run stem /path/to/file.tmu`
6. 添加集成测试 `1001_1.scm`

## Why
当 Mogan 启动时带有文件参数（如通过文件管理器双击打开 tmu），`TeXmacs_main` 原来会设置 `has_initial_file = true` 并跳过 `ensure_window()`。由于 `ensure_window()` 是唯一创建 `tmfs://startup-tab` 缓冲区的地方，新进程永远不会创建启动页。

此外，命令行路径直接调用 C++ 的 `buffer_load` 和 `new_buffer_in_this_window`，未经过 Scheme 的 `load-buffer` 流程，导致打开的文件不会被记录到最近打开列表中。

## How
### C++ 修复
- 将 `ensure_window()` 的调用移到文件处理循环之前，无条件执行
- 将 `load_welcome_doc()` 也移到 `ensure_window()` 之前
- 第一个命令行文件使用 `new_buffer_in_this_window` 在已创建的启动页窗口中打开
- 第一个命令行文件加载后，通过 `eval_scheme` 调用 `buffer-notify-recent` 记录到最近打开列表
- 后续文件继续通过 `exec_delayed` 执行 `switch-to-buffer` 打开

### xmake.lua 修复
- 在 `stem` target 的 `on_run` 回调中导入 `core.base.option` 模块
- 通过 `option.get("arguments")` 获取 `xmake run` 后面的用户参数
- 将用户参数追加到 `params` 列表中传给 Mogan 进程

### 测试
- `test-initial-state`：验证初始状态有且仅有启动页
- `test-load-buffer-preserves-startup`：验证 `load-buffer` 后启动页保留
- `test-load-buffer-adds-to-recent`：验证 `load-buffer` 后文件进入最近打开列表
- `test-close-startup-tab-protected`：验证 `cpp-buffer-close` 无法关闭启动页
- `test-multiple-buffers-startup-preserved`：验证加载多个文件后启动页仍保留
